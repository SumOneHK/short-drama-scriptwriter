# 项目目录与最终交付

## 一、默认项目根目录

```text
projects/<project-slug>/
```

项目根目录继续使用英文或拼音 `kebab-case`。  
具体产物文件统一中文。

## 二、最小稳定目录结构

下面这棵树展示用户明确确认方向并完成 `T02` 锁题后的 `默认稳定必出` 骨架，不把条件产物一起画进去。确认前的调研、候选池和沟通汇总只在对话里完成，不创建项目目录。

```text
projects/<project-slug>/
├── 项目状态.json
├── 00-立项/
│   ├── 项目简报.md
│   └── 锁题摘要.md
├── 01-策划/
│   ├── 项目设定.md
│   ├── 世界观设定.md
│   ├── 故事梗概.md
│   ├── 故事大纲.md
│   ├── 故事节拍表.md
│   ├── 人物小传.md
│   ├── 分集大纲.md
│   └── ...
├── 02-剧本/
│   ├── 第001集.md
│   ├── 第002集.md
│   └── ...
├── 03-交付/
│   ├── 完整剧本总稿.md
│   ├── 剧本圣经.md
│   └── 交付清单.md
└── 90-内部工作稿/
    └── 质检检查点.md
```

规则：
- `项目状态.json` 是全流程状态真相源，负责记录当前阶段、批次进度和各门禁 `qcStatus`。
- `上面这棵树不是最终全量树，而是默认稳定骨架。`
- 条件产物统一看 [21-产物矩阵与创建时机.md](21-产物矩阵与创建时机.md)。
- `分集大纲.md` 是正文前硬门槛，没有通过就不进入剧本。
- `分集大纲.md` 可以按批次补完，但只有全量完成并通过独立大纲质检后，才允许进入 `/剧本`。
- `故事节拍表.md` 是所有项目默认产物，用来先钉全剧关键翻面和阶段高潮。
- `世界观设定.md` 是所有项目默认产物，用来先钉故事依赖的秩序、规则、代价和后果；现实题材也保留轻量版本。
- `市场快照.md` 不是每个项目都默认生成；确认前不创建，确认后只有需要归档市场判断时才创建。
- `阶段大纲.md`、`场景参考卡.md`、`道具设定卡.md` 都是条件产物，不在默认骨架里预创建；但 `30` 集及以上项目默认必须补 `阶段大纲.md`。
- `人物小传.md` 默认同时承担角色剧情定位和最小资产锚点，不再并行维护第二份角色主文件。
- `剧本圣经.md` 只在 `/交付` 阶段汇总生成；`01-策划/` 不预创建策划汇总大文件。
- `90-内部工作稿/` 不作为主要交付层。
- `分集追踪.md` 建议在 `/策划` 通过后、首批正文前初始化，后续保持单文件累计回填。
- `质检检查点.md` 是所有独立质检共用的累计台账，不因阶段不同而拆分多份。
- 关键集的 `6` 节拍不单独立文件，默认并入对应批次的 `批次写作单-EPxxx-EPxxx.md` 或当前集写作备注。
- `分集追踪.md` 保持单文件累计回填。
- `批次写作单 / 单集达标检查 / 复盘报告` 默认按批次或里程碑命名，避免后续覆盖。
- `试写样场-*.md` 和 `改稿指令单-*.md` 都是可选工作稿，用来找人物声线、收束改稿，不替代正式大纲和正文。
- `试写样场-*.md` 只服务“这一场能不能立住”的试写，不承担连续性记录和批次规划。
- `批次写作单-*.md` 只服务当前批次写作，不是第二份分集大纲。
- `单集达标检查-*.md` 只服务某一集是否允许落稿或返工后是否重新过关。
- `分集追踪.md` 只服务累计稳定事实、悬念债务、关系变化和承接口径，不替代上游结构设计。

## 三、条件产物与禁止预创建

条件产物的"何时创建 / 触发条件 / 等级（必出 / 条件必出 / 可选工作稿 / 禁止预创建）"统一以 [21-产物矩阵与创建时机.md](21-产物矩阵与创建时机.md) 为单一真相源；本文不重复列清单，避免双份维护漂移。

判断顺序：
1. 先看 `21-产物矩阵与创建时机.md` 的 `T 编号 → 产物 → 等级 → 创建时机`
2. 默认骨架（本文第二节那棵树）只列必出产物；条件产物即使触发也按"按需创建"规则补，不预先 stub 空文件
3. 禁止预创建清单（如 `剧本圣经.md / 完整剧本总稿.md / 交付清单.md` 等正式产物只在对应阶段触发后才生成）以 21 矩阵的"禁止预创建"段为准

## 四、最小目录心智
- 先把 `00-立项 / 01-策划 / 02-剧本` 这三层主产物跑通
- `90-内部工作稿/` 不是默认每个文件都要生成
- 除 `分集追踪.md` 外，其余内部工作稿都按问题出现再启用

## 五、主流程产物落盘

### `/立项`

写入：
- 确认前：不创建任何项目文件
- 确认后必出：`00-立项/项目简报.md`
- 确认后必出：`00-立项/锁题摘要.md`
- 确认后按需归档：`00-立项/市场快照.md`
- 确认后按需归档：`00-立项/待选题材池.md`
- 确认后按需归档：`00-立项/故事理解.md`（仅改编项目）

### `/策划`

写入：
- `01-策划/项目设定.md`
- `01-策划/世界观设定.md`
- `01-策划/故事梗概.md`
- `01-策划/故事大纲.md`
- `01-策划/故事节拍表.md`
- `01-策划/人物小传.md`
- `01-策划/阶段大纲.md`（`总集数 >= 30` 默认必出；`20-29` 集强推荐）
- `01-策划/分集大纲.md`
- `01-策划/场景参考卡.md`（精品或资产重项目可提前产出工作版）
- `01-策划/道具设定卡.md`（精品或资产重项目可提前产出工作版）

硬门槛：
- 先评审 `T04-T09`：项目设定、世界观设定、人物小传、场景/道具锚点、故事梗概、故事大纲、故事节拍表、阶段大纲（命中产物矩阵时）
- 再进入 `T10 分集大纲`；如果 `T08-T09` 暴露结构问题，先回补世界规则、角色压力链、场景/道具锚点
- `分集大纲.md` 全量完成并通过独立 `大纲质检` 后，才进入 `/剧本`

### `/剧本`

写入：
- `02-剧本/第001集.md`
- `02-剧本/第002集.md`
- `...`

规则：
- 剧本可以分批生成
- 但最终统一落到 `02-剧本/`
- 不把批次目录当成最终交付结构

### `/交付`

写入：
- `03-交付/完整剧本总稿.md`
- `03-交付/剧本圣经.md`
- `03-交付/交付清单.md`
- `03-交付/制作理解稿.md`（`deliveryMode = production-enhanced` 时）
- `03-交付/制作交付参数表.md`（`deliveryMode = production-enhanced` 时）
- `03-交付/场景参考卡.md`（`deliveryMode = production-enhanced` 且需要场景交底时）
- `03-交付/道具设定卡.md`（`deliveryMode = production-enhanced` 且需要道具交底时）
- `03-交付/海外制作版分场表.md`（海外项目且 `deliveryMode = production-enhanced` 时）

补充规则：
- `剧本圣经.md` 标准交付默认汇总 `卖点与项目定位 / 项目设定 / 故事梗概 / 故事大纲 / 故事节拍表 / 阶段大纲（若有） / 世界观设定 / 人物小传`
- 只有进入 `production-enhanced` 或交付层已有正式 `场景参考卡 / 道具设定卡` 时，`剧本圣经.md` 才纳入场景和道具章节
- `完整剧本总稿.md` 只汇总正式单集里的 `场次正文`，不整合 `本集创作简卡 / 本集回填 / 复盘报告 / 检查单 / 试写样场 / 批次写作单`
- 如果 `01-策划/` 里只有工作版 `场景参考卡 / 道具设定卡`，交付阶段要先补成最终版，再进入 `剧本圣经.md`
- `03-交付/场景参考卡.md` 和 `03-交付/道具设定卡.md` 是最终制作资产卡；`剧本圣经.md` 只是把它们嵌入为汇总副本，不能因为圣经里已有对应章节就删除独立卡

### 内部工作稿层

按需写入：
- `90-内部工作稿/批次写作单-EP001-EP005.md`
- `90-内部工作稿/单集达标检查-EP001-EP005.md`
- `90-内部工作稿/试写样场-*.md`（按需）
- `90-内部工作稿/复盘报告-策划阶段.md`
- `90-内部工作稿/复盘报告-EP001-EP005.md`
- `90-内部工作稿/复盘报告-交付前.md`
- `90-内部工作稿/改稿指令单-*.md`（按需）

累计稿：
- `90-内部工作稿/分集追踪.md`

用于质量控制，不替代主产物。
