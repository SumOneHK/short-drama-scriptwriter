# 写作方法索引

这份文件是 skill 的 `场景视角入口`，与 `SKILL.md`（流程视角入口）形成双轨。

- `SKILL.md`：项目从立项到交付的固定流程（T01-T18），适合新项目从头跑
- `22-写作方法索引`（本文）：按"我现在遇到什么问题"找方法，适合中途接手 / 卡壳排查 / 局部优化

原则：
- 先选当前阶段或当前场景真正需要的方法，不一次性加载全部方法
- 方法只服务写作判断，不复制进正式产物
- 正式分集稿仍然保持轻模板，复杂判断留在工作稿和质检里

## 一、按阶段选方法

### `/立项`

重点看：
- 目标市场、目标受众、题材类型和制作边界是否已经锁定
- 项目能不能一句话说清“为什么会被点开”
- 题材、核心设定、风格元素是否互相服务，而不是混乱叠加

输出判断：
- 观众为什么点开
- 为什么能连载
- 为什么能拍
- 哪些内容不能做

### `/策划` T04-T09

重点看：
- 故事引擎：主角强欲望、强阻碍、代价和阶段兑现是否成立
- 主角代入：是否有可投射身份、站队点、受压点、翻盘入口和强反击方式
- 主角饱满感：是否同时有平凡面和不平凡面，反差是否能落到行为
- 世界观/角色/场景/道具基底：规则是否压迫人物，人物行动是否触发代价，场景和道具是否能承载可见冲突
- 故事节拍：关键翻面、假稳、最低谷和终局是否已经钉住
- 市场本地化深度：题材 / 矛盾 / 压迫机制 / 爽点是否长在目标市场真实社会纹理里
  - 国内：是否用了 `直播 / 校园 / 末日 / 体制 / 医疗 / 法律 / 商战 / 互联网大厂 / 豪门家族` 的原生机构和原生道具，而不是所有题材都写成"保安 / 登记册"
  - 海外：是否用了 `restraining order / NDA / shadow ban / cancellation / Common App / Ivy League` 等美国原生机制
- 价值观与 trope 红线：是否符合目标市场主流爽点；如果是跨市场母体，是否完成了 trope 重写矩阵

输出判断：
- 这部剧为什么成立
- 主角为什么值得跟
- 对手为什么有压迫
- 中段为什么不会塌
- 这部剧为什么是"目标市场原生"作品，不是直译版

### `/策划` T10 和 `分集大纲`

重点看：
- 一卡：前三集定类型、立人物、抛主危机；前十集持续兑现同一主卖点
- 单集存在性：每集删掉后是否会丢掉不可替代的过程、翻面、情绪或后果
- 钩子机制：钩子是否改写局势，而不是只补信息
- 悬念机制：是否有明确的信息差，并且挂在人身、关系、输赢或尊严上
- 拉扯机制：是否有压低、错误放松、再压和阶段爆发

输出判断：
- 每集为什么必须存在
- 每集结尾如何改变下一集目标、危险、关系或信息差
- 每批如何避免重复回报、重复钩子和重复场次骨架

### `/剧本`

强制前置：
- 写每一集前先按 [12-单集写作协议.md 第二·补章 大纲→场次执行稿转化五步法](12-单集写作协议.md) 走一遍：观众口径压一句 → 四链拆解 → 场次分配 → 每场动作骨架 → 首场前 4 行硬规则
- 三套样例覆盖三类项目：高概念走"看见后果再讲规则"路径；现实主义走"行动+发现+反击"路径；海外走"contractions+中文注释补潜台词"路径
- 五步法骨架不全（缺第二阻力或翻面）就开始填台词的，回改 12 问，不直接落正文

重点看：
- 单集情节四链：情节承接、冲突升级、价值闭环、下集勾连
- 三种密度：情绪密度负责停留，信息密度负责理解，情节密度负责追更
- 短剧节奏：开头先情绪冲击，前段给局势变化，中后段形成强期待，结尾再抛钩子
- 画面感：写时间、地点、光影、细节、动作和少量关键镜头
- 对白：先动作，再反应，最后一句台词补刀；能展示就别解释

输出判断：
- 开场是否抓人
- 过程是否真的发生冲突和变化
- 本集是否有可复述的高光、痛点、爽点或关系翻面
- 结尾是否让人想立刻看下一集

### `/复盘`

重点看：
- 策划视角：阶段推进、信息投放、悬念拉扯和题材引擎是否还成立
- 编剧视角：人物声音、主冲突、主回报、承接、尾钩和场次动作是否成立
- 导演视角：场景可执行性、视觉支点、资产复用和空间调度是否成立
- 全剧交付前：按用户 / 观众、编剧、导演 / 制作三种视角完整读完全季，专门查机械质检难发现的疲劳、重复、情绪不落地、终局过满和制作不可执行问题

输出判断：
- 通过、需修改或阻塞
- 当前最严重的 `1-3` 个问题
- 下一批必须延续什么、禁止再犯什么

## 二、生成前优先动作

### 1. 主角可写性

写主角时先确认：
- 普通身份或可投射身份
- 观众愿意站队的特质
- 开局扎心遭遇或明确受压点
- 幻想机遇或翻盘入口
- 替观众完成的强反击方式
- 平凡面和不平凡面的反差

如果主角只剩职业、身份或性格标签，先回改人物小传，不直接写正文。

### 2. 首次出场

主角、核心关系位、核心对手首次出场时，优先选一种核心印象：
- 神秘
- 压迫
- 反差
- 高光
- 救场

常用方式：
- 局部特写
- 动作登场
- 配角衬托
- 声音登场
- 场景反差
- 道具登场
- 氛围铺垫

一次出场只服务一个最核心印象，不把所有方式硬塞进同一场。

### 3. 单集情节四链

每集落正文前先写清：
- 情节承接：上一集的果，如何变成本集的因
- 冲突升级：本集哪里让危机、关系或输赢发生升级
- 价值闭环：本集结束后，主角处境、认知、立场或行动方向变了什么
- 下集勾连：下一集必须回答的问题是什么

四链缺一项，单集容易变成功能桥段。

### 4. 钩子类型

每集钩子优先落到具体变化上：
- 新人物
- 新物品
- 新状况
- 身份颠覆
- 人性背刺
- 胜负逆转
- 真相反转

如果钩子只是一个资料名、抽象名词或“另有隐情”，默认不成立。

### 5. 信息差结构

悬念优先从三种信息差里选：
- 观众知道，角色不知道
- 观众不知道，角色知道
- 观众和角色都只知道一部分

信息差必须挂在人物安危、关系、输赢、尊严或情感上；纯设定谜题不优先。

### 6. 密度检查

写作顺序建议：
1. 先用情节密度搭主线骨架
2. 再用信息密度删废话和前置信息
3. 最后用情绪密度把关键节点打到观众心里

单集最小判断：
- 情绪：开头有没有明确情绪，中段有没有小释放或再压
- 信息：有没有新信息，且不是只靠解释台词
- 情节：有没有承接、升级、价值转变和下集勾连

密度口径冲突时的优先级：
- `11-密度设计.md`：给"密度档位 + 推荐场次数"作为创作起点（写之前的目标值）
- `12-单集写作协议.md` 第六章：给"行数硬阈值 + 场次内动作骨架"作为正文硬门槛（写之后的判稿值）
- 冲突时**以 12-单集写作协议 为准**：场次数可以妥协，正文厚度和动作骨架不可妥协
- 实操：如果按 11 推荐的 `3` 场写不出 12 要求的 `28` 行，先考虑合并 / 拆集 / 提密度档；不要降低正文硬阈值

### 7. 展示优先

对白和画面的稳定顺序：
1. 先让动作发生
2. 再写他人反应
3. 最后给一句真正有分量的台词

优先用动作、道具、眼神、停顿、空间变化和群体反应传递身份、威胁、目的和权力关系。

### 8. 对白口语化与本地化

完整方法库见 [12-单集写作协议.md 第七章](12-单集写作协议.md)。`/剧本` 阶段每场写完都要过这条最小检查：

- 单句长度：中文 `<= 15` 字 / 英文 `<= 12` word；超过就拆
- 国内项目：是否仍有书面化、剧本腔、翻译腔；称呼系统是否符合关系层级
- 海外项目：是否完全 contractions 化；中文注释是否补潜台词而不是字面翻译
- 声线分层：同场对手两人句长 / 句式 / 词汇锐度能否听出差别
- 删掉"最说明的那一句"后这场是否仍然成立

国内市场细则补充见 [07-抖音国内市场指南.md](07-抖音国内市场指南.md)，海外市场细则补充见 [08-欧美市场指南.md](08-欧美市场指南.md)。

## 三、四工作稿职责决策树

写正文阶段有四份工作稿（试写样场 19 / 批次写作单 07 / 单集达标检查 08 / 改稿指令单 20）容易职责混淆。按以下决策树挑：

```
当前要做什么？
├─ 还没开始写本集，但人物声音 / 关系张力 / 关键场拿不准
│  └─→ 试写样场 19（写 1-3 个样场试出来再正式推批次）
│
├─ 准备开始当前批次，要锁先写哪 1-2 集代表样稿、关键集 6 节拍、批次旧债回收和新债预算
│  └─→ 批次写作单 07（每批一份；只服务本批次决策，不代替分集大纲）
│
├─ 单集已经写了第一版，要判断"现在能不能正式落稿"
│  └─→ 单集达标检查 08（覆盖单集硬门槛；不通过先回去改）
│
└─ 单集 / 批次已经发现明显问题，要把改动收束成可执行的修改单
   └─→ 改稿指令单 20（按"局部场次 / 单集 / 整批 / 上游结构"分级；列出必改项和判定标准）
```

互斥规则（三个最容易出错的边界）：

- `批次写作单 07` 是"写之前"的决策稿；`单集达标检查 08` 是"写之后"的判分稿；`改稿指令单 20` 是"判分不过之后"的修改清单。三者按时间先后串联，不互相替代
- 单集只有 `1-2` 个具体问题且判断清晰时，直接在 `单集达标检查 08` 里写改法，不另起 `改稿指令单 20`；`改稿指令单 20` 只在问题数 ≥ 3、跨多集或意见冲突时启用
- 试写样场 19 不是分集正文的草稿；样场过关后正文要重写，不允许把样场直接当 `1-1` 落盘

启用顺序的最小路径：
- 大多数项目：批次写作单 07 → 单集达标检查 08（不通过时再补改稿指令单 20）
- 长线 / 海外 / 精品项目：试写样场 19 → 批次写作单 07 → 单集达标检查 08（按需 改稿指令单 20）

## 四、常见误用

- 把方法索引复制进正式项目文件
- 在正式分集稿里塞完整检查单
- 只填“钩子类型”，正文没有对应的局势改写
- 写了“信息差结构”，但所有人其实知道同样的信息
- 把情绪密度写成持续大喊，把节奏快写成不停吵架
- 用一堆镜头术语替代场次动作
- 写人物反差只停在设定里，正文没有行为落点
