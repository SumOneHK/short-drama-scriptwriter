# 质量门禁

## 问题分级

- `P0`：根本性问题，必须回退，不能推进
- `P1`：明显削弱成片质量，原则上本阶段修完再走
- `P2`：优化项，可在下一轮统一处理

## 各节点最低通过条件

所有门禁都必须由模型读实际内容后判断。格式、字段、行数只是辅助观察，不能代替戏剧判断；自动脚本、关键词扫描、平均分和无发现输出都不能作为放行依据。

放行只认独立复检结果，定义口径以 [18-过程质检与独立复检.md](18-过程质检与独立复检.md) 的 `clean-context 硬性判定` 为准。同一会话内的自检不能写成门禁通过，无论说法是 `自我复审 / 二次评估 / 切换视角再读 / 我现在以审稿人身份重读` 都属于自检。`clean-context / subagent` 的可用条件、授权要求和证据字段只在 `18` 维护，本文件不重复定义执行方式。

### `立项质检`

覆盖任务：`T01-T03`

必须同时满足：
- 市场、受众、题材、规格、AI 风格明确
- 题材和方向已由用户明确确认，且 `项目简报.md / 锁题摘要.md` 写清确认依据；只有系统推荐、市场排序或”建议优先”不算锁题
- 已完成最小市场判断
- 主卖点能一句话讲清
- 风险项和不做项明确
- 已经能解释”为什么能连载、为什么适合 AI 竖屏呈现”
- 合规风险预判已完成；命中敏感题材时已在 `项目简报.md / 锁题摘要.md` 写清规避策略
- 商业化模式已明确写成 `商业化后置` 或具体模式（`IAA / IAP / 付费 APP / 平台付费定制`）；未明确商业化时不得默认按 IAA 处理；命中具体商业化模式时必须显式锁定对应策略骨架

不通过怎么办：
- `qcStatus.kickoff = 需修改`
- `currentStep = 立项`
- 不进入 `策划基底`

### `策划基底质检`

覆盖任务：`T04-T09`

必须同时满足：
- `项目设定 / 世界观设定 / 人物小传 / 故事梗概 / 故事大纲 / 故事节拍表` 已形成可判断版本
- 核心关系、主线目标、主要对手、题眼持续机制成立
- `T05-T07` 已形成同轮校准后的最小故事基底：世界规则能压迫人物，角色行动能触发规则代价，场景/道具能承载可见冲突
- `T07` 场景与道具至少已有资产锚点；资产重项目已说明是否单独出卡
- `T08` 的梗概和大纲互相支撑，不是两个口径
- `T09` 的节拍和阶段设计能看出关键翻面、阶段回报和中段变奏
- 不是只有包装，没有结构引擎

不通过怎么办：
- `qcStatus.planningFoundation = 需修改`
- `currentStep = 策划基底`
- 不进入 `分集大纲`

### `大纲质检`

覆盖任务：`T10-T11`

必须同时满足：
- `分集大纲.md` 已全量完成
- 每集存在性理由明确
- 每集有明确观看回报和继续观看动力
- 前中后段没有明显重复桥段
- 尾钩、承接、阶段回报和中段变奏成立
- 剧作层追更设计成立：每集已写清 `本集观众一句话 / 本集情绪回报 / 本集尾钩 / 尾钩如何改变下一集目标、危险、关系或信息差`
- 若商业化模式不是 `商业化后置`，商业卡点已按项目商业化模式落地（详见 `references/24-卡点设计.md`）

不通过怎么办：
- `qcStatus.outline = 需修改`
- `currentStep = 分集大纲`
- 不进入 `剧本批次`

### `剧本质检`

覆盖任务：`T12-T14`

必须同时满足：
- 代表样稿和当前批次对上大纲
- 每集读完能说出具体吸引点、记忆点和观看回报
- 角色声音稳定，且对白满足本项目的 `口语化 / 本地化` 硬要求
- 国内项目：对白没有书面化、剧本腔、翻译腔、解说词；称呼系统符合关系层级；网感词只用稳定词且不撑主角
- 海外项目：英文对白 contractions 化（`I'm / don't / won't / gonna`）；每句紧跟中文注释（默认 `<= 15` 字、补潜台词不是字面翻译）；没有 `How dare you / You are courting death` 类翻译腔；register 分层（teen / young adult / professional / antagonist / parental）正确
- 正式稿格式符合标准场次执行稿：场次标题、人物行、动作行、台词行和 `OS / VO / 屏幕字 / SFX` 标记完整
- 场次不是结果摘要，而是能看到人物目标、阻力升级、反应选择、二次压迫、翻面和余波
- 开场承接、主回报、尾钩都落到正文
- 当前集的有效正文足以支撑目标时长；如果场数少，场内必须有足够动作升级和对白拉扯
- 模型复检能指出至少一个真实有效的高光场面；如果只能复述简卡，不算通过
- 同批次至少抽查 `1-2` 集做声线一致性检查：把对白单独提出去掉 `角色名：`，仍能凭句式分辨是谁在说

不通过怎么办：
- 当前批次 `scriptBatches[].status = 需修改`
- `currentStep = 剧本批次`
- 不进入下一批，也不进入 `交付整合`

## 放行原则

- 任何 `P0` 存在时，禁止推进
- `P1` 是否允许带过，必须在独立质检里明确写理由
- 如果拿不准是否算 `P0`，先按 `P0` 处理

## 特别禁止

- 没有全量大纲质检通过就写正文
- 批次正文没过独立质检就继续下一批
- 用同一上下文里的自评替代独立复检（包括"换视角再评""自我审稿""切换为读者身份"等任何同会话内的二次评估）
- 模型自行声称"现在切换为 clean-context 模式"或"我以独立复检身份判断"；`lastIndependentReviewMode = clean-context` 只能在用户明确发出复检指令且实际发生隔离复检时写入
- 用户没有明确发出质检/复检指令时，模型自行 spawn subagent 或隔离任务跑质检并把结果当门禁
- 用脚本质检、自动评分或字段扫描替代模型读稿复检
- 把过程工作稿混进最终交付
- 上游已经改动却不回退下游门禁，直接继续往下写
