# AI 漫剧制作口径

本文是 `AI 制作竖屏剧 / 漫剧` 的制作口径单一真相源。它不用于限制题材，不把奇幻、大场面、跨时空、群像或高概念自动判为不可做；它用于让剧本为下游 AI 图像 / 视频 / 分镜 / 配音流水线提供清楚、稳定、可复用的创作接口。

## 底层逻辑：AI 制作能力服务剧本，不替剧本降级题材

AI 制作竖屏剧 / 漫剧的核心特征：
- 用 AI 图像 / 视频模型生成单帧、短片或镜头段，再经剪辑、配音、字幕和后期合成
- 可以承载写实、AI 真人、3D、国漫、日漫、欧美漫画、赛博、暗黑哥特、romantasy、Isekai、规则世界等多种风格
- 视频模型能力持续增强，因此选题阶段不要因为“真人拍摄难”或“传统动画成本高”自动排除高视觉题材

由此带来的核心要求：
1. `角色识别锚点`：同一角色要有稳定外形、服装、道具或姿态锚点，不能只靠“长得好看”维持辨识
2. `视觉系统锚点`：高概念、奇幻和异世界要有可复用的颜色、符号、场景和道具系统
3. `规则后果可见`：世界规则不能只靠台词解释，必须在动作、道具变化、空间变化或人物代价里看见
4. `情绪表达可见`：复杂微表情不稳定时，用动作、沉默、道具、屏幕字、OS / VO 辅助表达
5. `镜头接口清楚`：动作行要能被下游拆成镜头，不把多个复杂动作糊成一长句
6. `口型和配音可控`：长台词拆短句，必要时用反应镜头 / OS / VO 缓冲
7. `商业成本不前置`：除非用户明确要求预算控制或制作降本，本文件不作为选题降级依据

## 一、稳定锚点 vs 高复杂度：写正文时主动交代清楚

### 稳定锚点（AI 制作更容易保持一致）

`场景类型`：
- 单一固定场景（同场内角色对峙不切镜）
- 室内对话场（光线稳定）
- 半身或近景镜头
- 同一角色单镜头（独白 / 沉默）

`动作类型`：
- 站立对峙
- 慢速走位（但不要走出场景）
- 道具特写（戒指 / 文件 / 手机）
- 静止反应（眼神 / 微表情简单）
- 服装单一状态下的对话

`叙事手段`：
- 屏幕字推进（手机短信 / 文件 / 弹幕）
- OS / VO 推进
- 道具镜头（戒指 / 项链 / 信件 / 手帕）
- 声音设计（SFX / 音乐切入）

`节奏切换`：
- 场内余震镜头 / 表情停顿
- 慢推 / 慢拉镜头（同帧渐变）

### 高复杂度（不是禁用项，但要写清楚）

`场景类型`：
- 室外大场景（光线复杂）
- 多角度同场景（每切一个角度都要重新生成保持一致）
- 历史 / 古代场景（道具多 / 服装繁复）
- 群像场景（3+ 人同框）
- 跨时空切换场（每个时空都要单独建场）

`动作类型`：
- 高速动作（跑步 / 追逐 / 打斗）
- 翻滚 / 跳跃 / 爬高
- 舞蹈 / 武打 / 大幅肢体动作
- 拥抱 / 接吻（角色面部接触一致性极难）
- 复杂物理交互（吃饭 / 喝水 / 抽烟）

`叙事手段`：
- 大场面奇观（爆炸 / 变身 / 魔法）
- 多角色同时出现的群像戏
- 长镜头（角色长时间在镜头内说话，口型对齐压力大）
- 闪回（场景切换 + 时间标记 + 角色变体造型）

`节奏切换`：
- 快速切镜头（每个镜头都是新生成）
- 蒙太奇（多场景叠化）

### 高复杂度场的写作转化

如果高复杂度桥段只是为了“看起来大”，优先改成更清楚的角色行动；如果高复杂度桥段是题材核心，可以保留，但要拆清楚视觉锚点、动作顺序和规则后果：

| 高复杂度原型 | 更适合 AI 下游执行的写法 |
| --- | --- |
| 男女主激烈打斗 5 场 | 每场只写一个明确胜负动作 + 一个道具或伤痕变化；避免连续动作糊成一段 |
| 主角追着反派跑过半个城市 | 拆成“发现目标 / 追逐阻断 / 关键拦截 / 反派逃出”四个可见节点 |
| 婚礼大场面 100 人参加 | 保留大场面，但把剧情爆点落在新娘、戒指、誓词、宾客反应屏幕字等可识别锚点 |
| 主角变身 / 觉醒奇观 | 写清觉醒前状态、触发物、视觉变化、代价和关系翻面 |
| 群像大战 | 按阵营或关键角色拆镜；每个镜头只服务一个输赢变化 |
| 一场连续的长对白（90 秒同人物） | 拆成 3 段，每段穿插对方反应 / 道具特写 / OS 短句缓冲 |

## 二、角色一致性管理

### 一致性预算

每个项目需要先规划“角色识别系统”，用于帮助下游维持角色一致性：

- 主角：默认 3-5 套造型变体（日常 / 工作 / 重要场合 / 危机 / 终局）
- 核心关系位 / 核心对手：2-3 套
- 主要配角：1-2 套
- 一次性角色：1 套（且不应在剧情中反复出现）
- 群众 / 路人：可用 AI 默认或换不同人脸（不需保持一致）

角色多、造型多不是禁止项；但一旦角色承担长线叙事功能，必须有固定识别锚点和造型触发条件。

### 造型锚点

每个保留一致性的角色必须在 `人物小传.md` 写清：
- 外形关键词：发型 / 发色 / 眼睛 / 体型 / 肤色（短词，可在 prompt 里复用）
- 固定识别锚点：每次出现都不变的标志（一个特定首饰 / 一道伤疤 / 一种眼神 / 一种姿态）
- 服装变体清单：每套衣服按 `造型 1：日常款 / 造型 2：婚礼款` 等具体描述
- 不允许"看 AI 怎么生成"——必须先锁住

### 一致性失败的兜底

如果某一集出现角色一致性问题（脸 / 体型 / 服装漂移），优先按以下顺序兜底：
1. 用近景 / 半身 / 背影代替全身（减少需要一致的元素）
2. 用 OS / VO / 屏幕字补足角色辨识（不依赖看脸）
3. 用固定识别锚点（戒指 / 伤疤 / 标志性物件）做识别
4. 重新生成（最贵，最后选）

写场次执行稿时主动给一致性失败留兜底空间——不要把全片唯一辨识落点放在角色脸上。

## 三、镜头组织建议

### 单集场景组织

按目标时长给出场景组织建议（不是硬限制）：
- 45-75 秒：2-3 场（同场尽量延伸）
- 60-90 秒：3-4 场
- 90-120 秒：4-5 场
- 90-150 秒首集：5-6 场（首集允许更多场，但每场要充实）

如果一集必须使用更多场景，重点检查是否每个场景都有不可替代的冲突、信息、情绪或视觉后果；没有就合并。

### 单场镜头组织

单场内的镜头数（同场不同角度）：
- 默认 3-5 个镜头
- 同一镜头可用 2 次（如 1-1 开场镜头 + 1-1 结尾回应同视角）
- 不超过 8 个镜头单场（再多就该拆场）

### 场景复用率

一个常驻场景在全剧出现的次数：
- A 级常驻（如主角家 / 公司 / 反派老巢）：≥ 10 次
- B 级阶段（如某段时期专用）：3-8 次
- C 级一次性：1-2 次

资产重项目要先明确高复用场景、阶段场景和一次性奇观场的职责；不要为了换场景而换场景。

## 四、AI 配音和口型对齐

### 配音特点

AI 配音（豆包 / 火山引擎 / 11Labs / Sora 配音等）：
- 长句和复杂情绪台词不稳定
- 短句、克制、明确情绪稳定
- 口型对齐有限（尤其多音节英文）

写作要求：
- 单句默认 ≤ 15 字（中文）/ ≤ 12 word（英文）；这一硬规则在 AI 漫剧中是配音稳定性的硬约束，不只是节奏要求
- 复杂情绪用 `△ 动作` + 短台词组合，不要用一句长台词承担全部情绪
- 大笑 / 哭嚎 / 嘶吼这类极端声音需在 `△` 标 SFX 而不是写成台词

### 口型对齐策略

减少口型对齐压力：
- 多用反应镜头（角色在听不在说）
- 多用 OS / VO 替代长对白
- 长台词拆成段，中间穿插 `△` 动作
- 海外项目优先选不强调口型的镜头景别（中景 / 远景）

## 五、奇观与特效组织

奇观和特效不作为选题禁区，但要按“可见后果”组织：
- 静态光效（背光 / 高光 / 阴影）— 稳定锚点
- 简单粒子（雪 / 雨 / 火星）— 稳定锚点
- 物体发光（道具 / 文字闪烁）— 中复杂度
- 烟雾 / 尘土 / 水花 — 中复杂度
- 爆炸 / 火灾 — 高复杂度
- 角色变身 / 飞行 / 隐身 — 高复杂度
- 大场面战斗 / 群体施法 — 高复杂度

写作要求：
- 高概念项目每次奇观都要改变目标、危险、关系或信息差
- 单集可以有多个特效点，但每个特效点必须服务一个明确戏剧动作
- 不要用连续奇观替代人物选择；奇观越强，人物代价越要清楚

## 六、与策划阶段的对接

写 `项目设定.md / 世界观设定.md / 故事大纲.md / 分集大纲.md` 时，必须按本文校准视觉系统和下游接口：

- `项目设定.md` 资产与制作 section 要写清 AI 制作风格、角色识别锚点、核心视觉符号和下游资产口径
- `世界观设定.md` 必须写清规则后果如何被看见，而不是只写规则名
- `故事大纲.md` 的"中段升级"和"后段失控"要靠人物选择、代价、关系翻面和视觉符号升级撑住，不靠空换场景
- `分集大纲.md` 的"本集主要场景"字段要说明该场景承担什么冲突或翻面

## 七、与剧本阶段的对接

写 `第XXX集.md` 时：
- 单集场景数按本文 §3.1 检查每个场景是否不可替代
- 单场镜头数按 §3.2 拆成可执行动作节点
- 配音稳定性按 §4
- 奇观与特效组织按 §5
- 高复杂度场按 §1.3 表格主动拆清楚

`场次执行稿模板`（05 / 06）的动作行 `△` 在写复杂动作时，要拆成清楚的动作节点；如果是大场面或复杂奇观，在场次前置字段标注 `⚠ 高复杂度场`，让制作交付时主动规划。

## 八、与制作交付的对接

进入 `production-enhanced` 时，制作交付包必须按本文为下游分镜 / 视频 / 配音引擎补充：

- `场景参考卡` 的视觉锚点必须包含 AI 提示词可复用的关键词
- `道具设定卡` 的关键道具要写清是否需要特写、特写时如何保持一致性
- `角色资产摘要` 的造型变体要标注每套造型对应的剧情触发条件
- `制作交付参数表` 要明确写"高复杂度场清单"，让分镜引擎主动规划

详见 `14-制作交付整理.md` 和 `21-产物矩阵与创建时机.md` 的 production-enhanced 段。

## 九、与下游引擎的接口契约

本 skill 只到 `场次执行稿`，不生成分镜 / 视频 / 配音脚本。但 `场次执行稿` 必须为下游引擎留下可消费的接口：

- 场次标题 `### 1-1 日内 地点`：分镜引擎以此切分
- `人物：` 行：角色一致性引擎以此挂载造型 ID（资产 ID 命名见 `14-制作交付整理.md` 资产命名统一规范）
- `△ 动作行`：视频引擎以此生成镜头 prompt（建议每个 `△` 一个镜头）
- `角色名：` 台词行：配音引擎以此分配人声
- `OS：/ VO：`：配音引擎独立人声轨
- `屏幕字（类别）：`：合成引擎以此生成屏幕文字图层；类别建议保留 `叠加 / 手机 / 文件 / 系统 / 姓名牌` 等，用于区分叠加提示字、画内信息字和环境标签字
- `SFX：`：音效引擎以此插入音效

写正文时不要破坏这些标记的可解析性：
- 标记必须在行首
- 一个 `△` 行内描述一个镜头单元，不要写成长段落
- 屏幕字独立成行，不要嵌入对白行内；类别写在标记括号里，不写进正文说明
- 复杂动作分多个 `△` 行写，每行一个具体动作
