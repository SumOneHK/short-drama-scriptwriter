# AI 漫剧制作约束

本文是 `AI 漫剧` 这一品类特有制作约束的单一真相源。这部 skill 的名字里有"AI 漫剧"四个字，意味着所有创作选择都要先过这套制作可行性。

## 底层逻辑：AI 漫剧不是 AI 真人剧也不是动画

AI 漫剧的核心特征：
- 静态或半静态画面 + 镜头运动 + 角色一致性 + AI 配音 + 后期合成
- 不是真人拍摄；不是 3D 动画；不是 2D 帧帧绘制
- 用 AI 工具（Midjourney / Sora / 通义万相 / 即梦 / Kling 等图像/视频引擎）生成单帧或短片，再经剪辑、配音、字幕拼接成片

由此带来的核心约束：
1. `角色一致性` 是头号瓶颈：同一角色在不同镜头里很难保持完全一致的脸 / 服装 / 体型；变体越多越烧钱
2. `镜头切换成本` 高：每切一个新场景或新角度都意味着重新生成；不是真人剧"换镜头几乎免费"
3. `物理动作复杂度` 受限：跑酷、打斗、跳跃、翻滚等高动态镜头 AI 生成不稳；静态对峙 / 半身镜头反而稳定
4. `群像成本` 极高：3+ 人同框时一致性更难；每多一个人面部和服装都要单独控制
5. `特效奇观` 昂贵：爆炸、变身、奇幻视觉效果生成不稳定，且每次生成都要单独排查瑕疵
6. `表情精度` 受限：复杂微表情（眼神戏、克制、暗示）AI 生成不到位；偏向用对白 + 动作 + 道具表达情绪
7. `口型对齐` 一般：AI 配音的口型和 AI 生成的脸难完全对齐；优先用反应镜头 / OS / VO 缓冲

## 一、廉价 vs 昂贵：写正文时主动避坑

### 廉价（AI 漫剧能稳定产出）

`场景类型`：
- 单一固定场景（同场内角色对峙不切镜）
- 室内对话场（光线稳定）
- 半身或近景镜头
- 同一角色单镜头（独白 / 沉默）

`动作类型`：
- 站立对峙
- 慢速走位（但不要走出场景）
- 道具特写（戒指 / 文件 / 手机）
- 静止反应（眼神 / 微表情简单）
- 服装单一状态下的对话

`叙事手段`：
- 屏幕字推进（手机短信 / 文件 / 弹幕）
- OS / VO 推进
- 道具镜头（戒指 / 项链 / 信件 / 手帕）
- 声音设计（SFX / 音乐切入）

`节奏切换`：
- 场内余震镜头 / 表情停顿
- 慢推 / 慢拉镜头（同帧渐变）

### 昂贵（写正文时尽量避免或显式优化）

`场景类型`：
- 室外大场景（光线复杂）
- 多角度同场景（每切一个角度都要重新生成保持一致）
- 历史 / 古代场景（道具多 / 服装繁复）
- 群像场景（3+ 人同框）
- 跨时空切换场（每个时空都要单独建场）

`动作类型`：
- 高速动作（跑步 / 追逐 / 打斗）
- 翻滚 / 跳跃 / 爬高
- 舞蹈 / 武打 / 大幅肢体动作
- 拥抱 / 接吻（角色面部接触一致性极难）
- 复杂物理交互（吃饭 / 喝水 / 抽烟）

`叙事手段`：
- 大场面奇观（爆炸 / 变身 / 魔法）
- 多角色同时出现的群像戏
- 长镜头（角色长时间在镜头内说话，口型对齐压力大）
- 闪回（场景切换 + 时间标记 + 角色变体造型）

`节奏切换`：
- 快速切镜头（每个镜头都是新生成）
- 蒙太奇（多场景叠化）

### 廉价 vs 昂贵的写作转化

把昂贵桥段改写成廉价路径：

| 昂贵原型 | 廉价改写 |
| --- | --- |
| 男女主激烈打斗 5 场 | 改成 1 场对峙 + 道具压制 + 一击翻面 |
| 主角追着反派跑过半个城市 | 改成主角在一个固定场景拦截反派 + 反派逃出 + 主角冷静下一步 |
| 婚礼大场面 100 人参加 | 改成婚礼室内核心对峙 + 屏幕字"宾客哗然" + OS 表达氛围 |
| 主角变身 / 觉醒奇观 | 改成"觉醒前后状态对比"（眼神变化 + 道具发光 + 屏幕字标记） |
| 主角和反派在街头公开打耳光 | 改成室内或半私密场公开打耳光 + 旁观者反应近景 |
| 主角抱着孩子奔跑 | 改成主角抱着孩子站定 + 远处脚步声渐近 + 切到反派出现 |
| 一场连续的长对白（90 秒同人物） | 拆成 3 段，每段穿插对方反应 / 道具特写 / OS 短句缓冲 |

## 二、角色一致性管理

### 一致性预算

每个项目的"一致性预算"决定能保稳的角色数量和造型变体数量：

- 主角：默认 3-5 套造型变体（日常 / 工作 / 重要场合 / 危机 / 终局）
- 核心关系位 / 核心对手：2-3 套
- 主要配角：1-2 套
- 一次性角色：1 套（且不应在剧情中反复出现）
- 群众 / 路人：可用 AI 默认或换不同人脸（不需保持一致）

总预算公式（粗估）：
- A 级常驻角色 × 3 套 + B 级功能角色 × 2 套 + C 级一次性 × 1 套 ≤ 项目能管理的一致性峰值

超预算的项目要么砍角色，要么砍变体。

### 造型锚点

每个保留一致性的角色必须在 `人物小传.md` 写清：
- 外形关键词：发型 / 发色 / 眼睛 / 体型 / 肤色（短词，可在 prompt 里复用）
- 固定识别锚点：每次出现都不变的标志（一个特定首饰 / 一道伤疤 / 一种眼神 / 一种姿态）
- 服装变体清单：每套衣服按 `造型 1：日常款 / 造型 2：婚礼款` 等具体描述
- 不允许"看 AI 怎么生成"——必须先锁住

### 一致性失败的兜底

如果某一集出现角色一致性问题（脸 / 体型 / 服装漂移），优先按以下顺序兜底：
1. 用近景 / 半身 / 背影代替全身（减少需要一致的元素）
2. 用 OS / VO / 屏幕字补足角色辨识（不依赖看脸）
3. 用固定识别锚点（戒指 / 伤疤 / 标志性物件）做识别
4. 重新生成（最贵，最后选）

写场次执行稿时主动给一致性失败留兜底空间——不要把全片唯一辨识落点放在角色脸上。

## 三、镜头切换成本

### 单集场景预算

按目标时长决定的"场景数预算"：
- 45-75 秒：2-3 场（同场尽量延伸）
- 60-90 秒：3-4 场
- 90-120 秒：4-5 场
- 90-150 秒首集：5-6 场（首集允许更多场，但每场要充实）

每多一场都意味着重新生成场景；写大纲时如果发现"必须 6+ 场才能讲清"，先并集或拆集。

### 单场镜头预算

单场内的镜头数（同场不同角度）：
- 默认 3-5 个镜头
- 同一镜头可用 2 次（如 1-1 开场镜头 + 1-1 结尾回应同视角）
- 不超过 8 个镜头单场（再多就该拆场）

### 场景复用率

一个常驻场景在全剧出现的次数：
- A 级常驻（如主角家 / 公司 / 反派老巢）：≥ 10 次
- B 级阶段（如某段时期专用）：3-8 次
- C 级一次性：1-2 次

资产重项目要按这个预算反向校准世界观和分集大纲；总集数 60 集只允许出现 4-6 个 A 级常驻场景。

## 四、AI 配音和口型对齐

### 配音特点

AI 配音（豆包 / 火山引擎 / 11Labs / Sora 配音等）：
- 长句和复杂情绪台词不稳定
- 短句、克制、明确情绪稳定
- 口型对齐有限（尤其多音节英文）

写作要求：
- 单句默认 ≤ 15 字（中文）/ ≤ 12 word（英文）；这一硬规则在 AI 漫剧中是配音稳定性的硬约束，不只是节奏要求
- 复杂情绪用 `△ 动作` + 短台词组合，不要用一句长台词承担全部情绪
- 大笑 / 哭嚎 / 嘶吼这类极端声音需在 `△` 标 SFX 而不是写成台词

### 口型对齐策略

减少口型对齐压力：
- 多用反应镜头（角色在听不在说）
- 多用 OS / VO 替代长对白
- 长台词拆成段，中间穿插 `△` 动作
- 海外项目优先选不强调口型的镜头景别（中景 / 远景）

## 五、特效预算

特效成本由低到高：
- 静态光效（背光 / 高光 / 阴影）— 廉价
- 简单粒子（雪 / 雨 / 火星）— 廉价
- 物体发光（道具 / 文字闪烁）— 中等
- 烟雾 / 尘土 / 水花 — 中等
- 爆炸 / 火灾 — 昂贵
- 角色变身 / 飞行 / 隐身 — 极贵
- 大场面战斗 / 群体施法 — 极贵

写作要求：
- 高概念项目把奇观分散到多场，不集中在一场
- 单集特效场不超过 1 处；首集允许 1-2 处
- 全剧奇观场总数 ≤ 总集数 × 0.3（即 60 集项目奇观场 ≤ 18 处）

## 六、与策划阶段的对接

写 `项目设定.md / 世界观设定.md / 故事大纲.md / 分集大纲.md` 时，必须按本文的成本预算反向校准：

- `项目设定.md` 资产与制作 section 要写清"AI 漫剧子风格"（写实 / 真人 / 3D / 国漫 / 日漫 / 赛博 / 暗黑 / 萌系）和对应预算
- `世界观设定.md` 必须能在 4-6 个 A 级常驻场景里讲完故事（除非项目按超长线 100+ 集设计）
- `故事大纲.md` 的"中段升级"和"后段失控"要避免"靠不断换新场景"撑新鲜感
- `分集大纲.md` 的"本集主要场景"字段要在场景预算内

## 七、与剧本阶段的对接

写 `第XXX集.md` 时：
- 单集场景数按本文 §3.1 预算；超出回改大纲
- 单场镜头数按 §3.2 预算
- 配音稳定性按 §4
- 特效预算按 §5
- 廉价 vs 昂贵改写按 §1.3 表格主动转化

`场次执行稿模板`（05 / 06）的动作行 `△` 在写复杂动作时，优先按廉价路径写；如果必须昂贵，在场次前置字段标注 `⚠ 高成本场`，让制作交付时主动评估。

## 八、与制作交付的对接

进入 `production-enhanced` 时，制作交付包必须按本文为下游分镜 / 视频 / 配音引擎补充：

- `场景参考卡` 的视觉锚点必须包含 AI 提示词可复用的关键词
- `道具设定卡` 的关键道具要写清是否需要特写、特写时如何保持一致性
- `角色资产摘要` 的造型变体要标注每套造型对应的剧情触发条件
- `制作交付参数表` 要明确写"高成本场清单"，让分镜引擎主动分配预算

详见 `14-制作交付整理.md` 和 `21-产物矩阵与创建时机.md` 的 production-enhanced 段。

## 九、与下游引擎的接口契约

本 skill 只到 `场次执行稿`，不生成分镜 / 视频 / 配音脚本。但 `场次执行稿` 必须为下游引擎留下可消费的接口：

- 场次标题 `### 1-1 日内 地点`：分镜引擎以此切分
- `人物：` 行：角色一致性引擎以此挂载造型 ID（资产 ID 命名见 `14-制作交付整理.md` 资产命名统一规范）
- `△ 动作行`：视频引擎以此生成镜头 prompt（建议每个 `△` 一个镜头）
- `角色名：` 台词行：配音引擎以此分配人声
- `OS：/ VO：`：配音引擎独立人声轨
- `屏幕字（类别）：`：合成引擎以此生成屏幕文字图层；类别建议保留 `叠加 / 手机 / 文件 / 系统 / 姓名牌` 等，用于区分叠加提示字、画内信息字和环境标签字
- `SFX：`：音效引擎以此插入音效

写正文时不要破坏这些标记的可解析性：
- 标记必须在行首
- 一个 `△` 行内描述一个镜头单元，不要写成长段落
- 屏幕字独立成行，不要嵌入对白行内；类别写在标记括号里，不写进正文说明
- 复杂动作分多个 `△` 行写，每行一个具体动作
