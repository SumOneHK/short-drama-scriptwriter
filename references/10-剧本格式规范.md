# 剧本格式规范

最终剧本不是文学稿，而是给 AI 漫剧直接使用的 `场次执行稿`。

## 一、开发顺序

推荐顺序：

1. `T04 项目设定`
2. `T05-T07 世界观 / 人物 / 场景道具基底`
3. `T08 故事梗概 / 故事大纲`
4. `T09 故事节拍表 / 阶段大纲`（`总集数 >= 30` 默认必出；`20-29` 集强推荐）
5. `T10 分集大纲`
6. `T11 大纲质检`
7. `T12-T14 分集剧本批次创作与质检`

规则：
- 不从一句话想法直接跳正文
- 不从一句话集纲直接跳正文
- 先让 `T04-T09` 形成可判断策划基底，再进入 `T10 分集大纲`
- `T05-T07` 同轮设计但分别落盘；`T08-T09` 稳定后再回补必要深化项
- `分集大纲` 通过后，才写剧本

说明：
- `分集大纲.md` 不只是“每集一句话”
- 文件顶部先放 `分集总览表`，用于快速评审全季推进
- 它至少要达到单集 treatment 的最低深度，能看出本集定位、开场承接、主要动作、主要场景、情绪走向、结尾钩子和下一集接口
- `故事节拍表.md` 用来钉全剧关键翻面和阶段高潮；短项目可合并相邻节拍，长项目则应展开得更清楚

## 二、正文文件粒度

- 一集一个 `Markdown`
- 文件名统一：`第001集.md`
- 最终正文统一进入 `02-剧本/`
- 不按批次目录作为最终剧本目录

## 三、默认正文标准

每集固定四块：

1. `第X集`
2. `本集创作简卡`
3. `场次正文`
4. `本集回填`

### 本集创作简卡

至少包含：
- 开场承接
- 本集戏剧单位
- 本集情绪落点
- 本集最小状态变化
- 本集观看回报
- 本集新关键信息点
- 本集信息投放顺序
- 本集最值钱画面
- 本集可见后果
- 本集中段错误放松或小释放
- 结尾钩子

### 场次正文

每场固定顺序：

1. 场次标题
2. 场次目标补充
3. 人物行
4. `△` 或 `A：` 动作行
5. 台词 / `OS：` / `VO：`

正式分集稿的场次格式是硬规则。故事再成立，如果缺 `人物：`、缺动作行标记、把声音/屏幕字混进普通动作说明，仍然不能视为标准 `场次执行稿`。

兼容写法：
- 场次标题默认写成 `### 1-1 日 外 主任医师办公室` 或 `### 1-1 日外 主任医师办公室`
- 场次标题至少要同时包含：`场景序号 / 时间 / 内外景 / 地点`
- 场次标题下默认补这几项简短信息：
  - `承接点：`
  - `场景目标：`
  - `场景状态差量：`
  - `视觉记忆点：`
  - `动作升级点：`
  - `可见后果：`
- 简单项目可以把这几项压成 `2-3` 行短句，但不要完全省略成只剩标题
- `人物：` 一行写本场出场人物；首次出场可直接补身份说明，如 `裴淮章（男主，心外科主任医师）`
- 如果是聚合角色或群像角色，首次出场也要补功能身份说明，如 `学生志愿者若干（现场执行志愿者）`、`媒体与家属成员（群像）`
- 动作行默认继续用 `△`；如果要兼容外部团队稿式，也允许使用 `A：`
- `角色名：` 表示现场可见角色对白
- `OS：` 表示人物心理活动或未出镜短句，只在无法靠动作/反应外化时使用
- `VO：` 表示旁白、广播、录音、视频转述、系统播报等画外音
- `屏幕字：` 表示手机、闸机、弹窗、名单、文件、弹幕、姓名牌等画面文字。推荐用括号标清类别，如 `屏幕字（叠加）：`、`屏幕字（手机）：`、`屏幕字（文件）：`、`屏幕字（系统）：`、`屏幕字（姓名牌）：`；叠加提示字和剧情世界内的画内文字分开计量。
- `SFX：` 表示会影响节奏或剧情理解的关键声音；普通环境声不必每次标
- 如确有回忆段，可在对应场次前后增加 `【闪回】` / `【闪回结束】`
- 如甲方或制作方要求，可在动作行里偶发加入 `【**近景**】` 这类景别提示，但它只是兼容标记，不把正文改成分镜稿
- 场次开头除了 `时间 / 地点 / 内外景`，优先补 `1-2` 个和剧情、人物状态或权力关系强相关的环境或光影细节
- 每场尽量至少留 `1` 个视觉记忆点，不让场次只剩功能对白
- 人物情绪、压迫感、危险感优先写成可见细节、动作或他人反应，不直接写抽象判断句
- 真正重要的节点允许少量兼容性镜头提示，如 `【近景】 / 【剪影】 / 【叠化】 / 【定格】 / 【升格】 / 【降格】 / 【切】`，但只在开场钩子、爽点瞬间、情绪爆发、悬念揭晓等关键节点使用
- 如果正文里出现独立旁白段落，旁白中的关键人名可按对方要求加粗
- 对白优先在动作和反应之后补锋利，不要用对白替动作
- 重要身份、目的、威胁和权力关系，优先用动作、道具、他人反应和现场气氛传达
- `OS / VO` 是按需工具，不是默认推进手段：
  - `OS` 只在角色心理确实无法靠动作、反应、他人观察外化时使用
  - `VO` 只在开场定调、闪回衔接、录音/广播/信件/视频转述等场景使用
  - 不要用大段 `OS / VO` 代替本可拍出来的动作和关系变化
  - 同一场里如果 `OS / VO` 已经说清的信息，动作和对白就不要再重复解释

### 海外项目

- 角色、场景、关键道具采用稳定的 `中文 / English`
- 英文对白后接中文注释
- 最终制作语种示例默认用英文标点

## 四、不要这样写

- 不要把正文写成小说段落
- 不要用大量说明代替冲突
- 不要用自报家门式台词替代人物亮相和权力建立
- 不要让对白把观众本来能自己看懂的信息全说破
- 不要让 `OS / VO` 变成“解释剧情的第二条对白轨道”
- 不要把 `广播：`、`机械音：`、`系统提示：`、`黑伞灯浮字：` 等写成未归类的普通台词；应统一改为 `VO：`、`屏幕字（系统/画内/叠加）：` 或 `SFX：`
- 不要用一串抽象形容词假装有画面感
- 不要把整篇正文写成镜头提示清单
- 不要跳过大纲直接写剧本
- 不要整集只有“拿到信息 / 解释信息 / 预告下集”，却没有可见后果和关系变化
