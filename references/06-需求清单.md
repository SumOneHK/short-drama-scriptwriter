# 需求清单

在开始写任何题材或剧本前，先收齐最小可用需求。

## 必须确认

| 字段 | 为什么重要 | 缺失时是否优先追问 | 默认值 |
| --- | --- | --- | --- |
| 目标市场 | 决定题材、本地化和对白语言 | 是 | 不默认 |
| AI 漫剧子风格 | 决定角色造型、场景质感和镜头写法 | 是 | 不默认 |
| 目标受众 | 决定主爽点、关系结构和情绪分配 | 建议 | 不默认 |
| 剧本格式 | 确认是否按标准 `场次执行稿` 交付 | 建议 | 固定为场次执行稿 |
| 核心题材 | 决定故事引擎 | 建议 | 取最接近用户描述的方向 |
| 选题策略 | 决定优先求稳、微创新、制作友好还是高概念创新 | 建议 | 不默认；用户未指定时先给不同策略候选，不替用户锁题 |
| 受众画像 | 决定关系结构和情绪密度 | 建议 | 若是女频短剧，默认 18-34；若是男频项目，按用户描述路由 |
| 单集时长 | 决定镜头密度和台词长度 | 建议 | 不默认，先按平台建议区间确认 |
| 总集数 | 决定分卷和反转密度 | 建议 | 不默认，先按项目规模与商业模式确认 |
| 制作约束 | 决定是否真能做出来 | 建议 | 中等预算、有限常驻场景 |
| 变现模式 | 决定卡点设计走向（卡点是 IAA / IAP / 平台付费定制完全不同） | 是 | 不默认；明确标注 `IAA / IAP / 付费APP / 平台付费定制` 之一；当前 AI 漫剧主流为 `IAA`，未明确时按 IAA 处理 |
| 卡点设计策略 | 决定 `分集大纲` 必须标注的卡点字段 | 是 | 按变现模式不同：IAA 必填 `完播抓手 + 追更钩 + 广告卡位`；IAP 必填 `S/A/B 档付费触发位 + 不可逆设计`；平台付费定制必填 `会员独享 + 中后段密度`。详见 `references/24-卡点设计.md` |
| 下游 AI 工具链对接 | 决定剧本交付边界与命名口径是否需要兼容下游分镜/视频/配音引擎 | 建议 | 不默认；如对接 `分镜生成器 / 视频生成器 / 配音引擎`，需在交付层补口径 |
| 母体来源市场 | 决定是否需要跨市场 trope 重写 | 是 | 三个枚举值之一必须明确写出：`无母体` / `国内母体` / `海外母体`；不允许留空。原创项目写 `无母体`，不写"默认无"。与目标市场不一致时必须按 `02-策划细则.md` 的 trope 重写矩阵处理 |

## 建议补充

- 参考作品或禁止参考
- 情绪方向：爽、甜、虐、悬疑、暗黑、热血、轻喜、治愈
- 目标受众：男频、女频、泛受众
- 题材类型：玄幻仙侠、都市现代、悬疑灵异、历史架空、末世、科幻、游戏、国运、武侠、兵王、奇幻、军事、其他
- 选题策略：主流稳态 / 类型微创新 / 制作友好 / 高概念创新 / 混合策略
- 核心设定：从 `流派 / 身份 / 金手指 / 情节` 四类里各取最关键的点，不要求堆满
- 风格元素：从 `风格 / 元素` 两类里选最能定义传播气质的点
- 变现模式：`IAA`（广告变现，AI 漫剧主流）/ `IAP`（小程序付费短剧 / 海外 episode-app 付费部分）/ `平台付费定制`（长视频平台买断）
- AI 漫剧子风格：写实、AI 真人、3D、国漫、日漫、赛博、暗黑、萌系等
- 是否允许重特效、群像大战、跨国多场景
- 海外项目具体优先市场：美国 / 泛欧美 / 指定国家
- 是否有可借用的母体来源：神话 / 民俗 / 经典 / 行业 / 校园 / 豪门 / 规则世界 / 其他
- 是否希望做高概念创新：（仅作为可选项）身份反转 / 规则反转 / 阵营反转 / 时间线反转 / 因果反转 / 其他
- 卡点设计策略：
  - IAA 项目：单集完播抓手（5s / 25s / 50s）/ 追更钩类型 / 中插广告卡位 / 是否设激励视频解锁
  - IAP 项目：免费章范围 / 首付费卡集号 / 付费阶梯节奏 / 单集卡点强度（S/A/B）/ 不可逆翻转设计
  - 详见 `references/24-卡点设计.md`
- 下游 AI 工具链对接：是否对接分镜生成器、视频生成器、配音引擎；是否需要在交付包补统一资产命名口径

市场路由提醒：
- `抖音国内` 项目优先参考 `15-国内选题与设定词库.md`
- `TikTok 欧美` 项目优先参考 `16-美国英语市场选题与设定词库.md`
- 海外项目复用同一套维度框架，优先按美国英语市场词库完成组合

## 推荐追问顺序

按这个顺序追问，避免一次把用户问烦：
1. 目标市场
2. AI 漫剧子风格
3. 目标受众
4. 选题策略、题材和核心设定
5. 风格元素和情绪方向
6. 时长和集数
7. 制作约束
8. 是否存在偏离标准 `场次执行稿` 的额外交付要求
9. 发行模式

## 推荐交付包

从零到一项目默认按以下顺序产出（与 SKILL.md 主流程、`21-产物矩阵与创建时机.md` 编号对齐）：

立项层（T01-T03）：
1. 对话内市场判断（不落盘）
2. 对话内候选题材池（题材未定时，不落盘）
3. 对话内故事理解（仅改编项目，不落盘）
4. 用户明确确认方向、目标市场、受众、规格、制作边界等必要立项信息
5. 初始化项目目录，生成项目状态、项目简报和锁题摘要
6. 立项质检 → 通过后进入策划

锁题提醒：`市场快照 / 待选题材池 / 故事理解` 在确认前都只是对话内材料，不落项目文件。只有用户明确确认方向和必要立项信息后，才初始化项目并生成 `项目简报 / 锁题摘要`；用户仍在比较或要求继续调研时，不得替用户做最终选择，也不得写项目文件。

策划层（T04-T11）：
7. 项目设定
8. 世界观设定
9. 人物小传
10. 场景/道具锚点（轻量项目内嵌到项目设定，资产重项目额外出场景参考卡 / 道具设定卡）
11. 故事梗概
12. 故事大纲
13. 故事节拍表
14. 阶段大纲（`总集数 >= 30` 默认必出；`20-29` 集强推荐）
15. 策划基底质检 + 策划结构质检 → 通过后写分集大纲
16. 分集大纲
17. 大纲质检 → 通过后进入剧本

剧本层（T12-T15）：
18. 分集追踪（首批正文前初始化）
19. 分集剧本正文（按批次生成，每批先写代表样稿）
20. 剧本质检（每批） + 交付前全剧体验通读复盘 + 全剧剧本复核（交付前）

交付层（T16-T18）：
21. 完整剧本总稿 / 剧本圣经 / 交付清单（标准交付包）
22. 制作理解稿 / 制作交付参数表 / 场景卡 / 道具卡 / 海外制作版分场表（production-enhanced 时追加）
23. 交付质检 → 通过后才算正式交付
