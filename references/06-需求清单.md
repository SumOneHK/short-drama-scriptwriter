# 需求清单

在开始写任何题材或剧本前，先收齐最小可用需求。

## 必须确认

| 字段 | 为什么重要 | 缺失时是否优先追问 | 默认值 |
| --- | --- | --- | --- |
| 发行平台 | 决定竖屏观看语法、平台合规和规格建议 | 是 | 国内写 `抖音`；出海写 `TikTok` |
| 目标市场 | 决定题材本地化、对白语言和文化锚点 | 是 | 不默认；出海常见为 `美国英语首版 / 泛欧美 / 东南亚 / 拉美 / 指定国家或地区` |
| AI 制作风格 | 决定角色造型、场景质感、视觉符号和镜头写法 | 是 | 不默认 |
| 目标受众 | 决定主爽点、关系结构和情绪分配 | 建议 | 不默认 |
| 剧本格式 | 确认是否按标准 `场次执行稿` 交付 | 建议 | 固定为场次执行稿 |
| 核心题材 | 决定故事引擎 | 建议 | 取最接近用户描述的方向 |
| 选题策略 | 决定优先求稳、微创新、AI-native 视觉型还是高概念创新 | 建议 | 不默认；用户未指定时先给不同策略候选，不替用户锁题 |
| 受众画像 | 决定关系结构和情绪密度 | 建议 | 若是女频短剧，默认 18-34；若是男频项目，按用户描述路由 |
| 单集时长 | 决定镜头密度和台词长度 | 建议 | 不默认，先按平台建议区间确认 |
| 总集数 | 决定分卷、反转密度和阶段大纲触发 | 建议 | 不默认，先按项目规模和故事引擎确认 |
| AI 制作口径 | 决定交付层是否需要资产命名、角色识别锚点和场景/道具卡 | 建议 | 不默认；不作为选题降级依据 |
| 商业化模式 | 仅在用户明确要商业化、投流或付费设计时决定卡点字段 | 否 | `商业化后置`；不默认 IAA |
| 商业化卡点策略 | 仅商业化模式明确时决定 `分集大纲` 追加字段 | 否 | 不默认；详见 `references/24-卡点设计.md` |
| 下游 AI 工具链对接 | 决定剧本交付边界与命名口径是否需要兼容下游分镜/视频/配音引擎 | 建议 | 不默认；如对接 `分镜生成器 / 视频生成器 / 配音引擎`，需在交付层补口径 |
| 母体来源市场 | 决定是否需要跨市场 trope 重写 | 是 | 三个枚举值之一必须明确写出：`无母体` / `国内母体` / `海外母体`；不允许留空。原创项目写 `无母体`，不写"默认无"。与目标市场不一致时必须按 `02-策划细则.md` 的 trope 重写矩阵处理 |

## 建议补充

- 参考作品或禁止参考
- 情绪方向：爽、甜、虐、悬疑、暗黑、热血、轻喜、治愈
- 目标受众：男频、女频、泛受众
- 题材类型：玄幻仙侠、都市现代、悬疑灵异、历史架空、末世、科幻、游戏、国运、武侠、兵王、奇幻、军事、其他
- 选题策略：主流稳态 / 类型微创新 / AI-native 视觉型 / 高概念创新 / 混合策略
- 核心设定：从 `流派 / 身份 / 金手指 / 情节` 四类里各取最关键的点，不要求堆满
- 风格元素：从 `风格 / 元素` 两类里选最能定义传播气质的点
- 商业化模式：`商业化后置` / `IAA` / `IAP` / `平台付费定制`；未明确时不倒推剧本
- AI 制作风格：写实、AI 真人、3D、国漫、日漫、欧美漫画、赛博、暗黑哥特、romantasy、萌系等
- 视觉偏好：现实向 / 奇幻 / 异世界 / 学院 / 哥特 / 末日 / 赛博 / 童话黑化 / 其他
- 海外项目具体优先市场：美国 / 泛欧美 / 指定国家
- 是否有可借用的母体来源：神话 / 民俗 / 经典 / 行业 / 校园 / 豪门 / 规则世界 / 其他
- 是否希望做高概念创新：（仅作为可选项）身份反转 / 规则反转 / 阵营反转 / 时间线反转 / 因果反转 / 其他
- 商业化卡点策略（仅用户明确要求商业化时追问）：
  - IAA 项目：单集完播抓手 / 追更钩类型 / 中插广告卡位 / 是否设激励视频解锁
  - IAP 项目：免费章范围 / 首付费卡集号 / 付费阶梯节奏 / 单集卡点强度（S/A/B）/ 不可逆翻转设计
  - 平台付费定制：会员独享段 / 长头部建制 / 中后段密度策略
  - 详见 `references/24-卡点设计.md`
- 下游 AI 工具链对接：是否对接分镜生成器、视频生成器、配音引擎；是否需要在交付包补统一资产命名口径

市场路由提醒：
- `发行平台=抖音` 且 `目标市场=国内` 项目优先参考 `15-国内选题与设定词库.md`
- `发行平台=TikTok` 且 `目标市场=美国英语首版 / 泛欧美` 项目优先参考 `16-美国英语市场选题与设定词库.md`
- `发行平台=TikTok` 但目标市场不是英语市场时，先用同一套维度框架组织题材，再补对应市场本地化调研

## 推荐追问顺序

按这个顺序追问，避免一次把用户问烦：
1. 发行平台（抖音 / TikTok）
2. 目标市场和对白语言
3. AI 制作风格
4. 目标受众
5. 选题策略、题材和核心设定
6. 风格元素和情绪方向
7. 时长和集数
8. 是否存在偏离标准 `场次执行稿` 的额外交付要求
9. 商业化模式（可明确后置；不要反复追问）

## 进入 T02 后的立项产物分工

本文件只负责需求收集，不重复维护完整创作流程。T01-T18 的产物顺序、创建时机和触发条件以 [21-产物矩阵与创建时机.md](21-产物矩阵与创建时机.md) 为准。

立项阶段只记住三条：
- `T01`：市场判断、候选题材池、改编理解都先在对话中完成；用户仍在比较时不落盘
- `T02`：用户明确确认方向和必要立项信息后，初始化项目并生成 `项目状态.json / 项目简报.md / 锁题摘要.md`
- `T03`：独立立项质检通过后才进入策划

`项目简报.md` 与 `锁题摘要.md` 不互相替代：
- `项目简报.md`：记录用户原始需求、已确认约束、默认假设、平台/市场/规格/AI 风格等基础信息，以及轻量市场判断
- `锁题摘要.md`：记录最终创作决策，包括一句话定义、最终题材、核心卖点、连载引擎、压迫来源、阶段回报、不可偏离边界
- 两份文件共享硬字段时只保持短字段一致，不重复写同一段市场分析、题材论证或卖点推导

默认目录建议：

```text
projects/<project-slug>/
```

详细目录结构和最终交付规则见：
- [00-项目目录与最终交付.md](00-项目目录与最终交付.md)
