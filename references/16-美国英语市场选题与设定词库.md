# 美国英语市场选题与设定词库

这份词库用于 `发行平台=TikTok` 且 `目标市场=美国英语首版 / 泛欧美英语市场` 的 `/立项` 和 `/策划`。

用途：
- 帮助美国英语市场项目快速组织选题
- 帮助把欧美常见高点击 trope 和 AI 竖屏剧 / 漫剧高适配 genre 拆成可组合的创作维度
- 帮助在策划阶段把题材组合落成真正可连载的故事引擎

使用原则：
- 复用与国内相同的四层框架：`目标受众 / 题材类型组合 / 核心设定组合 / 风格元素组合`
- 海外项目优先使用美国英语市场天然成立的 genre 和 trope 组合
- 这是美国英语市场词库，不是全部欧美国家或全部出海市场的统一词库
- 默认先锁 `1 个目标受众 + 1-2 个题材类型 + 2-4 个核心设定 + 2-4 个风格元素`
- 选定后要回答：这些组合在美国英语用户眼里为什么天然成立、为什么能持续追看、为什么不会像中国爽文的直译版

- 美国英语市场当前主力仍偏女性向 romance / revenge / fantasy

- 男性向可以做，但不建议作为默认主路由

### 价值观 / 弧光池（美国主流）

正面（默认可用）：
- `American Dream`：白手起家、跨阶层
- `Underdog story`：弱者反杀
- `Self-made`：靠自己不靠家
- `Grit / resilience`：哪怕被打趴也能站起来
- `Comeback / redemption arc`：跌倒再起、悔过自新
- `Reinvention`：换城市、换身份、重塑人生
- `Found family`：选择性家人，不靠血缘
- `Speaking truth to power`：对权威说不
- `Owning your truth`：公开自己的身份 / 性向 / 选择

反面（默认避免，谨慎使用 — 这些在欧美 = 红旗，不是爽点）：
- `Toxic / toxic relationship`：精神控制、贬损、操控；只能写给反派
- `Gaslighting`：让对方怀疑自己的认知
- `Love bombing`：开局过度示好、再撤回作为操控
- `Red flags`：早期警示信号（控制欲、嫉妒、隔离）
- `Abuse / DV / IPV`：家暴；不能美化也不能用"爱"包装
- `Stalking`：跟踪 / 监视；如果是反派可以，主角追求要避开
- `Coercive control`：强制控制；隔离、经济控制、信息控制
- 中式霸总桥段（`抓手腕 / 强吻 / 公开羞辱 / 命令式占有`）→ 默认全部读作 toxic / harassment

特别提醒：
- "霸总" 在美国语境里要么变成 `dark romance`（明确风格化、禁忌、双方知情），要么变成 `protective / loyal CEO`（保护欲不是控制欲）
- "抢婚 / 强娶" 不能直接套；要改成 `last-minute objection at the wedding`（电影合法 trope）

## 五、组合规则

优先考虑：
- `题材类型组合` 决定世界和主冲突
- `核心设定组合` 决定 trope 引擎和续看欲望
- `风格元素组合` 决定传播气质和观感

避免问题：
- 不要把 tropes 堆成标签墙，仍然要回到一个清晰的主关系和主冲突
- 不要把“身份秘密”当作唯一推进手段，中段仍然要有权力变化、关系翻面和后果升级
- 不要让超自然题材只剩设定解释，必须尽快落到欲望、代价和禁忌
- 不要让 AI-native 奇幻只剩视觉奇观；每个奇观必须改变人物关系、目标、危险或信息差

市场口径提醒：
- 美国英语市场优先使用 `billionaire / heiress / alpha / forbidden love / campus / thriller / family secret` 这类当地用户天然熟悉的 trope 和 genre
- AI 竖屏剧可以优先使用 `romantasy / villainess / Isekai-lite / gothic curse / academy / witch / vampire / survival rules`，但必须在第一场看见规则后果
- 如果题材来自国内母体，先在 `/策划` 中改写成美国语境里天然成立的关系、权力和欲望结构

## 六、策划时必须回答

从词库中选完组合后，必须进一步回答：
- 这个组合为什么适合美国英语市场用户
- 这个组合的核心情绪回报是什么
- 这个 trope 在美国语境里是否天然成立
- 主关系为什么会持续拉扯
- 中段如何变奏，不变成反复误会或反复揭身份
- 这个组合最值钱的视觉符号是什么
- 这个组合对角色、场景、道具提出了什么资产要求
