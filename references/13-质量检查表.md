# 质量检查表

内容检查只保留两张主检查表：
- `策划 / 大纲质检`
- `剧本质检`

状态门禁仍按 `项目状态.json` 的六个质检节点执行。这里说的“两张主检查表”只指内容检查维度，不替代 `立项质检 / 策划基底质检 / 策划结构质检 / 大纲质检 / 剧本质检 / 交付质检`。

不要把质检再做成一轮大文件生产。`/复盘` 每个里程碑默认只产出一份 `复盘报告-*.md`，正式分集稿只保留轻量 `本集创作简卡`，完整检查问题放在工作稿和复盘里。

快速定位：
- 判断大纲能不能进正文：读"策划 / 大纲质检"
- 判断正文批次能不能通过：读"剧本质检"
- 全剧完成后查体验断点：读"交付前全剧体验通读复盘"
- 只查禁区或平台风险：读"平台安全检查"

## 质量北极星

质量检查最终只问一件事：这是不是一批有吸引力、能交付、能让观众继续看的剧本。

固定判断口径：
- 结构完整但不好看，不通过
- 信息清楚但没有冲突、情绪和后果，不通过
- 设定新但没有人物欲望和关系张力，不通过
- 每集都能讲清功能但删掉后不影响观看体验，不通过
- 模板填得满但正文没有动作、反应、翻面和记忆点，不通过

## 一、策划 / 大纲质检

在 `分集大纲.md` 完成后执行，通过后才进入 `/剧本`。

重点检查：
- 市场、题材、规格是否已经锁定
- 目标受众、题材类型、核心设定、风格元素是否已经形成清晰组合，而不是混乱堆砌
- `项目设定` 是否先锁住项目基底，再去补深化项，而不是一开始把方法词全部堆满
- `世界观设定.md` 是否存在，而且不是只在强设定题材里才写；现实题材也应有轻量版秩序说明
- 故事梗概是否真的讲清“这是什么戏、为什么抓人、为什么值得继续做”，而不是包装话或流水账
- 读完故事梗概后，是否已经大致知道整部剧怎么被卷入、怎么升级、怎么失控、怎么兑现，而不是只记住一个题眼
- 高概念或强世界观项目的故事梗概，是否把真正驱动主线的规则、代价、阵营和常驻关系写进去了，而不是只剩一句设定包装
- 核心角色版 `人物小传` 是否已经足够支撑故事大纲和故事节拍表，而不是先有结构、后补角色
- 如果主角声线、核心关系张力、首次亮相高光或关键对峙场还拿不准，是否已经先做 `试写样场`，而不是硬推完整 `分集大纲`
- `试写样场 / 批次写作单 / 单集达标检查 / 分集追踪` 是否各司其职，没有互相替代、重复维护
- 故事大纲是否把整部剧的阶段推进、阶段升级和结局收束写清，而不是只把关键节点重新说一遍
- 故事节拍表是否把全剧关键翻面、阶段小高潮、最低谷和终局推进钉住
- 故事梗概和故事大纲是否各司其职，而不是一个写成另一个的缩写版
- 故事大纲和故事节拍表是否同口径，而不是各讲各的主线
- `世界观设定.md`、`故事大纲.md` 和 `故事节拍表.md` 是否同口径，世界秩序、关键翻面和阶段后果是不是讲的是同一套故事
- `世界观设定.md` 是否主要在写秩序、规则、禁忌、代价和世界后果，而不是把完整故事又讲了一遍
- `故事梗概.md` 是否只保留真正推动主线的世界规则和代价，而不是把世界观文件整段搬过来
- `故事大纲.md` 是否主要在写阶段推进、升级、失控和收束，而不是重复解释世界如何运转
- 故事节拍表和分集大纲是否各司其职，而不是一个写成另一个的展开版
- 故事节拍表是否已经给关键节拍补出 `对应集数或阶段` 的落点，而不是只写抽象节点名
- 故事大纲、故事节拍表、项目设定里的推进语汇和回报语汇是否真的匹配当前题材，而不是沿用别的题材母体旧话术
- 人物是否齐全，功能是否明确
- `人物小传` 是否把主要角色写清为 `身份 / 特征 / 境遇 / 行动 / 终局状态`
- `人物小传` 是否已经覆盖 `主角 / 核心关系位 / 核心对手 / 主要配角`，并按重要程度递减补出关键配角或功能角色
- 主要角色的 `人物小传` 是否已经补出 `外形关键词 / 固定识别锚点 / 常用服装层 / 首次出场必须交代的信息`
- `人物小传` 里写的特征和行为是否真的会落到主线，而不是堆无关设定
- 主角是否有明确 `可投射身份 / 观众站队点 / 开局受压点 / 翻盘入口 / 强反击方式`
- 主角是否容易让观众理解“为什么难、为什么忍、为什么反击”，而不是只剩设定说明
- 是否把“善良”写成“软弱”、把“忍耐”写成“窝囊”，导致观众不想跟主角走
- `项目设定.md` 是否写清题材叙事约定、关键场景、关键道具和 AI 制作口径
- `世界观设定.md` 是否写清秩序、规则、禁忌、代价和世界后果
- `世界观设定.md` 是否写清 `世界观一句话 / 时空与覆盖范围 / 公众秩序 / 私密秩序 / 明规则 / 潜规则 / 阵营与权力结构 / 代价与禁忌 / 世界如何压迫主角`

市场本地化深度检查（每条都必须能给出可证否结论；只能写定性结论的，复检无效）：

判定方法：每条检查项，复检时必须从正式产物里**举出 3 个具体证据**作为 PASS 依据，举不出 3 个就判 `需修改`。证据必须是具体的人物动作、机构名称、道具、对白原句、场次序号或集号，不能是"整体感觉到位"。

举例（以"压迫机制使用目标市场原生机构"这一条为例）：
- 不可证否的复检结论：`整体压迫机制本地化到位`
- 可证否的复检结论：`已识别 3 处原生压迫机构 —— EP003 第 2 场 NDA 出现，EP005 第 1 场 restraining order 出现，EP012 第 3 场 IRS audit 出现；通过` 或 `仅识别 2 处，少于 3 处门槛；EP005 仍在用"保安拦门"的国内式压迫，需修改`

逐条检查：
- 题材 / 设定 / 矛盾 / 爽点是否长在目标市场真实社会纹理里，不是"英文壳中文芯"或"中文壳欧美芯"的浮空设定
  - 证据要求：从正式产物里举出 3 个具体设定（不是抽象题材描述），每个设定必须能在 `15` 或 `16` 词库里找到对应词条
- 项目压迫机制是否使用目标市场原生的机构 / 道具 / 法律 / 财务系统，而不是所有题材都默认写成"保安 / 登记册 / 门禁"
  - 证据要求：从分集大纲或正文里举出 3 个具体机构 / 道具 / 程序的使用集号或场次；不允许只在 `项目设定` 里写过却从未在正文出现
- 国内项目：是否落到了 `直播 / 校园 / 末日 / 体制 / 医疗 / 法律 / 商战 / 互联网大厂 / 豪门家族` 等题材的原生机构（详见 `15-国内选题与设定词库.md` 的 `题材-机构落地对应`）
- 海外项目：是否落到了 `restraining order / NDA / 401k / IRS audit / Common App / shadow ban / cancellation campaign` 等美国原生机制
  - 证据要求：列出 ≥ 3 个机制名 + 对应集号 + 该机制在该集如何具体改变了局势（不是只出现一个名词）
- 价值观爽点是否符合目标市场主流，避免红线：
  - 国内主流：寒门贵子 / 扮猪吃老虎 / 公开清算 / 家国情怀；红线：拜金 / 家暴美化 / PUA 美化 / 复仇至上
  - 美国主流：American Dream / Underdog / Reinvention / Found family / Redemption arc；红线：toxic / gaslighting / love bombing / DV 美化
  - 证据要求：举出 3 个爽点兑现集号 + 兑现方式（具体动作）；同时必须明确写"已自查无红线 trope"或列出红线规避方案
- 海外项目如果来自国内母体，是否完成了 `国内母体 → 美国市场 trope 重写矩阵`（详见 `02-策划细则.md` 的对照表），特别是：
  - `霸总抓手腕 / 强吻` 是否改成 `dark romance` 或 `protective CEO`
  - `家暴 + 复合` 是否改成 `离开 + 自我重塑 + 复仇`
  - `切流 / 限号 / 网信办` 是否改成 `algorithm shadow ban / brand sponsor pulled / cancellation campaign`
  - `学区房 / 985 / 高考` 是否改成 `school district / Ivy League / Common App / SAT`
  - `公开打耳光` 是否改成 `viral takedown / receipts dropped / public callout`
  - 证据要求：跨市场母体项目必须列出"母体来源 trope → 重写后 trope → 落点集号"的对照表，至少 3 行
- 国内项目如果来自海外母体（dark romance / werewolf alpha / Ivy League scandal），是否完成了海外 trope 的国内重写
  - 证据要求：同上，至少 3 行重写对照
- 视觉与资产本地化是否成立：
  - 国内：地标 / 招牌 / 车牌 / 菜系 / 街头细节符合所选地域
  - 海外：欧美面孔 / iPhone / 美式车牌 / NYT / Starbucks 等基础锚点到位
  - 证据要求：举出 3 个具体地标 / 道具锚点 + 出现集号
- 地域分层是否锁定（国内：一线 / 新一线 / 二三线 / 县城 / 东北 / 广东 / 川渝 / 江浙沪；海外：NYC / LA / Texas / suburbs / rural）
  - 证据要求：写出锁定地域；正文中举出 3 个该地域专属符号（方言词 / 当地餐饮 / 地标）
- 节日 / 季节锚点是否被利用（海外项目优先把关键集卡在 Thanksgiving / Halloween / NYE / Prom / Christmas）
  - 证据要求：海外项目至少 1 个关键集卡节日，写出集号 + 节日名 + 节日如何改变本集冲突
- 多元身份处理是否到位：海外 first-gen / 移民 / LGBTQ+ / 跨族裔 romance 是否真实文化层而不是 colorblind 浮空
  - 证据要求：项目命中此项时，至少举出 1 集的具体文化冲突描写；不命中此项写"项目无多元身份主线"

任何一条无法举出符合证据要求的具体证据，整体本地化深度检查判 `需修改`，不进入下一节点。

复杂项目补充：
- 长线、海外或复杂项目，默认在同一份 `复盘报告-*.md` 里增加三种专业视角 section：
- 策划视角：查阶段推进、阶段任务、信息投放节奏、中段续航
- 编剧视角：查人物声音、场次动作、回报密度、尾钩具体性
- 导演视角：查场景可执行性、视觉升级、资产复用和是否过度同场景同写法

通过标准：
- `分集大纲.md` 完整
- 没有明显空集 / 功能集
- 没有重大设定缺口
- 可以进入 `/剧本`

不通过动作：
- 直接回改 `故事梗概.md / 故事大纲.md / 人物小传.md / 项目设定.md / 分集大纲.md`
- 不进入 `/剧本`

## 二、剧本质检

在每个正文批次完成后执行。

批次补充：
- 所有项目正文批次都先看 `1-2` 集代表样稿
- 代表样稿如果已经出现模板化、承接不落地、场次骨架重复，不继续铺满本批
- 长线、海外项目或精品要求项目，对代表样稿和复检强度按更严格口径执行

重点检查：
- 每集是否对得上 `分集大纲.md`
- 每集是否真的有观看动力，而不是只完成剧情功能
- 每集是否能用一句普通观众能复述的话讲清，不依赖内部术语
- 是否出现“设定正确但读起来费劲”的情况；如果有，先低术语重写，不继续铺本批
- 读完每集后，是否能说出一个具体吸引点：高光画面、关系翻面、爽点、痛点、危险升级或悬念变化
- 每集是否有足够的戏剧单位，而不是只有“解释局势 / 确认信息 / 预告下集”
- 每场是否至少能看出 `承接点 / 场景目标 / 场景状态差量 / 视觉记忆点 / 动作升级点 / 可见后果` 这组增量信息，而不是只有地点、人物和对白
- `开场承接` 是否真正落进第一场第一句动作或第一句对白，而不是只停留在任务栏
- 从 `EP002` 开始，第一场是否真的带出了上一集尾钩里的 `人物 / 道具 / 地点 / 动作`，而不是只在同一条故事线上继续往前走
- `批次写作单`、`分集追踪.md` 和正文 `1-1` 是否对同一条承接使用了同一核心短语，而不是三处各说一套
- `单集达标检查` 是否真正承担“当前集能不能落稿”的判断，而不是把剧情再写一遍
- `分集追踪.md` 是否真的只累计稳定事实、悬念债务、关系变化和承接口径，而不是写成下一批剧情计划
- 开头是否先给出情绪冲击，而不是先解释设定
- 若本集承担主角首次亮相，这次亮相是否真的立住了人物分量，而不是只靠旁白或别人夸
- 前段是否已经形成局势变化，而不是只重复原危机
- 中后段是否已经形成明确期待，观众能不能说出主角下一步要干什么
- 当前单集是否完成 `情节承接 / 冲突升级 / 价值闭环 / 下集勾连`，而不是只完成剧情功能
- 本集钩子是否属于具体变化类型，并真正改写下一集目标、危险、关系或信息差
- 本集悬念的信息差是否清楚，且挂在人身、关系、输赢、尊严或情感上
- 本集如果使用了藏法，是否有具体可见线索，而不是靠 `似乎 / 仿佛 / 另有秘密 / 眼神复杂` 这种作者提示
- 暗藏信息是否没有吞掉当集显性事件；观众第一遍仍能复述本集发生了什么和主角下一步要做什么
- 显影场是否回扣了前文线索，并让观众产生“原来早给过”的回看感；如果只靠新资料空降解释，需回改
- 单集或当前批次是否只有直线受压或直线打脸，没有 `压 / 松 / 再压` 的变化
- 关键拉扯段是否给过“危机解了”的错误预期，并在放松后重新加压
- 集时长是否撑得住
- 是否先看“戏剧单位是否足够”，而不是只看场次数是否达标
- 本集戏剧单位是否至少能看出 `目标 / 阻力 / 动作 / 翻面 / 后果`
- 对 `60-90` 秒主流单集，如果只有 `2` 场，单场内部是否真的有翻面、动作升级或空间切换；否则通常戏不够
- 如果单集只有 `1` 场，这一场是否真的具备完整小序列的承载力，而不是一场被拉长的说明戏
- 如果一集主要由 `接电话 / 看文件 / 独处沉默 / 走进新地点` 组成，却没有 `2` 个以上表演节拍或冲突转折，是否已经并场、加压或重写
- 情绪密度是否够
- 情节密度是否够
- 信息密度是否清楚
- 术语密度是否可控；开篇前三集是否避免同集塞入过多专有名词
- 抽象规则是否都落成了可见后果，如谁被忘掉、谁变老、谁失声、谁被封口、谁公开失势
- 屏幕字是否已分清 `叠加提示字 / 画内信息字 / 环境标签字`，且优先呈现具体事实和结果，而不是替作者解释规则
- 主要回报是否真的变成了可见后果，如公开失势、关系翻面、物件易手、规则变化、立即反噬，而不是只确认某条线索存在
- `本集题眼回响` 是否成立，而不是开篇设定之后就再也不回头
- `本集关系饥饿点` 是否具体，能让观众明确知道自己在等什么
- `本集公众后果` 是否成立；如果没有公众层，是否至少形成了不可逆后果
- 是否强化了本剧已经锁定的 `核心情绪线`
- 关键信息是否做到 `快 / 准 / 新 / 狠`
- 情绪曲线是否成立
- 主回报是否明确
- 尾钩是否具体
- 单集是否至少形成 `情绪冲击 / 局势变化 / 强期待 / 尾钩` 里的主要节奏链条
- 人名、场景、道具、规则是否连续
- 场次标题是否统一为 `### 1-1 日内 地点` 这类格式，而不是 `### 场 1：内景，地点`
- 每场是否都有 `人物：` 行，且首次出场人物补了身份说明
- 动作行是否用 `△` 或约定的 `A：` 标记，而不是整段小说式动作描述
- 台词是否使用 `角色名：`，没有把广播、系统播报、屏幕文字误写成普通角色对白
- 该用 `OS：/ VO：/ 屏幕字（类别）：/ SFX：` 的地方是否正确标出
- 首次出场角色的 `人物：` 行是否已补身份说明，尤其是海外项目的中英混写角色、聚合角色、群像角色和功能角色
- 专业内容是否已经压缩成可见输赢和后果
- 海外项目英文对白是否自然，且带中文注释
- 海外项目英文对白是否完全 contractions 化（`I'm / don't / won't / gonna / wanna`），未刻意提级时是否仍出现 `do not / I am / cannot`
- 海外项目是否仍有翻译腔（`How dare you / You are courting death / I will never forgive you / What is the meaning of this`）需要替换成自然口语
- 海外项目中文注释是否在补 `潜台词 / 情绪意图`，而不是字面翻译；是否默认 `<= 15` 字
- 海外项目角色 register 分层是否正确（teen / young adult / professional / antagonist / parental）
- 海外项目感叹 / 咒骂强度是否符合本项目分级（PG / dark romance / thriller / 成人）
- 国内项目是否仍有老旧剧本腔（`你这个混蛋 / 你以为你是谁 / 我会让你付出代价`）或书面化（`难道你不知道吗 / 我无法接受这种事实`）
- 国内高概念项目是否把专有词翻译成日常可懂的剧情句，例如 `被世界删了 / 有人偷了寿命 / 证人被封口`
- 国内项目称呼系统是否符合关系层级（关系近用 `哥/姐/老张`，翻脸场用全名，不要全场全名互称）
- 国内项目网感词是否使用 `半年以上稳定词`，没有出现三个月内的过气热梗，且没有撑主角
- 同场对手两人对白把 `角色名：` 去掉后是否仍能凭句式判断是谁在说
- 这一集是否至少有一句"只属于这个角色 / 这段关系 / 这个场景"的台词
- AI 资产是否能稳定复用
- 常驻角色、常驻场景、核心道具是否被真正复用，而不是写到后面不断新增资产
- 是否出现角色变体失控、场景变体失控、道具状态失控
- 连续几集是否出现相同句式、相同场次骨架、相同威胁方式
- 是否出现固定开场句、固定施压句、固定反应句跨集复用
- 当前批次是否存在连续多集只抛新债、不推进旧债务
- 当前批次的悬念是否都只剩“真相是什么”，却没有挂住人物情绪和代价
- 人物是否真的在场，还是只是被台词提到却误写进场次
- 关键集是否给出足够明确的视觉支点，而不是只靠对白讲完
- 大场面或奇观场是否先被写成可生产序列，而不是一句抽象场面词直接压给正文和分镜
- 场景是否写出了和剧情强相关的时间 / 地点 / 光影 / 环境细节，而不是只报一个空地点
- 是否用具体细节替代了抽象情绪判断，如“很痛苦 / 很压迫 / 很危险”
- 对白是否真的发生在动作里，而不是所有人干站着轮流说
- 每场是否至少有一个最小 `发声单位`；如果出现纯动作静默场，是否真的值这个静默效果
- 关键节点的镜头提示是否只在必要处点到为止，而不是把正文写成分镜清单
- 是否出现自报家门式台词，尤其在角色首次出场、权力场、冲突场
- `OS / VO` 是否只在必要时出现，而不是代替本可拍出来的动作、反应和关系变化
- 必须出现的 `OS / VO / 屏幕字 / SFX` 是否缺失；如果声音、弹窗、屏幕文字、内心独白没有被标记，格式质检不得通过
- 重要信息是否本可通过动作、道具、他人反应或现场气氛传达，却被对白直接说破
- 对白是否在动作和反应之后补锋利，而不是先把局势讲平
- 关键台词是否至少完成了 `推进剧情 / 塑造人物 / 制造悬念 / 激化情绪` 中的一项
- 删掉最说明的那句台词后，这场是否仍然成立；若仍成立，说明当前对白过满
- 是否存在只解释前情、重复已知信息、替观众总结局势、说完后局势没有变化的台词
- 如果一场戏结束后，人物位置、关系、危险、下一步动作都没变，这场是否应该删、并或重写
- 当前主要事件是否完成了 `承接 / 升级 / 价值转变 / 下集勾连`
- 主角当集的行动是否仍然符合已锁定的 `投射身份 / 站队点 / 受压点 / 翻盘入口 / 强反击方式`
- 如果问题只集中在开场、尾钩、关键对峙、首次亮相或单个关系翻面，是否优先做了 `局部返工`
- 局部返工后，是否同步回填了 `分集追踪.md`、必要时回改了 `分集大纲.md`，并重过当前集检查

通过标准：
- 没有明显空集、功能集、重复钩子
- 当前批次至少有稳定的观看回报、记忆点和继续观看动力
- 没有严重命名漂移和设定冲突
- 可以继续下一批或进入交付

不通过动作：
- 先回改对应集剧本
- 如已影响上游逻辑，同步回改 `分集大纲.md`
- 必要时先收一份 `改稿指令单-*.md`，把本轮问题收束到 `3-5` 条再改
- 不继续下一批，不进入 `/交付`
- 所有项目默认每批都先过复盘，再继续下一批

## 三、交付前全剧体验通读复盘

在全部单集剧本完成并通过批次质检后、进入 `T15 全剧复核` 前执行。

不要只按“是否符合大纲 / 是否格式达标 / 是否有硬伤”检查；这一轮要模拟完整观看体验，站在用户、编剧和导演三个角度读完全季。

用户 / 观众视角：
- EP001 是否在最短时间内建立情绪、问题和追看理由
- 前 5 集是否兑现题材承诺，而不是只铺设定
- 每 3-5 集是否出现新的可见回报、关系翻面或不可逆后果
- 中段是否有疲劳感：同一类威胁、同一类反转、同一类会议或文件解释是否重复
- 观众能否用低术语复述每个阶段“主角现在要赢什么”
- 结局是否满足观众情绪，但没有写成超出现实边界的万能胜利

编剧视角：
- 主线因果是否从第一集推到最后一集，而不是靠后段补设定
- 主角行动是否持续主动，还是只等线索送上门
- 反派压迫是否升级，还是只重复“卡资料 / 群嘲 / 威胁”
- 证据获得是否有来源、代价和公开边界
- 人物声线是否越写越混，关键配角是否有自己的选择和代价
- 终局是否完成题材承诺，同时保留合法程序边界

导演 / 制作视角：
- 关键集是否有能被画面记住的视觉支点
- 是否连续多集依赖同一场景、同一站位、同一文件特写
- 手机界面、公告栏、投影、票箱、合同等屏幕 / 道具信息是否能被制作清楚表达
- 大会、群像、公开对峙是否可用局部反应、手部、屏幕字和道具动作执行
- 场景和道具复用是否形成连续性，而不是看起来像重复素材
