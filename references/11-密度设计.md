# 密度设计

这份规则专门处理三种密度：
- `情绪密度`
- `情节密度`
- `信息密度`

目的是让每一集在有限时长里既不空，也不挤。

## 一、为什么要单独设计密度

公开的编剧流程和微短剧行业实践基本都指向同一件事：
- 先做 `outline / treatment / beat sheet`
- 再写 `script`
- 竖屏短剧必须优先保证 `清晰、推进、情绪冲击、尾钩拉力`

这三种密度分别回答三件事：
- `情绪密度`：这一集有没有让观众产生明确感受
- `情节密度`：这一集有没有发生足够多的有效推进
- `信息密度`：这一集给了多少新信息，观众能不能吃得下

## 二、三种密度的定义

### 1. 情绪密度

看的是：
- 这集有几次明确情绪落点
- 情绪有没有从“压迫”走到“回报/代价/翻页”
- 情绪是不是只是重复一种施压
- 整部剧是否先钉住一条 `核心情绪线`

不是看：
- 台词喊得多不多
- 人物哭得多不多

补充口径：
- 短剧通常是情绪先行，不是慢慢铺情节后才让观众产生感觉
- `核心情绪线` 先决定这部剧主要让观众爽什么、怒什么、心疼什么、上头什么
- 情绪密度不是持续高吼，而是 `压 / 放 / 反差 / 动作` 带来的连续起伏
- 开头先打中一种明确情绪，中段至少给一次小释放、小反击或错误放松，结尾再把情绪期待拉高

### 2. 情节密度

看的是：
- 这集有没有最小状态变化
- 每场是否都承担推进
- 有没有有效的翻面、代价、升级、揭露
- 主要事件是否都服务主线
- 事件结束后人物处境是否真的变了

不是看：
- 场次数量多不多
- 事件名词堆得多不多

补充口径：
- 不是所有发生的事都算情节；只有服务主线、带因果、会升级冲突并改变人物处境的事件，才算有效情节
- 一个有效情节单元，优先满足 `因果锚定 / 冲突驱动 / 价值转变`
- 单集默认检查 `承接上一集 -> 冲突升级 -> 价值转变 -> 下集勾连` 这条链是否成立

### 3. 信息密度

看的是：
- 这集新增了几个关键新信息点
- 哪些信息是观众知道、主角不知道
- 哪些信息是主角知道、反派不知道
- 信息是不是通过冲突和动作抛出来
- 开场是否很快把人物、危机和痛点给清楚
- 这一集是否至少给出一个真正新信息

不是看：
- 设定说明写了多少
- 世界观词汇写了多少

补充口径：
- 信息密度优先按 `快 / 准 / 新 / 狠` 执行
- `快`：开场很快给到人物、危机和痛点，不慢热解释
- `准`：关键台词优先带潜台词，一句同时推进剧情、塑造人物、传递冲突
- `新`：每集至少给一个之前没有的新信息，不能整集只重复上一集已经成立的压迫
- `狠`：没有推进、塑造、悬念或情绪作用的句子和段落，优先删掉

## 三、建议的创作流程

### `/策划` T04 项目级密度画像

先定 `项目级密度画像`：
- 情绪密度：偏高 / 中高 / 中
- 情节密度：偏高 / 中高 / 中
- 信息密度：偏低 / 中 / 中高

还要先钉：
- `核心情绪线`
- `信息分发口径`

项目级判断要结合：
- 平台
- 题材
- 单集时长
- 总集数
- 受众

### `/策划` T08-T10 密度结构设计

把密度从抽象判断变成结构设计：
- 开篇阶段的情绪峰值安排
- 每个阶段块的推进强度
- 每一集的新信息点预算
- 哪些集负责“压迫”，哪些集负责“兑现”，哪些集负责“翻页”
- 开篇是否已经决定先打情绪、再给核心信息、再推情节
- 中段是否已经决定哪里做价值转变，避免一直热闹但人物处境不变

### `/剧本`

把密度变成单集控制：
- 本集主情绪峰值是什么
- 本集最小状态变化是什么
- 本集戏剧单位有几个，分别承担什么
- 本集新增几个关键信息点
- 本集是否出现“信息多但戏少”或“戏多但信息乱”
- 本集是否强化了全剧的 `核心情绪线`
- 本集主要事件是否完成 `承接 / 升级 / 价值转变 / 下集勾连`
- 关键台词是否做到 `快 / 准 / 新 / 狠`
- `正文层厚度` 是否成立：不要把 `本集创作简卡`、场次前置信息或字数误当成时长承载，真正看的是场次正文里有没有足够的动作回合、反应回合、阻力升级和可见后果

补充口径：
- 单集优先拆成若干 `戏剧单位` 来看，不只按场次来看
- 每个戏剧单位尽量同时带上 `动作 + 阻力 + 变化 + 后果`
- 如果一集新信息很多，但没有哪一条信息立刻改变人物处境、关系强弱或下一步行动，通常还是“信息多但戏少”
- 如果一集大多数场次都只是 `起因 -> 说清 -> 得出结果`，哪怕信息不少，成片也会显得过快

### `/复盘`

复盘不只看“好不好看”，还要看：
- 情绪密度是否疲劳
- 情节密度是否塌陷
- 信息密度是否过载或过空

## 四、按单集时长给建议区间

这些是推荐区间，不是死规则。

与 `12-单集写作协议.md` 优先级关系（避免冲突）：
- 本文给出"密度档位 + 推荐场次数"作为创作起点
- `12-单集写作协议.md` 第六章给出"正文厚度 + 行数 + 场次内动作骨架"作为正文硬门槛
- **当两者冲突时**：以 `12-单集写作协议.md` 的 `行数硬阈值 + 场次内动作骨架` 为准；本文的"场次数推荐"只是起点
- 例：本文写"`60-90` 秒优先 `3-4` 场"，但 `12-单集写作协议` 写"`60-90` 秒不少于 `28` 行有效正文"——如果按 `3` 场写不出 `28` 行，先回改"是否合并 / 拆集 / 换更密的密度档"，不要降低正文硬阈值
- 本文给的"密度档位"不是与场次数等价，而是约束"情绪落点 / 推进单元 / 新信息点"等**戏剧单位数量**

总规则：
- 先看这一集有没有足够的戏剧单位，再看它被拆成几场是否合理
- `场次数` 是参考指标，不是质量本体；正文厚度才是质量本体
- 特殊情况下，哪怕只有 `1-2` 场，只要单场内部已经具备明确目标、冲突、翻面、动作变化、情绪推进和结尾追更力，且整集行数仍然能过 `12-单集写作协议` 的硬阈值，仍然可以成立
- 反过来，哪怕场次够多，如果每场都只是在解释、搬运信息、重复同一种对线，也仍然会显得空

### 45-75 秒

- 场次数：`2-3`
- 明确情绪落点：`1` 次主落点，必要时 `1` 次次级反应
- 有效推进单元：`2-3`
- 新关键信息点：`1-2`

适合：
- TikTok 美国首版
- 高钩子、高 cliff 的关系戏

### 60-90 秒

- 场次数：`3-4` 为默认优先，`2` 场只作为例外
- 明确情绪落点：`1-2`
- 有效推进单元：`3-4`
- 新关键信息点：`1-3`
- 有效正文行数：默认不少于 `28` 行；如果低于这个量级，必须确认单场内部有连续翻面和足够反应回合

适合：
- 抖音主流 AI 漫剧
- 大多数甜宠、复仇、身份翻转、奇幻都市项目

补充规则：
- 对 `60-90` 秒主流单集，默认优先按 `3-4` 场组织
- 只有当单集本身是强对峙、强动作、强反转，且单场内部能完成明显翻面、空间切换或动作升级时，才允许用 `2` 场撑起一集
- 极少数情况下，如果单场本身就是完整的小序列，内部有连续翻面、动作推进、关系变化和强钩子，`1` 场也可以成立
- 如果一集只有 `2` 场，同时又没有内部翻面、动作层次或场域变化，通常会落成“信息有了，戏不够”
- 如果多数场次只有 `1-2` 条动作 + `1-2` 轮对白直达结果，通常会落成“计划写得很满，成片却只有 `40-50` 秒感”的薄集
- 遇到时长承载风险，优先补 `第二动作 / 第二阻力 / 第二反应 / 第二轮压迫`，必要时并拆单集，不要靠解释性台词拖时长

### 90-120 秒

- 场次数：`3-5`
- 明确情绪落点：`2`
- 有效正文行数：默认不少于 `34` 行
- 有效推进单元：`4-5`
- 新关键信息点：`2-4`

适合：
- 精品化项目
- 设定稍复杂但仍要求移动端可消费的项目

## 五、开篇阶段的密度建议

### EP01-EP03

- `情绪密度` 要偏高：快速让用户感到压迫、吸引、羞辱、危险、反击或心动
- `情节密度` 要高：不要写成设定说明会
- `信息密度` 要克制：只给理解主冲突和主关系所需的信息
- 开篇优先先打中一种明确情绪，再很快给出人物、危机和痛点

### EP01 首集加长口径

首集可以比常规集长，但只能因为它承担了更多建制任务，不能因为开场慢。

首集默认要比常规集多承担：
- 主角处境和痛点
- 目标市场能一眼识别的题材承诺
- 主要关系或核心对手的第一次压迫
- 主角第一次主动动作或反击
- 一个完整小回报
- 一个足以进入 EP002 的具体尾钩

执行建议：
- 常规 `45-75` 秒项目，首集通常按 `75-120` 秒设计，优先 `3-4` 场或 `4` 个以上戏剧单位
- 常规 `60-90` 秒项目，首集通常按 `90-150` 秒设计，优先 `4-5` 场或 `4-5` 个戏剧单位
- 国内免费 APP / IAA 长线项目，首集可到 `100-180` 秒，但必须有完整情绪兑现和可切片高光
- 如果首集和常规集一样只有 `2-3` 个戏剧单位，就不要标“首集加长”；先补主角处境、第一次压迫、第一次回报和尾钩

### EP04-EP10（如项目采用较长开篇阶段）

- `情绪密度` 要维持并变形，不能只重复同一种压迫
- `情节密度` 要继续高，至少要有第一次明显翻页或代价
- `信息密度` 可以略升，但每集仍建议只给 `1-3` 个关键信息点

## 六、单集节奏公式

短剧节奏不是单纯写快，而是让观众在一集内连续得到：
- 情绪冲击
- 局势变化
- 强期待
- 结尾钩子

对 `60-90` 秒主流集长，默认参考这 4 个节奏节点：
- 开头几秒先给 `情绪冲击`
- 前段尽快给出 `局势变化`
- 中后段形成 `强期待`
- 结尾必须抛出 `下一集必须回答的问题`

### 1. 情绪冲击

优先让观众先感到：
- 危险
- 羞辱
- 心疼
- 愤怒
- 震惊
- 好奇

规则：
- 开场优先让情绪发生，不优先解释背景
- 情绪冲击必须和主线相关，不用支线小插曲抢开场

### 2. 局势变化

情绪之后要尽快让事情发生变化。

常见变化：
- 主角从被动接招变成必须做选择
- 危机升级成更明确的任务
- 隐藏信息暴露
- 关系位置变化
- 代价突然增加

规则：
- 不要让一整段前戏都停留在重复原危机
- 观众在单集前段就要感觉到“事情已经比刚开始更严重，或者方向已经变了”

### 3. 强期待

强期待不是普通悬念，而是让观众明确知道：
- 主角接下来必须做什么
- 不做会失去什么
- 为什么现在就得做

有效强期待优先包含：
- 明确目标
- 明确代价
- 明确时限或压力

### 4. 结尾钩子

结尾不是把这一集说完，而是要再抛一个更危险的新动作或新变化。

规则：
- 结尾钩子优先落在主线上
- 优先改变下一集的目标、局势或关系
- 如果结尾只是硬断句，不算有效钩子

### 默认执行口径

- 不把 `3 秒 / 15 秒 / 45 秒` 当死板秒表
- 把它当成单集节奏顺序：`先打情绪 -> 再变局势 -> 再锁期待 -> 最后抛钩子`

## 七、戏剧单位口径

为了避免单集写成“解释 + 对线 + 尾钩”，正文优先按 `戏剧单位` 来检查内容是否扎实。

最小要求：
- 一个戏剧单位里，至少要有明确目标、阻力、动作推进和变化结果
- 变化结果优先落到 `关系 / 权力 / 危险 / 资源 / 位置 / 规则` 的可见后果上
- 同一场里可以有多个戏剧单位，但不能整场只有平铺直叙的信息交换
- 常规 `60-90` 秒单集，默认要能拆出 `5-7` 个可拍剧情拍点；首集或 `90` 秒以上单集默认要能拆出 `7-10` 个剧情拍点
- 每个剧情拍点至少包含 `人物 + 动作 + 阻力或后果`；只写“尾钩落地 / 场域压迫 / 主角反击”不算剧情拍点

常见失效写法：
- 只拿到一条信息，没有任何现场后果
- 只靠人物吵架，局势不动
- 只确认“原来如此”，但角色下一步没被逼出来
- 只有结尾钩子，前面没有真正的戏剧过程
- `1` 分钟左右的单集，优先保证至少 `3` 个明确爆点
- 节奏快不等于语速快；节奏快指的是情绪反应快、局势升级快、选择压力来得快、下一步目标明确得快

## 八、拉扯感的默认执行口径

拉扯感不是简单拖剧情，也不是只让角色一直受苦。

更稳的默认口径是：
- 先钉 `阶段爽点中点`
- 再把主角压进足够具体的谷底
- 中间给一次或多次 `错误放松`
- 在观众刚松下来时重新 `再加压`
- 最后再把爽点真正炸开

适合优先设计拉扯感的项目：
- 复仇
- 豪门关系戏
- 身份翻转
- 双时间线反杀
- 惊悚或逃生
- 任何强调“爽点要炸”的项目

不建议写成：
- 一路只受苦，没有中间松紧变化
- 一路只打脸，没有真正压下去的谷底
- 只有结果，没有错误放松和再加压

项目级先回答：
- 每个主要阶段的爽点中点是什么
- 在爽点前，主角最深会被压到哪里
- 哪一刻会给观众“危机解了”的错误预期
- 哪一下会在错误放松后重新加压

批次和单集层再回答：
- 这一批主要拉扯的是哪一段
- 这一集承担 `压 / 松 / 再压 / 炸开` 里的哪一个节点

如果一个阶段已经只有直线受压或直线打脸，说明这段拉扯还没有设计成立。

## 八点五、竖屏 9:16 视觉语言提醒

本 skill 不写镜头、不写分镜，但写正文时仍要让画面服务于 `9:16` 的播放形态。这一节只服务"写场景前置字段和动作行时该选什么样的画面"，不替代分镜稿。

竖屏天然友好的视觉语言：
- 近身、特写、半身——脸部表情、手部动作、关键道具优先
- 垂直运动——上下楼梯、抛接、跌落、抬头/低头、电梯、观景台、通信设备从口袋掏出
- 前景元素——栏杆、玻璃、雨幕、人群、通过前景遮挡实现"信息差暴露"
- 双人对峙正面或竖直构图——身高差、压迫俯拍、仰拍权力反转
- 单一焦点——竖屏不擅长群像广角，单场尽量只锁 `1-2` 个视觉焦点

竖屏写作时优先做：
- 视觉记忆点优先选竖直方向上发生的动作（举起、按下、抬头、低头、坠落、上升）
- 道具特写优先于全景；权力对峙优先于群像调度
- "走到下一个地点"这类水平推进，能并就并，不要单独成场

竖屏写作时避免：
- 大幅水平横移、群像并列、远景全景定场
- 同一场内频繁左右切换视角；竖屏切来切去观众容易丢轴
- 主要靠环境信息暗示——竖屏看不清远景细节，要靠道具/动作/反应直接传达

口径：
- 这一节只是写作时的取景偏好提示，不写进正文模板字段，也不替代场次执行稿结构
- 项目本身是横屏改竖屏或多端发行时，先在 `项目设定.md` 的 `资产与制作` section 写清主发行形态

## 九、长线项目的额外要求

`60` 集以上项目，密度设计不能只看单集。

至少还要设计：
- 每 `8-12` 集一次明显情绪高峰
- 每 `10` 集一个剧情块目标
- 每 `10-20` 集一次关系/身份/权力翻页
- 支线的信息投放节奏，不要只顾主线

## 十、三种常见失衡

### 1. 情绪密度过低

表现：
- 角色一直在做事，但观众没感觉
- 每集都有推进，但没有爽点、痛点、欲望点

修法：
- 增加明确回报
- 增加人物反应
- 把抽象推进改成可见后果

### 2. 情节密度过低

表现：
- 一集看完像只完成了“铺垫”
- 连续多场都在说同一件事

修法：
- 压缩场次
- 每场只保留一个主要推进动作
- 强制补 `状态变化`

### 3. 信息密度失衡

过高表现：
- 一集塞太多设定、关系、秘密、规则
- 观众只能“理解”，不能“感受”

过低表现：
- 一直在吵、一直在压迫，但没有新认知
- 尾钩只有情绪，没有具体兴趣点

修法：
- 默认每集先定 `1-3` 个关键信息点
- 先保留最影响下一集追更的那几个
- 其余信息后移

### 4. 情绪、情节、信息三者没咬合

表现：
- 情绪有了，但剧情不往前走
- 信息有了，但观众不在乎
- 情节很快，但人物没有感觉

修法：
- 先确认这一集的 `核心情绪线` 有没有被强化
- 再确认主要事件是否真的改变了人物处境
- 最后确认新信息是否直接影响下一步目标、代价或选择

## 十一、默认执行

如果用户没有明确指定特殊风格，默认采用：
- `情绪密度`：中高
- `情节密度`：中高
- `信息密度`：中

原因：
- 这更符合抖音和 TikTok 竖屏短剧的消费方式
- 也更适合 AI 漫剧在有限资产下维持清晰表达
