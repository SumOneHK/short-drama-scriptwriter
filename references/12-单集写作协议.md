# 单集写作协议

这份协议服务 `/剧本` 阶段，目的是避免越写越飘。

快速定位：
- 开始写一批正文前：读"写一批正文前最少要回看什么"
- 分集大纲转场次执行稿：读"大纲→场次执行稿的转化样例"
- 关键集拆法：读"关键集先做 6 节拍"
- 写单集前自问：读"写每一集前先答 12 个问题"
- 写场面：读"每场都要过四问"和"正文层的硬规则"
- 对白 / 本地化 / 屏幕字：读"对白规则"
- 写完一批或改稿：读"写完一批后必须回填"和"改稿优先级"

## 一、写一批正文前最少要回看什么

开始写 `EPxx-EPxx` 前，至少回看这些内容：
- `00-立项/锁题摘要.md`
- `01-策划/故事梗概.md`
- `01-策划/故事大纲.md`
- `01-策划/故事节拍表.md`
- `01-策划/人物小传.md`
- `01-策划/项目设定.md`
- `01-策划/世界观设定.md`
- `01-策划/阶段大纲.md`（若有）
- `01-策划/分集大纲.md`
- `90-内部工作稿/分集追踪.md`（首批正文前必须已经初始化；之后回看累计稿，不再视为可选）
- 当前批次对应的 `90-内部工作稿/批次写作单-EPxxx-EPxxx.md`（若已有）
- 当前批次对应的 `90-内部工作稿/单集达标检查-EPxxx-EPxxx.md`（若已有）
- 上一批已经完成的剧本

不要只看模板或一句话大纲就直接写正文。
如果真实项目文件已经存在，优先回看已批准产物，不要回到模板重新想一遍。
不要在正文阶段依赖提前汇总的 `剧本圣经.md`；按当前批次需要读取上面的分项源文件，减少上下文负担和重复口径。
`分集追踪.md` 是首批正文前的硬门槛产物；如果它还不存在，先初始化一版，再进入正文。
`批次写作单` 和 `单集达标检查` 都是按需文件；如果当前批次靠 `分集大纲.md` 已经写得很清楚，可以先不补。
正式分集稿只写轻量 `本集创作简卡`，不要把完整检查单写进 `第XXX集.md`。检查问题进入 `单集达标检查`，批次策略进入 `批次写作单`。

补充：
- 写正文前，先确认当前批次对应的关键翻面是否对得上 `故事节拍表.md`
- 如果 `故事节拍表.md` 的关键节拍还没真正落进当前批次的 `分集大纲` 或 `批次写作单`，先回改上游，不直接硬写正文
- `批次写作单` 用来决定这一批怎么写，不用来累计长期状态；长期状态继续写进 `分集追踪.md`
- `单集达标检查` 用来决定当前集能不能落稿，不用来另写剧情

批次补充：
- 所有项目每批先写 `1-2` 集代表样稿
- 代表样稿不过关，不继续铺本批
- 代表样稿重点检查：声线差异、第一场承接、动作具体度、公开场是否真的有场面调度
- 长线、海外项目或精品要求项目，对代表样稿的要求更严格
- 代表样稿还要检查动作骨架差异：相邻集的 `第一动作 / 施压动作 / 反应动作 / 翻面动作` 不能连续复用同一种句式或同一种物理动作
- 批次超过 `10` 集时，先在 `批次写作单` 写出 `动作变体池`；同一动作骨架出现第 `3` 次前必须换空间、换道具、换权力关系或换后果
- 如果在代表样稿前就已经拿不准人物声音或关系张力，先试写 `1-3` 个样场，再决定要不要正式推进本批
- 试写样场优先选 `主角首次亮相场 / 核心对峙场 / 高光翻盘场`

## 二、关键集先做 6 节拍

以下集数先做 `6` 节拍，再写场次：
- 首个强卡点集
- 身份或秘密重大揭露集
- 大反杀集
- 关系确认或决裂集
- 结局集

推荐 6 节拍：
1. 承接上一集尾钩
2. 升压
3. 临界点
4. 转折动作
5. 余震
6. 新债务

规则：
- 不单独要求再立一份 `分集节拍.md`
- 关键集的 `6` 节拍直接写进当前批次的 `批次写作单-EPxxx-EPxxx.md`、当前集写作备注，或对话中的临时写作拆解即可
- 如果一集必须先靠节拍拆解才能写清，说明它还不适合直接进入正文
- 这一层是正式写作方法，不是随手备注

## 二·补、大纲→场次执行稿的转化样例

从分集大纲一句话到场次执行稿必须有可复现的转化路径。这一节是 `分集大纲 → 单集正文` 的方法单一真相源，写每一集前先按这个路径做转化。

### 转化五步法

1. `观众口径压一句`：用普通观众能复述的话讲本集会发生什么，不用项目术语
2. `四链拆解`：分别写一行 `情节承接 / 冲突升级 / 价值闭环 / 下集勾连`
3. `场次分配`：按 `规格建议` 把本集拆成 `3-4` 场，每场对应 `承接+起势 / 升压 / 临界翻面 / 余震+尾钩` 中的一段
4. `每场目标→可见动作骨架`：每场把目标翻成 `第一动作 / 第一阻力 / 反应 / 第二阻力 / 反制 / 翻面 / 余波`，先骨架再台词
5. `首场前 4 行硬规则`：第一场 `1-1` 前 `4` 行必须有可见冲击点；不允许从背景交代、独白或解释规则开场

补充动作：`藏与显影检查` 不改变五步法顺序，但在第 2-4 步之间同步完成。先标清本集 `显性事件 / 暗藏信息 / 可发现线索 / 回收接口`：显性事件必须让观众当下看懂，暗藏信息只能藏进可见细节，不能藏掉人物目标、冲突和后果。

### 样例 A：高概念项目（沿用 EP001）

大纲条目：
- 一句话：这个人明明站在台上，世界却说他没出生过
- 本集回报：主角发现"自己被世界删除"的压迫机制
- 尾钩：同僚集体认不出主角，名册上空了一格

四链拆解：
- 承接：剧前序幕，无可承接
- 升级：被无视 → 被否认 → 被名册删除
- 闭环：主角拿到第一个反向证据（旧纸上仍写着他的名字）
- 勾连：旧纸上的字开始自行消失，新人物逼近

场次分配（4 场，60-90 秒规格）：
- 1-1 日内 朝堂：站班→请示→被空气穿过
- 1-2 日内 名册室：自查名册→自己一格空白
- 1-3 夜内 旧屋：翻旧物→旧纸上仍有他的名字（小回报）
- 1-4 夜内 旧屋：字开始消失→门外脚步逼近（尾钩）

1-1 开场前 4 行（看见后果，不讲规则）：

```
### 1-1 日内 朝堂
人物：许平（被删者，30）/ 周临（同僚，35）/ 众官
△ 许平举笏出列，朗声请示。
△ 周临像没听见，绕过他往前走。
△ 众官全部越过许平，站位自动合拢，仿佛他不在。
许平：诸位——
```

不是先讲规则，而是先让观众看见"被空气穿过"的状态；规则名（如 `命籍 / 见证人 / 暗文`）在本集都不出现。

### 样例 B：现代爽剧 / 甜宠 / 都市情感（短样例）

大纲条目：
- 一句话：女主在年会上发现未婚夫和闺蜜在一起
- 本集回报：女主当众撕婚约但保住了项目
- 尾钩：神秘投资人递来一张名片

四链拆解：
- 承接：上集女主刚被升职，准备公开订婚
- 升级：察觉异样 → 撞见私情 → 当众反击
- 闭环：女主反击成功，项目权未失
- 勾连：男主（神秘投资人）递名片

场次分配（3 场，45-75 秒规格）：
- 2-1 年会大厅：举杯敬酒→未婚夫迟疑→女主察觉
- 2-2 化妆间走廊：撞见未婚夫和闺蜜
- 2-3 年会返场：当众反击→投资人递名片（尾钩）

2-1 骨架：
- 第一动作：女主举杯
- 第一阻力：未婚夫迟迟不举
- 反应：女主放下杯，转身往后台走（不哭、去查）
- 第二阻力：闺蜜笑着替未婚夫敬酒
- 翻面：女主收到一条匿名短信
- 余波：女主走向化妆间走廊（接 2-2）

### 样例 C：海外 TikTok / 美国英语（短样例，含中文注释）

大纲条目：
- 一句话：The stepmother announces an "engagement"—but it's the protagonist she's planning to sell off.
- 本集回报：女主当众拒婚，但发现继母手里捏着母亲的医疗记录
- 尾钩：陌生男人递来一张律师名片

3-1 开场骨架（英文是 source of truth，中文注释只补潜台词）：

```
### 3-1 日内 Stepmother's mansion, living room
人物：Ava（被设计的女儿，21）/ Caroline（继母，45）/ Caroline's friends
△ Caroline 举杯，朝 Ava 微笑。
Caroline: Everyone—I have an announcement.（庆祝口气，其实是宣判。）
△ Ava 还没意识到自己是主角。
Caroline: Ava's engaged.（轻描淡写，因为她从没问过 Ava。）
△ Ava 手里的酒杯顿住。
Ava: I'm what?（只问得出这一句。）
```

注意：`Ava's engaged.` 用 contractions、不用 `Ava is engaged.`；中文注释补 `因为她从没问过 Ava` 这种潜台词，不写 `艾娃订婚了` 这种字面翻译。

### 转化避坑

- 大纲条目里的"尾钩"必须在最后一场最后两行落地；写成"暗示有秘密 / 留下伏笔"这类抽象句的，回改大纲
- 大纲条目里的"藏"必须能落成可见线索；写成"她隐约觉得不对 / 他似乎藏着秘密 / 这里埋下伏笔"的，回改为具体道具、反应、屏幕字、环境矛盾或动作错位
- 第一场前 `4` 行必须有具体动作或可见冲击；用"主角望着镜子思考人生"或"主角回忆童年"开场的，回改
- 高概念项目走样例 A 的"看见后果→规则藏后"路径；现实主义走样例 B 的"行动+发现+反击"路径；海外项目走样例 C 的"contractions + 注释补潜台词"路径
- 拆成 2 场时，每场内部必须有 ≥ 2 次翻面；只有 1 场时按第十三章 `单场承载追问` 处理，否则按规格拆 3-4 场
- 骨架不全（缺第二阻力或翻面）就开始填台词的，是 12 问没答清就硬写正文；回改 12 问

## 三、正文易读性优先

正文首先要让普通观众一遍看懂。

每集落稿前先压成一句观众能复述的话：
- EP001：这个人明明站在台上，世界却说他没出生过。
- EP002：他找到第一个被偷寿命的人，证人刚要开口就被封住。

如果一句话必须依赖项目内部术语才能讲清，先回改本集理解链。

### 低术语写作

- 每集只新增 `1-2` 个必须记住的新名词。
- 同一集不要同时解释多套名词，例如命牌、命镜、命灯、废诏、审诏、暗文、见证人全部一起讲。
- 开篇前三集优先使用观众能直接理解的表达：`被世界删了 / 有人偷了寿命 / 证人被封口 / 名字被抹掉`。
- 专有词第一次出现必须绑定可见后果：命牌掉粉、命灯快灭、旧纸记得名字、金字封住喉咙。
- 屏幕字不要像规则说明书。优先写具体事实：`许平三十年寿数，转给周临`；少写抽象规则：`见证人在场 / 暗文显形 / 受益者未还`。

#### 易读性可量化硬规则

为了避免"是不是术语过载"成为主观判断，本节给出可证否标准。判稿时按这些标准逐项打勾，全部满足才算易读性达标。

**硬规则分两层**：
- `生成时规则`（模型写正文时主动遵守）：选词风格、规则词不在第一次出现时解释、屏幕字优先写具体事实、术语藏后果
- `校验时规则`（必须由 `scripts/check_episode.py` 做确定性计数 + 独立复检审核）：叠加屏幕字字数、画内文字可读性、环境标签字、规则词频次、前置字段比例、有效行数

校验时规则即使模型自我感觉"应该过线了"，结论也无效；必须运行 `python3 scripts/check_episode.py --regulation-words "命格,暗文,命债,见证人" 02-剧本/第XXX集.md`，把脚本输出贴进 `单集达标检查` 或 `质检检查点.md`，作为可证否证据。

`新术语` 的判定（每集新增 ≤ 2 条）：
- 凡是普通观众第一眼无法用日常词复述、必须看到"专门解释 / 屏幕字 / 旁白"才能理解的名词，都算"新术语"；包括项目自创规则名（`命格 / 暗文 / 命债`）、原创机构名（`审命司 / 命录司`）、原创道具名（`命牌 / 命灯 / 命镜`）
- 不算新术语：日常生活词（`手机 / 朋友圈 / 派出所 / 律师函 / 房产证`）、已知文化符号（`Thanksgiving / SAT / 朝堂 / 御史`）、人物姓名
- 同一集中累计出现 ≥ 3 个新术语时，本集判易读性 `需修改`，必须把超出的术语藏进可见后果或推迟到后续集

`屏幕字字数` 按画面文字分层检查。`屏幕字` 是制作标记，不是单一字数配额：
- `叠加屏幕字`：作者额外叠在画面上的提示、规则说明、弹幕式总结、转场字幕。单条不超过 12 个汉字（英文不超过 8 个 word），全集累计不超过 30 个汉字（英文不超过 20 个 word）；超过判硬阈值越线。
- `画内信息文字`：剧情世界内本来存在的手机短信、群聊、系统弹窗、直播评论、文件、公告、公示、账本、名单、票据、病历、成绩单、考勤、档案、合同、判词等。这类不进入叠加屏幕字 30 字配额，但单条建议不超过 24 个汉字，整集建议不超过 120 个汉字；超过只记 `WARN`，由独立复检判断是否变成"读材料"。
- `环境标签字`：姓名牌、人名牌、门牌、路牌、招牌、地名、机构标识等低负担标签。单独统计，不纳入叠加屏幕字配额；但如果标签被写成一整句说明，独立复检仍可判为易读性问题。
- 推荐在制作标记中写明分类：`屏幕字（叠加）：...`、`屏幕字（群聊）：...`、`屏幕字（文件）：...`、`屏幕字（系统）：...`、`屏幕字（病历）：...`、`屏幕字（成绩单）：...`、`屏幕字（姓名牌）：...`。海外稿对应 `Screen Text (overlay): ...`、`Screen Text (phone): ...`、`Screen Text (file): ...`。
- 未加分类但内容明显含 `公告 / 公示 / 明细 / 账本 / 名单 / 合同 / 授权 / 群聊 / 短信 / 病历 / 成绩单 / 考勤 / 档案 / access denied / system alert` 等画内信息词时，脚本按画内信息文字处理；无法判断的默认按叠加屏幕字处理。
- 用屏幕字写规则的（`见证人不在场则失效 / 命格不可逆 / 受益者必须现身`）一律判 `需修改`，不能靠标成 `画内 / 文件 / 证据` 绕过，必须改成可见后果。

`规则词在正文中占比` 的硬阈值（每场出现 ≤ 1 次，整集 ≤ 3 次）：
- "规则词"指项目自创的抽象规则名（如 `命格 / 暗文 / 命债 / 命纹 / 见证人 / 受益者`）
- 单场出现规则词 ≥ 2 次，本场判 `需修改`：必须把第二次起改写成具体后果或具体人物动作
- 全集累计规则词出现 ≥ 4 次，本集判 `需修改`：先回改大纲降低规则密度，再写正文
- 例外：当一个规则词第一次出现时，旁边紧跟可见后果（如 `命格不可逆——同僚视而不见`），本次出现可以记 0.5 次

`规则藏进可见后果` 的判定：
- 一个新规则不能在出现的同一句里被解释；必须先让观众看见后果（人物动作 / 物理变化 / 他人反应），再让规则名出现（如果出现）
- 如果一集里有任何一条新规则是通过"角色对白解释 / 旁白说明 / 屏幕字定义"传递的，本集判易读性 `需修改`

### 藏与显影

`藏` 的目标是让观众获得发现乐趣，不是让观众困惑。

每集写前先分两层：
- `第一层`：当集显性事件。观众必须一遍看懂谁想要什么、谁在拦、发生了什么后果
- `第二层`：暗藏信息。只允许藏真相、动机、关系、规则或情绪的第二层意义

可用落点：
- 道具反常：钥匙、戒指、芯片、文件、氧气表、旧照片的状态和官方说法不一致
- 反应错位：某人对一个普通词、标志、声音或地点反应过大 / 过慢
- 台词避让：角色避开真实关系称呼，或反复用更安全的词替代
- 屏幕字异样：时间戳、权限、名单、费用、系统提示和现场叙述冲突
- 空间矛盾：被宣称封闭、废弃、不可达的地方留下使用痕迹
- 行动余波：人物提前做了一个当下不解释、但后续显影有意义的保护或破坏动作

写法边界：
- 不要用 `似乎 / 仿佛 / 另有秘密 / 意味深长` 当线索；这只是作者提醒
- 不要让 OS / VO 替观众解释暗线
- 不要为了藏而推迟当集回报；每集仍要有可见输赢、关系变化、危险升级或情绪释放
- 显影时优先回扣前文已经出现过的物件、动作、屏幕字或反应；只靠新证人 / 新文件突然解释，视为弱回收

### 规则藏在后果里

高概念规则不要靠人物解释，先让观众看见后果：
- 不先讲“无命者不入命籍”，先写同僚忘掉主角、名册空出一格、命牌变空白。
- 不先讲“审命三要素”，先写旧纸指向受害者、命牌显出转移关系、证人刚要说话就被封。
- 不先讲“命债暗文”，先写谁得了好处、谁失去寿命/名字/记忆、现场如何反噬。

### 代表样稿易读性门槛

EP001-EP002 代表样稿必须优先过易读性：
- EP001 只承担一个主理解链：主角被世界删除。
- EP002 只承担一个主理解链：第一名受害者出现，证人被封口。
- 设定名词可以存在，但不得抢过人物处境和现场后果。
- 读完正文后，如果只能说“规则很复杂”，而说不出“谁被害、谁封口、主角下一步干什么”，不继续铺本批。

## 四、写每一集前先答 12 个问题

1. 上一集尾钩，这一集前段怎么接？
   这层承接有没有真的落到第一场动作或对白里？
   上一集“下一集必须承接”、这一集“开场承接”和 `分集追踪.md` 的承接描述，是否用了同一核心名词短语？
   `分集追踪.md` 里的尾钩和承接是否保留了正文里的完整兴趣句，而不是被缩写成抽象名词？
   从 `EP002` 开始，这一集 `1-1` 是否直接带出了上一集尾钩里的 `人物 / 道具 / 地点 / 动作` 之一？
   `批次写作单` 里有没有提前写出 `上一集尾钩核心短语` 和 `开场第一动作或第一句对白`？
   `开场承接` 禁止写成 `承接上一集具体尾钩 / 上一集留下的场域疑问 / 继续调查` 这类泛词，必须直接写上一集回填里的完整承接句。
   第一场正文必须让这个承接句落成可见物、地点、动作或对白；只在简卡里写承接，不算真的承接。
2. 这一集最小状态变化是什么？
3. 这一集谁必须在场，谁不该出现？
4. 这一集有没有新增功能性角色，为什么非加不可？
5. 这一集兑现什么旧债务，新增什么新债务？
6. 这一集的观看回报是什么？
   这一集的回报类型和相邻两集是否不同？
   它改变的是输赢、关系、身份、资源、规则、舆论还是情绪位置？
   如果只是“主角再次让对手失控”，先回改分集大纲或批次写作单。
7. 这一集最值钱的画面是什么？
8. 这一集的尾钩具体拉力是什么？
9. 哪些名字、关系、道具、伤势、规则必须连续？
10. 如果删掉其中一场，剧情还能正常推进吗？
11. 这一集观众能否用一句日常话复述？是否术语过载？
12. 这一集主角的关键动作，是否仍然符合已经锁定的 `主角投射身份 / 主角站队点 / 主角受压点 / 主角翻盘入口 / 主角强反击方式`？

如果答不清，不要直接落正文。

补充：
- 每集落正文前，再用 `情节承接 / 冲突升级 / 价值闭环 / 下集勾连` 检查一次；四链缺一项，优先回改分集大纲或批次写作单
- 每集钩子先选清类型：`新人物 / 新物品 / 新状况 / 身份颠覆 / 人性背刺 / 胜负逆转 / 真相反转`，再写具体动作和后果
- 每集悬念先选清信息差结构：`观众知道角色不知道 / 观众不知道角色知道 / 双方都只知道一部分`，并确认它挂在人物安危、关系、输赢、尊严或情感上
- 每集若承担暗线，先选清 `藏与显影`：本集显性事件是什么、暗藏信息是什么、观众能从哪个具体细节发现异常、预期何时回收
- 每集回报先选清回报类型：`反击 / 证据 / 关系 / 身份 / 权力 / 情绪 / 规则 / 代价`，相邻集不要连续重复同一类型
- 不要连续多集都只新增悬念、不推进旧债务
- 如果当前批次已经连续 `2` 集只抛新债，下一集优先回收或部分兑现旧债，再继续叠新债
- 写正文前，在 `本集创作简卡` 里只补最少必要项：`本集戏剧单位 / 本集情绪落点 / 本集信息投放顺序 / 本集中段错误放松或小释放 / 本集可见后果`
- 每项尽量一行；如果写成多段说明，说明这部分应该回到 `分集大纲 / 批次写作单 / 单集达标检查`，不要挤占正文
- 如果问题只集中在开场、尾钩、关键对峙、单个关系翻面或首次亮相，优先做 `局部返工`，不要一上来整批回炉

## 五、每场都要过四问

写场次时，每场都先过这四问：
- 这场承接什么？
- 这场推进什么？
- 这场改变什么？
- 这场留下什么？

如果一场戏只是在解释、复述、搬运信息，这场通常要删、并、改。

建议再补两项最小增量信息：
- 这场的 `事实锚点` 是什么？
- 这场的 `场景状态差量` 是什么？
- 这场的 `视觉记忆点 / 动作升级点 / 可见后果` 分别是什么？

如果一场主要只是 `接电话 / 看文件 / 独处沉默 / 走进新地点`，却没有至少 `2` 个以上表演节拍、反应回合或冲突转折，这场通常撑不起目标时长。

如果一场的前置信息写得比正文还厚，这场通常还没真正写开。优先补可见动作、反应回合、状态差量和现场后果，不要继续往前置信息里加说明。

对白补充再过三问：
- 这场最说明的那句台词删掉以后，这场还成不成立？
- 角色身份、目的、威胁和权力关系，能不能先靠动作、道具、他人反应和现场气氛看懂？
- 这场对白是在补刀，还是在替动作做说明书？
- 这场有没有至少一个最小 `发声单位`；如果是纯动作静默场，为什么非静默不可？
- 这一场有没有把本可外化的信息偷懒写成 `OS / VO`？
- 这一场是否把必须标记的声音、屏幕字或内心独白漏标成普通动作/对白？

## 六、正文层的硬规则

- 正文是 `场次剧本`，不是分镜稿
- 正式场次格式必须稳定：`### 1-1 日内 地点`、`人物：角色A / 角色B`、`△` 动作行、`角色名：` 台词行、按需 `OS：/ VO：/ 屏幕字（类别）：/ SFX：`
- 场次标题必须含 `场景序号 / 时间 / 内外景 / 地点`；不要写成 `### 场 1：内景，地点`
- 每场至少有一行 `人物：` 和若干 `△` 动作行；首次出场人物必须在 `人物：` 行补身份说明
- 广播、录音、系统播报、屏幕弹窗、文件文字、手机信息、内心独白、闪回旁白必须使用对应标记，不能随手写成 `广播：/ 机械音：/ 系统提示：/ 黑伞灯浮字：`
- 不写机位、运镜、镜头号
- 默认不写景别；如需兼容外部团队格式，只允许在动作行中偶发使用 `【**近景**】` 这类简短景别提示，不扩展成分镜稿
- 不写 AI 提示词
- 不用长段解释性对白替代动作
- 不用屏幕字充当规则说明书；屏幕字优先呈现具体事实、名单、判词、转移结果或现场状态
- 高概念项目每场都要把抽象规则翻译成观众能看懂的结果：谁消失、谁变老、谁失声、谁得利、谁被公开打脸
- 每场最好都有可视化动作，不要只有说话
- `本集创作简卡` 和场次前置信息写得满，不等于正文已经够厚；如果去掉这些前置栏，这场正文本身仍要能成立
- 场次前置字段写完后，正文至少要补出 `目标动作 -> 第一阻力 -> 反应/选择 -> 第二阻力或二次压迫 -> 反制动作 -> 场内翻面 -> 余波/可见后果`，不能只用一条动作和一两句对白直达结果
- 常规 `60-90` 秒单集，全剧本场次正文默认不少于 `28` 行有效正文；`45-75` 秒不少于 `22` 行；`90-120` 秒不少于 `34` 行。首集加长默认不少于 `34-40` 行。低于这个量级时，先判断是否应该并集、拆集或回改分集大纲
- 如果场次正文行数少于或接近前置字段行数，默认先判为正文偏薄；优先把前置栏里的动作、阻力、反应和后果改写进正文
- 结尾钩子不要机械重复同一种句式
- 不要把固定开场句、固定施压句、固定反应句反复抄到下一集
- 不要把同一种动作骨架批量套用到不同集，例如 `角色没有后退 + 反手贴出道具 + 灯光显出痕迹` 这类组合只能作为一次具体场面，不能替代动作设计
- 长批次里每 `5-8` 集至少检查一次动作变体：公开压迫、私密对峙、道具取证、关系试探、空间封锁、身体阻挡、信息误导等推进方式要交替使用

## 七、对白规则

对白是这部 skill 最容易写飘的一层。本章节是 `对白方法的单一真相源`；其他文件提到对白时优先链回这里，不重复展开。

### 7.1 通用对白规则（所有项目）

- 句子尽量短，单句默认 `<= 15` 字（中文）或 `<= 12` 个 word（英文）；超过就拆
- 信息密度高，但不要像解说词
- 角色之间要有声线差异
- 能展示就别解释；重要身份、目的、威胁和局势变化，优先靠动作、反应和场面传达
- 首次出场、权力场、冲突场里，避免自报家门式台词
- 更稳的顺序通常是：先动作，再反应，最后才给一句真正有分量的台词
- 只解释前情、重复已知信息、替观众总结局势的台词优先删改
- 一场戏里删掉"最说明的那一句"后还成立，就先删那一句
- 对白必须发生在动作里；不要让所有人干站着轮流说话

### 7.2 国内项目对白口语化

国内对白的 `底层逻辑`：让观众听到的是"现实里这个人在这种情境下真会说出来的话"，不是"剧本里的人物在念剧本"。

口语化硬要求：
- 用日常口语词，不用书面化或翻译腔；典型反例与替代：
  - `难道你不知道吗？` → `你装不知道？` / `你真没听说？`
  - `我无法接受这种事实` → `我不信。` / `这不可能。`
  - `请你立刻离开这里` → `你走。` / `滚。`
  - `他对我做出了过分的行为` → `他动我了。` / `他越界了。`
  - `这个事情非常严重` → `事大了。` / `出事了。`
- 用碎句、省略、抢话、噎话、停顿；不要把每句都说完整
  - `我刚刚——` `闭嘴。`
  - `你是不是……算了。`
- 允许语气词：`啊 / 吧 / 嘛 / 呗 / 呢 / 哎 / 嗯 / 哼 / 啧`；但每场不超过 `2-3` 处，否则像水戏
- 允许日常虚词：`就 / 才 / 都 / 还 / 嘛 / 倒是 / 反正`；用来拉口语腔
- 角色互称用日常称呼，不一上来就全名
  - 关系近：`哥 / 姐 / 叔 / 姨 / 老张 / 小李 / 哎你`
  - 职场：`张总 / 李工 / 师傅 / 老师 / 主任`
  - 关系冷或翻脸场：直呼姓名或姓 + 全名
- 禁用"剧本腔"和老气台词：
  - `你这个混蛋` → 改 `你他妈` / `你疯了` / `你脑子有病吧`（按角色和分级）
  - `我会让你付出代价` → 改 `这事没完。` / `你等着。` / `走着瞧。`
  - `你以为你是谁` → 改 `你算老几。` / `你哪位？`

声线分层（写之前先选档）：
- 年龄：少年/学生（碎、跳、网感词、不完整句）；职场青壮（短、准、克制语气词）；中老年（语速慢、爱铺垫、惯用旧词）
- 职业：白领（话术化、客套、术语前置）；技术蓝领（短、直、行话夹杂）；江湖/地下（多省略、动作词替代描述词）；体制（公文味、爱用 `按规定 / 按程序 / 这事我得请示`）
- 性别×身份只是参考，不是硬规则；优先看人物前史和当场目的
- 同一场里，主对手两人的句长、句式、用词锐度必须能听出差别；如果两个人的台词换名字也成立，就回改

网感词与方言使用边界：
- 网络流行语优先选 `半年以上仍在使用` 的稳定词；三个月内的热梗默认不用，避免播出时已过气
- 主角默认不靠网感词撑人设；网感词放在配角、龙套、群像
- 方言只允许 `轻量受控`：`整 / 咋 / 啥 / 搞 / 拎得清 / 怼` 这类已经全国通用的词允许常态使用；纯方言句（粤语、闽南语、东北重音）只在必要的地域感场使用，且必须保证非方言区观众也能秒懂
- 不出现需要字幕脚注才能看懂的方言；如果非用，必须配一句普通话回应翻译

潜台词与省略：
- 对峙场默认让人物 `话只说半句`：把"你不要再来找我"压成"你别再来"，把"我已经知道是你做的"压成"我都知道了"
- 给一两次"压住一句话不说"的瞬间（用 `△` 写动作沉默替代）；这种比一直说话更狠

#### 7.2.A 国内场景化对白样本库（按高频场景对照）

每段给 `反例（书面/翻译腔/解说）` vs `正例（口语化/场域感/有潜台词）` 对比；模板里的对白看着像反例就回改。

**直播间 / 切片号 / MCN 经纪场景：**
- 反例：女主：你为什么要切我的流？你这样做是不是太过分了？
- 正例：女主：你切我？△ 看后台数据。 你切我。
- 反例：经纪人：这个合约对你的事业发展非常不利，我建议你慎重考虑。
- 正例：经纪人：这合同你签了，号没了。

**医院 / 病危 / 医患场景：**
- 反例：医生：根据病情判断，我们建议立即进行手术治疗。
- 正例：医生：得签字。 现在签。
- 反例：家属：这怎么可能！为什么会突然变成这样？
- 正例：家属：人呢——△ 抓医生袖子。 你刚不还说稳定吗。

**小镇 / 体制 / 公文味场景：**
- 反例：科长：根据规定，这件事我需要向上级请示后才能给你答复。
- 正例：科长：这事，我得请示。你先回。
- 反例：村长：作为村干部，我有责任维护村里的稳定团结。
- 正例：村长：村里的事，村里办。你别搅。

**职场 / 大厂 / 互联网场景：**
- 反例：同事：你这个方案存在严重的逻辑漏洞，无法通过评审。
- 正例：同事：方案有问题。这块过不去。
- 反例：领导：我对你的工作表现感到非常不满意。
- 正例：领导：3.25。你自己想吧。

**校园 / 暗恋 / 师生场景：**
- 反例：女主：我从很久以前开始就一直喜欢你了。
- 正例：女主：我——△ 把书塞他怀里。 我喜欢你。早就。
- 反例：班主任：作为一名学生，你必须把全部精力放在学习上。
- 正例：班主任：高三了。别整这些。

**豪门 / 婆媳 / 家宴场景：**
- 反例：婆婆：你嫁到我们家就要遵守我们家的规矩。
- 正例：婆婆：△ 放筷子。 我们家，没这规矩。
- 反例：女主：我和您儿子的事情，希望您能尊重我们的选择。
- 正例：女主：妈，我和他的事——我自己说了算。

**末日 / 求生 / 物资场景：**
- 反例：男主：我们必须立刻离开这里，否则会有生命危险。
- 正例：男主：走。现在。
- 反例：女主：这种污染源对人体的危害是非常严重的。
- 正例：女主：碰它你就完了。

通用规则：
- 越紧张的场，句子越短；权力大的一方往往话最少
- 不要让两个人轮流"念完整段台词"；一定要有抢话、噎话、动作打断
- 体制 / 职场 / 公文场用 `这事 / 那块 / 流程上 / 规矩上 / 这块 / 上面`，不直接写"按规定 / 根据规章"

### 7.3 出海项目（TikTok / 美国英语首版）对白本地化

海外对白的 `底层逻辑`：英文必须是美国当代用户在 TikTok 短剧里"会一口气说出来"的口语，不是中式英语直译，也不是英文教材腔。

source of truth 锁定（与交付层口径单一一致）：
- `02-剧本/第XXX集.md` 场次执行稿里的英文对白是全 skill 海外对白的 source of truth；中文注释只补潜台词或情绪意图，不参与制作派生
- 所有交付层派生产物（`海外制作版分场表 / 制作理解稿 / 制作交付参数表`）只能从此 source 派生；改对白只能改 source，不能从派生表反向修改正文
- 详见 [14-制作交付整理.md](14-制作交付整理.md) 的 `海外对白 source-of-truth 锁口径`

格式硬要求：
- 海外线只让 `对白 / 旁白 / 屏幕字` 使用英文，策划层和动作行 `△` 仍是中文
- 不能出现 `英文角色名：中文对白` 混写
- 每句英文对白后必须紧跟中文注释，格式为 `English line.（中文注释。）`；中文注释默认是 `潜台词或情绪意图`，不是字面翻译
  - 例：`Get out.（滚——压住怒气，没喊。）` 优于 `Get out.（出去。）`
- 中文注释默认 `<= 15` 字；超过就说明对白本身没写到位
- 角色名首次出场在 `人物：` 行写成 `English Name（中文身份 / 剧情功能）`

口语化硬要求（contractions 是默认形态，不是可选）：
- 默认必须缩略：`I'm / you're / he's / she's / we're / they're / it's / that's / what's / there's / here's`
- 默认必须缩略：`don't / doesn't / didn't / won't / wouldn't / can't / couldn't / shouldn't / isn't / aren't / wasn't / weren't / haven't / hasn't / hadn't`
- 例外：人物 `刻意拉高语级` 表示压制、警告、法律语境时才用全形（`I am not asking again.` / `You will regret this.`）
- `going to` 默认写 `gonna`，`want to` 写 `wanna`，`got to` 写 `gotta`，`have to` 写 `gotta`/`hafta`（青少年角色更明显）
- 句尾用碎句、省略、追问；不要完整复合句堆叠

口语反例 → 替代（杜绝翻译腔）：
- `How dare you!` → `Are you serious?` / `You did not.` / `Excuse me?`
- `You are courting death.` → `You're done.` / `You're dead.` / `You really wanna do this?`
- `I will never forgive you.` → `We're done.` / `Don't talk to me.`
- `You have crossed the line.` → `That's it.` / `You crossed it.`
- `It is none of your business.` → `Mind your business.` / `Stay out of it.` / `Not your problem.`
- `You misunderstand me.` → `That's not what I meant.` / `Wait—no.`
- `Please leave immediately.` → `Get out.` / `Out. Now.`
- `What is the meaning of this?` → `What is this?` / `What's going on?`
- `I demand an explanation.` → `Explain. Now.` / `Start talking.`

声线分层（按 register 选档）：
- Teen / college（YA / campus）：碎句、`like / literally / kinda / you know / honestly / no way / shut up / what the hell`；多用 `gonna/wanna`；不完整句多
- Young adult（20s romance）：克制 + 调情；`look / listen / hey / I'm serious / don't`
- Professional（CEO / 律师 / 医生）：克制、术语夹杂、避免脏话；`We need to talk.` / `Let's be clear.`
- Antagonist / dark：低语速、长停顿、冷词；`Try me.` / `Watch yourself.` / `You sure about that?`
- Older / parental：完整句更多、惯用 `young lady / sweetheart / I told you / don't you dare`

文化锚点池（让台词长在美国语境里，不是抽象 globalized 英语）：
- 学制：`senior year / junior / freshman / prom / homecoming / SAT / ACT / GPA / valedictorian / detention / principal's office`
- 法律 / 财务：`restraining order / NDA / prenup / 401k / credit score / SSN / IRS / lawyer up / Miranda`
- 节日 / 季节：`Thanksgiving / Halloween / Black Friday / Memorial Day / Fourth of July / spring break`
- 居住 / 地理：`zip code / HOA / suburbs / uptown / the Hamptons / brownstone / trailer park`
- 餐饮 / 日常：`Starbucks run / drive-thru / takeout / leftovers / Uber / DoorDash`
- 校园 / 社交：`group chat / DM / posted it / went viral / cancel / ghosted / blocked`
- 不要把中国元素直接英译塞进对白（如 `face / give face / save face`）；要么彻底本地化，要么显式标注 cultural setup

感叹 / 咒骂分级（按 TikTok 内容政策预判）：
- PG / 多数项目：`damn / hell / shut up / oh my god / what the hell / freaking`
- 中等强度（dark romance / thriller）：`asshole / shit / piss off / bastard / screw you`
- 高强度（仅成人题材）：`f-bomb` 不要超过每集 `1` 次，且不要落在正面主角；TikTok 切片时优先用 `bleep` 标记
- 宗教感叹（`Jesus / Christ / God / oh my god`）默认允许，但避免连续多次；信仰题材项目要预判

#### 7.3.A 海外场景化对白样本库（含中文注释）

每段给 `直译反例` vs `本地化正例（contractions + 注释补潜台词）` 对比。注释只补潜台词，不补字面意思。

**Teen / college（YA / campus）：**
- 反例：Teen: I cannot believe what you just said to me.（我不敢相信你刚才说的话。）
- 正例：Teen: Are you for real right now?（你装的吧——其实是又气又难过。）
- 反例：Teen: This party is very entertaining. I am enjoying it.
- 正例：Teen: This party's actually fire.（认输了，本来不想来的。）

**Young adult（20s romance）：**
- 反例：Female: I have decided to leave you because you have hurt me deeply.
- 正例：Female: I'm done.（不想再吵第十遍了。）
- 反例：Male: I want you to come back to me, I miss you very much.
- 正例：Male: Come home.（不会再放她走第二次。）

**Professional（CEO / 律师 / 医生）：**
- 反例：CEO: I need to discuss an important business matter with you immediately.
- 正例：CEO: My office. Now.（已经压不住怒气。）
- 反例：Lawyer: You should not say anything more without consulting your attorney.
- 正例：Lawyer: Don't say another word.（这话再说就是把柄。）

**Antagonist / dark：**
- 反例：Antagonist: You will pay for what you have done to me.
- 正例：Antagonist: You're gonna regret that.（语速放慢，一字一顿。）
- 反例：Antagonist: I am giving you one last chance to surrender.
- 正例：Antagonist: Last chance.（已经不会再有第二次。）

**Older / parental：**
- 反例：Mother: I am very disappointed in your behavior, young lady.
- 正例：Mother: Sit down. We're having a talk.（不是商量，是审判。）
- 反例：Father: I will not allow you to make such a foolish decision.
- 正例：Father: Don't you dare.（再说一句就动手了。）

**Stepmother / family villain（dark family）：**
- 反例：Stepmother: I have always considered you part of my family.
- 正例：Stepmother: We're family, sweetheart.（甜得发腻——其实下一秒就要算计她。）

中文注释 vs 字面翻译对比（注释要补潜台词，不补字面意思）：
- `Get out.（出去。）` ← 字面翻译，删掉
- `Get out.（滚——压住怒气，没喊。）` ← 注释补"压住怒气"这层潜台词
- `I'm done.（我完了。）` ← 字面翻译，错（Done 在情侣场是分手，不是"完了"）
- `I'm done.（不想再吵第十遍了。）` ← 注释补"为什么"
- `Try me.（试试我。）` ← 字面翻译
- `Try me.（来啊，等你犯错很久了。）` ← 注释补威胁的真实意图

cliffhanger 英文句式池（避免重复）：
- `身份揭露`：`You knew. The whole time.` / `You're his sister.` / `So it was you.`
- `背叛`：`You did this.` / `It was you all along.` / `I trusted you.`
- `权力反转`：`You work for me now.` / `I own this.` / `Your move.`
- `禁忌触发`：`Don't.` `Don't what?` / `If you walk out that door...`
- `生死危机`：`Behind you.` / `She's not breathing.` / `He has a gun.`
- `decision point`：`Choose.` / `Him or me.` / `Now or never.`

文化禁区（直接套用国内桥段会翻车）：
- 男主"霸道占有"、抓手腕、强吻、语言贬损 → 美国语境里属于 abuse / harassment，需要重新设计为 `dark romance` 框架并显式信号化
- 家暴 + 复合 / 出轨 + 被原谅：在欧美语境里属于不可接受 trope，必须改成 `离开 + 报复 + 重塑` 的弧光
- 家长包办婚姻、长辈强行干预成年子女恋爱：必须改成 `cultural community pressure` 而不是直接命令式
- 公开打耳光、当众羞辱不是默认爽点；要先建立 stakes 再放
- 涉枪 / 涉毒 / 自残：必须按 `08-欧美市场指南.md` 合规章节降级处理

### 7.4 声线一致性自检（每场最少做一遍）

每场写完后过一遍：
- 把所有对白单独提出来，去掉 `角色名：`，能否凭句式判断是谁在说？如果不能，回改
- 同场两个对手的句长、词汇锐度、节奏是否真的不同？
- 这一场至少有一句"只属于这个角色 / 这段关系 / 这个场景"的台词；如果换到另一部剧也能用，回改
- 海外线再加一问：英文对白是否完全 contractions 化？是否出现 `do not / I am / cannot / will not` 这种全形？如果有，必须有刻意提级理由
- 海外线再加一问：中文注释是否在补"潜台词 / 情绪意图"，而不是字面翻译？

### 7.5 OS / VO / 屏幕字 / SFX 标记

- `OS / VO` 是按需工具，不是默认推进手段；同一场里能靠动作、反应、关系变化看懂的内容，不要再用 `OS / VO` 重复
- 但该标记时必须标记：广播、录音、系统播报、屏幕弹窗、文件文字、手机信息、内心独白、闪回旁白
- 海外项目 OS / VO 也是英文 + 中文注释，不出现 `VO：中文台词`

## 八、写完一批后必须回填

写完批次后，至少回填：
- `分集追踪`
- 当前批次的新增悬念
- 已兑现悬念
- 关系变化
- 角色长期状态变化
- 场景、道具、造型、能力状态变化

如果正文修订影响了稳定事实，先修源头文档，再修正文。
尤其要同步：
- `本集观看回报`
- `尾钩`
- `下一集必须承接`
这三项必须和 `分集追踪.md` 保持同口径。
回填顺序必须从正文事实出发：先读本集正文和 `本集回填`，再同步 `分集追踪.md`。不要从旧分集大纲或上一版追踪倒填回正文。
如果正文修了尾钩或观看回报，`分集追踪.md` 的同集条目必须同步改；如果下一集已经写出，还要同步改下一集 `开场承接` 和 `1-1` 第一动作。
如果正文新增、删除或显影了暗藏信息，`分集追踪.md` 必须同步记录 `暗藏信息 / 可发现线索 / 显影状态 / 回收位置`，避免后续只埋不收或重复埋同一类线索。

分工提醒：
- 如果你是在“试人物声音或关键场”，优先开 `试写样场`
- 如果你是在“决定这一批先写哪几集、这一批怎么避免同构”，优先写 `批次写作单`
- 如果你是在“判断当前这一集能不能正式落稿”，优先过 `单集达标检查`
- 如果你是在“记录已经写完的稳定事实、承接和债务”，优先回填 `分集追踪`

## 九、改稿优先级

- 先判断这是 `局部场次问题 / 单集问题 / 整批问题 / 上游结构问题`
- `局部场次问题`：优先局部返工
- `单集问题`：优先重写该集开头、结尾或关键场
- `整批问题`：先出 `改稿指令单-*.md`，再逐条修
- `上游结构问题`：先回改 `故事大纲 / 故事节拍表 / 分集大纲`，不要只修正文表面

局部返工最小动作：
- 如果只改了开场承接、尾钩、关键对峙或首次亮相，先改局部，再判断是否影响上游
- 如果只改了开场承接，优先把 `批次写作单`、`分集追踪` 和正文 `1-1` 的三处口径一起改齐，不要只改正文一处
- 如果改动已影响 `本集观看回报 / 尾钩 / 下一集必须承接 / 悬念债务`，同步回填 `分集追踪.md`
- 如果改动已影响该集主目标、主冲突、主回报或关键翻面，同步回改 `分集大纲.md`
- 局部返工后至少重过一遍当前集的 `单集达标检查`
- 如果局部返工已经影响到整批节奏或承接口径，再补一份 `复盘报告-*.md`

## 十、什么情况不能直接写下一批

出现以下情况之一，先复盘：
- 人设开始漂
- 尾钩开始重复
- 主线推进变慢
- 情绪类型反复重复
- 设定或规则开始打架
- AI 资产已经不稳定

## 十一、专业题材降理解成本

如果题材带法律、医疗、商战、金融、刑侦、媒体、公关等专业内容：
- 一场戏只保留一个核心专业信息点
- 专业词能删就删
- 专业词必须用时，立刻翻成可见后果
- 不要让角色像在上课

目标不是“显得专业”，而是“观众一秒知道谁赢谁输”。

## 十二、现实题材不要随手写离谱事实

现代现实项目默认按“可感夸张”处理：
- 可以强情绪
- 可以强压迫
- 不要随手写明显失真的生理、时间、程序和社会反应

如果一句设定让观众先觉得“这也太假了”，它通常会直接吞掉爽感。

## 十三、写前先做单集承载检查

写前先问：
- 这一集为什么必须单独存在，而不是并到前后集
- 这一集有没有足够的戏剧单位，而不是只有字数或信息量
- 有没有 `1` 个主回报
- 有没有 `1` 个最小状态变化
- 有没有 `1` 个具体尾钩
- 有没有写清 `即时目标 / 阻力来源 / 过程链 / 存在性理由`
- 如果只看场次正文，不看前置字段，这一集还是否撑得住目标集长
- 是否已经提前看出哪一场最容易偏薄，并想好要补的 `第二动作 / 第二阻力 / 第二反应 / 第二轮压迫`

再按目标集长追问场次数和厚度：
- 当前这集有没有符合规格的场次数；对 `60-90` 秒主流单集，默认优先 `3-4` 场

首集额外追问：
- 首集是否明确写了 `常规单集时长 / 首集建议时长 / 首集建制任务`
- 如果首集比常规集长，它多出来的内容是否都在建制：主角处境、核心痛点、题材承诺、第一次压迫、第一次反击、完整小回报、EP002 尾钩
- 首集是否至少有 `4` 个有效戏剧单位；如果只有 `2-3` 个单位，通常撑不起“首集加长”
- 首集第一场是否先给情绪命中或危机，而不是先解释世界观、合同、规则或人物身份
- 首集结尾是否让观众知道 EP002 具体要追什么，不是只写“下一层禁句 / 另有秘密”

如果只有 `2` 场，再追问：
- 单场内部有没有明显翻面
- 单场内部有没有动作升级或空间切换
- 单场内部能不能真正承载 `开头情绪冲击 / 前段局势变化 / 中后段强期待`

如果只有 `1` 场，再追问：
- 这场本身是不是完整的小序列，而不是一场长对话
- 这场里是否同时具备明确目标、正面对抗、至少一次翻面、动作或局势变化
- 这场结束后，人物关系、胜负或下一集追更问题是否已经被改写

如果这些答不清，不要靠加解释和加台词硬拉时长。

写完第一版正文后，再追问：
- 多数场次是不是只剩 `1-2` 条动作 + `1-2` 轮对白就直达结果
- 这一集是不是本来就只承担了一个功能动作，例如 `查证 / 进门 / 交接 / 送人到下一集`
- 是否出现了 `任务层厚、正文层薄`
- 最薄的一场如果去掉前置栏，正文还剩不剩完整戏剧过程
- 是否至少有一场不是“说清了”，而是真的“发生了、升级了、留下后果了”
- 是否每场都符合标准格式：标题、人物行、动作标记、台词标记、OS/VO/屏幕字/SFX 标记

优先处理：
- 并入前后集
- 回改 `分集大纲` 里的存在性理由
- 合并相邻集
- 增加阻力
- 增加代价
- 增加具体兴趣点
- 增加 `第二动作 / 第二阻力 / 第二反应 / 第二轮压迫`
- 把一场大对峙拆成前压后炸的两场，或补一个真正改变局势的中间场

- [assets/templates/08-单集达标检查模板.md](../assets/templates/08-单集达标检查模板.md)
