# 制作交付整理

这份规则用于 `制作交付层`，不替代主写作流程。本文同时覆盖国内和海外项目，按节区分通用 / 国内特化 / 海外特化。

职责边界：
- 是否进入 `production-enhanced`、哪些制作增强产物需要创建，以 [21-产物矩阵与创建时机.md](21-产物矩阵与创建时机.md) 和 [05-交付细则.md](05-交付细则.md) 为准
- 本文件只维护命中制作增强后的执行细则：资产命名、渠道特化、海外对白 source-of-truth、制作字段和冲突项处理
- 标准交付包不需要读取本文，除非涉及海外对白派生、资产命名或制作交底

## 通用：边界与读取条件

- 只在用户明确要把剧本交给制作方、客户或外包团队时读取
- 不把客户版本要求直接写回正文规范
- 不因为交付层存在，就跳过 `/剧本 -> /复盘`
- 国内项目和海外项目共用资产命名规范、资产卡格式、source-of-truth 原则；只在交付物清单和审核要求上分化

## 海外对白 source-of-truth 锁口径（全 skill 单一真相源）

海外项目对白的唯一真相源（source of truth）是 `02-剧本/第XXX集.md` 场次执行稿里的 `英文对白`。其它一切派生产物都从英文对白派生，不允许反向修改：

- `场次执行稿` 里的英文对白 = source of truth；中文注释只补潜台词或情绪意图，不参与制作
- `海外制作版分场表.md` 的台词列只填英文，**不填中文注释列**；默认不允许"中英双语并列"出现在交付包里
- `制作理解稿.md / 制作交付参数表.md` 等任何中文向制作沟通材料里出现的对白引用，都按"中文复述+英文原句脚注"组织，绝不替换英文 source
- 如果制作方明确要求双语稿，必须在 `制作交付参数表.md` 的"双语版本要求"字段里显式记录客户来源、版本号和签字日期；不允许默认开启
- 修改对白只能改 `02-剧本/第XXX集.md` 场次执行稿里的英文 source；改完后所有派生制作产物必须按 `21-产物矩阵与创建时机.md` 的对应规则重新生成
- 对白格式以 `12-单集写作协议.md 第七章 7.3` 为准（contractions、register 分层、文化锚点、中文注释规则）；本文不重复展开，只规定"派生口径不许污染 source"

## 资产命名统一规范（全 skill 适用，含国内项目）

为了让下游分镜引擎、视频生成器和配音引擎能直接消费资产卡，所有项目的资产 ID 命名按以下格式：

- 场景：`SCN_{LocationType}_{Place}_{Mood}_{v01}` —— 详见 `assets/templates/15-场景参考卡模板.md`
- 道具：`PROP_{Category}_{Name}_{State}_{v01}` —— 详见 `assets/templates/16-道具设定卡模板.md`
- 角色：`CHR_{Tier}_{NameEN}_{LookVariant}_{v01}` —— 详见 `assets/templates/14-角色资产摘要模板.md`

通用规则：
- 全大写下划线分隔，避免空格 / 中文 / 斜杠 / 句号 / 特殊字符
- 中文名 / 中文备注写在独立字段，不进 ID
- 集号写在 `首次出现集数` 等字段，不进 ID
- 同名实体用 `_A / _B` 区分（如 `Mansion_A / Mansion_B`）
- 状态升级 / 翻新 / 重新设计时递增版本号

## 一、处理范围（通用）

命中 `production-enhanced` 后，国内或海外项目按触发条件额外整理：

- 统一交接故事与设定的 `剧本圣经`
- 制作方快速理解故事的 `制作理解稿`
- 客户版本要求单独落盘的 `制作交付参数表`
- 面向制作方、且明确要求一角色一卡时的 `角色资产摘要`
- 面向制作方的 `场景参考卡`（需要场景资产交底时；资产重 / 长线 / 强设定项目默认需要）
- 面向制作方的 `道具设定卡`（需要道具资产交底时；关键道具影响剧情或制作连续性时默认需要）
- 便于交接的 `交付清单`

海外项目额外要：
- 面向海外制作方的 `海外制作版分场表`

国内项目额外要（按渠道）：
- 抖音原生项目：信息流投放素材切片说明
- 小程序付费短剧：付费节点对应集号清单（仅商业化模式已明确为小程序付费时）
- 长视频平台定制：会员独享段说明（仅商业化模式已明确为平台付费定制时）

这些内容来自制作和交付要求，不应该直接混进剧本正文。

## 二、通用交付要求

以下规则两市场都适用：
- `9:16` 竖屏交付
- 重要人物首次出场要有姓名牌或身份提示
- 字幕、姓名牌、首帧和封面都要考虑安全区
- 字幕位置、样式和行数要统一
- 音频要单独说明：对白语言、口音、环境音、BGM、版权禁区
- 首帧、封面、集数展示属于交付参数，不属于正文写法
- 自审清单要覆盖黑边、分辨率、字幕与口型对应、SRT 对应、逻辑衔接、音频一致性

## 三、国内项目特化交付要求

### 3.1 渠道特化

`抖音端原生 / 信息流投放`：
- 0-5 秒前 hook 要立刻可识别，不依赖完整集理解
- 信息流切片要按 15-30 秒短剪辑准备：每集准备 2-3 个切片建议点
- 标题需配合切片，避免剧透关键反转
- 切片导流到独立 App 时，标题留追更悬念

`小程序付费短剧`：
- 只适用于商业化模式已明确为小程序付费的项目；商业化后置项目不提前生成付费节点表
- 付费集对应集号必须在交付清单里明确标注
- 首付费卡集必须有 S 档强翻面证据（详见 [24-卡点设计.md](24-卡点设计.md)）
- 解锁阶梯按 3-5 集一档准备，每档卡点强度有对照
- 平台审核打回常见点：付费卡前情绪情绪兑现不足、付费卡内容不可逆性弱、付费卡集时长不达标

`长视频平台定制 / 会员独享`：
- 会员独享段在阶段大纲明确标注，对应集号要写清
- 长头部（首集 / 前 3 集）建制集要有完整人物建立 + 题材承诺
- 中后段每 5-8 集一次大事件作为留存点
- 平台定制需提交平台特化交付参数表（如分级 / 关键词避词）

### 3.2 国内常见审核打回点

- 涉未成年：校园暴力 / 未成年人恋爱 / 涉未成年人成人化情节
- 涉公职：警察 / 检察官 / 法官 / 军人形象不严肃化或被反转
- 涉医：虚假疗效 / 医患极端冲突 / 超纲医疗操作
- 涉法：私刑正义 / 教唆犯罪 / 把违法行为正当化
- 价值观红线：拜金 / 炫富 / 复仇至上 / 家暴美化 / PUA 美化
- 标点 / 字幕 / 屏幕字必须用中文标点
- 主流地名 / 真实事件 / 真实人物未授权使用

详见 [01-立项细则.md](01-立项细则.md) 合规与题材敏感线段；交付时再过一遍。

### 3.3 国内字幕与屏幕字规则

- 字幕：中文标点 / 简体 / 一行不超过 14 字 / 双行不超过两段
- 屏幕字（叠加 / 手机 / 弹窗 / 文件 / 系统等）：中文为主；外语短语必须配中文备注；交付前保留类别标注，便于制作区分叠加提示字、画内信息字和环境标签字
- 姓名牌：中文姓名 + 简短身份（"张医生 / 急诊外科"）

## 四、海外项目特化交付要求

### 4.1 海外规格

- 海外项目常见规格区间：`45-120` 秒/集、`50-100` 集、`60-130` 分钟总时长
- 海外常见推荐区间：`60-80` 集、`80-120` 分钟总时长
- 海外交付中的字幕、姓名牌、屏幕字、SRT 和制作版分场表里的最终制作语种示例默认统一使用英文标点

### 4.2 海外面孔与口音路由

以下规则只在 `海外写实 / AI真人 / 人形3D` 项目里适用：

- 英语市场优先欧美面孔
- 西语/葡语市场可切拉美面孔
- 部分英语市场可切亚洲面孔版本
- 口音路由要跟市场一起锁定，例如美式英语、拉美西语、巴西葡语

海外首版默认 `美国英语`。如果用户没有明确扩市场，不要主动展开到西语或葡语版本。

### 4.3 海外平台审核打回点

- TikTok：性 / 暴力 / 自残 / 仇恨言论分级；未成年涉性禁止；脏话分级（PG / mature / 18+）
- Apple App Store：dark romance 类要标注 mature；DV 美化禁止
- ReelShort / DramaBox：付费集开头要重述上集结尾，引导付费用户接续观看
- Google Play / 海外 App：未成年涉风险题材必须分级

## 四、冲突项处理

上游交付文档可能互相冲突，例如：

- 一份文档要求字幕全大写
- 另一份文档要求正常句式
- 一份文档要求姓名牌加职业头衔
- 另一份文档要求只保留名字

这种字段不要写成全 skill 的永久规则。正确做法是：

1. 在 `制作交付参数表.md` 中记录来源文档和版本
2. 把冲突项列成待确认字段
3. 在交付清单里提醒制作方按本次执行口径执行

默认口径：
- 如果没有客户特批版本，海外交付统一按“带英文标点”执行
- 最终交付包不保留模板占位语；没有客户特别限制的字段，要直接写成当前执行口径，而不是留 `待补充`

## 五、交付产物

### 1. `制作理解稿.md`

用途：
- 用纯中文把故事、情绪、角色和重点场景讲清楚
- 给内部制作沟通或 `15-30` 分钟对齐会使用

### 2. `制作交付参数表.md`

用途：
- 单独承接客户版本要求
- 记录字幕、安全区、首帧、封面、口音、音效、敏感边界等执行参数

### 3. `完整剧本总稿.md`

用途：
- 汇总通过复盘的分集剧本
- 作为最终阅读版和交底版正文

### 4. `剧本圣经.md`

用途：
- 在交付阶段用一份文件统一交接 `项目设定 / 故事梗概 / 故事大纲 / 故事节拍表 / 阶段大纲（若有） / 世界观设定 / 人物小传 / 场景参考卡 / 道具设定卡`
- 让海外制作方、平台或外包团队不必反查多个中间文件
- 对世界规则、角色关系和关键资产较重的项目尤其重要

规则：
- 世界观、角色、场景、道具信息优先来自最终版文件，不要把策划工作版原样塞进去
- `剧本圣经.md` 中的场景 / 道具章节是交付汇总副本；独立的 `03-交付/场景参考卡.md` 与 `03-交付/道具设定卡.md` 才是制作资产卡 source of truth
- 独立场景 / 道具卡更新后，必须重新生成 `剧本圣经.md`，不能手工只改其中一份造成口径漂移
- 如果海外项目需要快速对齐故事、世界和资产，优先先看这份总册，再看 `制作理解稿.md` 和 `制作交付参数表.md`

### 5. `海外制作版分场表.md`

用途：
- 当对方需要表格化分场稿时提供
- 默认用于海外项目

规则：
- 它是从已通过复盘的 `场次执行稿` 转出的制作表，不是重新创作的新剧本
- `台词` 列只写英文（即场次执行稿里的英文 source of truth），**不填中文注释列**，默认不允许双语并列
- 客户明确要求双语稿时，必须先在 `制作交付参数表.md` 写清来源、版本号和签字日期，再在分场表新增独立的中文备注列；不能在主台词列里中英文混排
- 修改分场表台词等于修改 source；如确有改动，必须先回到 `02-剧本/第XXX集.md` 改英文，再重生成分场表，不能从分场表反向改回正文

### 6. `角色资产摘要.md`

用途：
- 仅在制作方或平台明确要求一角色一卡时使用
- 从 `人物小传.md` 提炼制作方和平台常要的角色理解材料
- 把角色秘密、底线、弧光终点和形象连续性单独讲清楚

### 7. `场景参考卡.md`

用途：
- 补足常驻场景和关键高光场景的视觉锚点
- 给后续制作、安全区和复用资产一个更稳定的参照
- 作为下游分镜 / 视频生成可直接引用的独立资产文件

### 8. `道具设定卡.md`

用途：
- 补足关键道具的识别元素、剧情用途和连续性规则
- 尤其适合奇幻、悬疑、法律、家族秘密等依赖关键物件推进的项目
- 作为下游分镜 / 视频生成可直接引用的独立资产文件

## 六、和主流程的关系

- `/立项` 决定市场、题材、规格
- `/策划` 锁故事引擎、角色和分集大纲
- `/剧本` 写正文
- `/复盘` 决定能否进入下一批
- `/交付` 只做整理和转换，不重写剧情

## 七、如果输入是小说或长文本

如果用户给的是 `小说 / 网文 / 长篇英文故事`，先做这一条预处理：

1. 先整理纯中文故事理解稿
2. 必要时先做内容清洗，再继续
3. 再进入锁题、策划和结构

不要直接把长篇原文生硬改成最终交付剧本。
