# 剧本层细则

## 目标

- 基于已通过的 `分集大纲.md` 写分集剧本正文
- 生成时可以分批，但最终剧本文件统一进入 `02-剧本/`

## 输出

- `02-剧本/第001集.md`
- `02-剧本/第002集.md`
- `...`

主模板：
- [../assets/templates/05-场次执行稿-中文模板.md](../assets/templates/05-场次执行稿-中文模板.md)
- [../assets/templates/06-场次执行稿-海外模板.md](../assets/templates/06-场次执行稿-海外模板.md)

## 规则

- 正文不能先于 `分集大纲.md`
- 如果 `分集大纲.md` 还只是“一句话集纲”，不直接进入正文
- 如果 `分集大纲.md` 还没有完成全量整合，或尚未通过独立 `大纲质检`，不进入正文
- 分批是生成方式，不是最终交付结构
- 不把最终剧本放在批次目录里
- 写正文时按当前集需要读取分项策划文件，不读取或维护一个提前汇总的 `剧本圣经.md`
- 正文阶段的主要质量判断要在写的时候完成；`本集创作简卡 / 场次前置信息 / 批次写作单 / 单集达标检查` 都是写作控制，不是等 `/复盘` 才补的装饰
- 正式 `第XXX集.md` 不承载完整检查单。它只保留轻量 `本集创作简卡`、`场次正文`、`本集回填` 三层；字段越多越容易把正文写成摘要，复杂判断应放回 `批次写作单 / 单集达标检查 / 复盘报告`
- `本集创作简卡` 每项尽量一行，只写会改变正文写法的锚点；如果一项需要长篇解释，优先回上游大纲或批次写作单处理，不在正式剧本里展开
- 当前上下文里的自检不算批次放行；每批是否允许继续，默认由隔离上下文的独立 `剧本质检` 决定。用户明确要求 `clean-context` 或批次质检时，当前会话应优先代为发起只读取正式源文件和必要规则的隔离复检任务；只有用户额外要求并行/代理时才把方式记为 `subagent`
- 正式正文必须使用标准 `场次执行稿` 格式，不能写成小说段落：场次标题用 `### 1-1 日内 地点`，每场必须有 `人物：`，动作行用 `△`，对白用 `角色名：`，心理/画外/屏幕/声音按需标 `OS：/ VO：/ 屏幕字（类别）：/ SFX：`

## 吸引力硬门槛

剧本质量不以格式完整为准，而以观众是否愿意继续看为准。

单集必须同时回答：
- 这一集开头凭什么抓住观众
- 这一集过程中发生了什么可见冲突，而不是只解释信息
- 这一集给了观众什么情绪回报、爽点回报、关系回报或悬念回报
- 这一集有什么记忆点，能让观众在下一集前还记得
- 这一集结尾具体改变了下一集的目标、危险、关系或信息差中的哪一项

写作前先做最小构造：
- 走 [12-单集写作协议.md 第二·补章 大纲→场次执行稿转化五步法](12-单集写作协议.md)：观众口径压一句 → 四链拆解 → 场次分配 → 每场动作骨架 → 首场前 4 行硬规则；分集大纲到场次正文的转化必须沿着这五步走，不允许直接从一句话大纲跳到台词
- 用 `情节承接 / 冲突升级 / 价值闭环 / 下集勾连` 确认这一集不是功能桥段
- 用 `钩子类型` 确认结尾是具体变化，不是抽象断句
- 用 `信息差结构` 确认悬念能让观众担心、追问或期待爆发
- 用 `情绪密度 / 信息密度 / 情节密度` 确认这一集能停留、能理解、能追更

正文厚度硬阈值（与前置字段对比）：
- 单场前置字段总行数（承接点 / 场景目标 / 场景状态差量 / 视觉记忆点 / 动作升级点 / 第二阻力 / 场内翻面 / 可见后果）必须 ≤ 该场正文行数的 `1/3`
- 例：前置字段 8 行，正文必须 ≥ 24 行；触线判正文偏薄
- 触线时不允许通过删前置字段绕过——必须把前置栏里写过的动作 / 阻力 / 反应 / 后果改写成可见正文

以下情况默认不合格，先返工再继续：
- 开场先讲背景，第一场没有压力、承接或动作
- 单集主要内容只是查资料、接电话、解释设定、确认线索
- 主角没有做选择，对手没有加压，局势没有被改写
- 结尾只抛一个抽象名词、资料名或“另有隐情”
- 本集看完没有可复述的高光画面、关系翻面、爽点或痛点

## 精彩度与读感门槛

结构完整不等于好看。代表样稿必须额外过读感检查：
- 这集有没有一个观众能复述的 `具体事件`，而不是“主角又拿一个物件逼对手回应”
- 主角有没有被真实刺痛、真实想要、真实选择；不能只做流程动作
- 对手压迫是否符合题材原生场域；直播题材应是切流、限流、封号、热搜、合同，校园题材应是储物柜、缺勤记录、社团、奖学金、同伴排挤，末日题材应是污染、物资票、安全区资格，不能所有题材都默认写成保安、登记册、门禁
- 每场至少有一句只属于这个角色、这个关系、这个场景的台词；如果换到另一部剧也能用，优先重写
- 结尾钩子要让观众问一个具体问题，例如“谁删了缺勤记录”“为什么第二枚戒指更早签名”“哪只萌宠感染了”，不要只写“下一层禁句”

正文禁用的工作稿语言：
- `第XXX个关键口径`
- `第XXX个拦截口径 / 反咬证词 / 尾钩问题`
- `推到台前`
- `下一层禁句`
- `入口阻隔 / 公开反应 / 二次压力` 等任务标签
- `主角从被挡在外变成能逼对手回应` 这类抽象状态句

如果这些词出现在正式 `第XXX集.md` 的场次正文里，说明写作者在复述批次设计，而不是写剧本。先删掉任务标签，再把它改成具体人物动作、具体压迫、具体反应和具体后果。

## 内部工作稿

开始正文批次前或正文过程中，按需准备：
- `90-内部工作稿/试写样场-*.md`
- `90-内部工作稿/分集追踪.md`
- `90-内部工作稿/单集达标检查-EP001-EP005.md`
- `90-内部工作稿/批次写作单-EP001-EP005.md`

但这几份都服务写作，不替代正文。

最小启用规则：
- 默认先用 `分集大纲.md` 直接写代表样稿
- 只有在当前批次还说不清承接、样稿、旧债回收、新债预算或关键集节拍时，再补 `批次写作单`
- 只有在人物声线、首次亮相、核心对峙或高光翻盘还拿不准时，再补 `试写样场`
- 只有在某一集准备正式落稿或返工后重新过关时，再补 `单集达标检查`
- `分集追踪.md` 不属于“正文前可有可无”的工作稿；它在首批正文前初始化，正文开始后持续累计
- 如果正式分集稿写出来像检查单、说明书或大纲复述，先删掉多余前置字段和方法说明，再重写场次正文

四类内部工作稿的默认分工（按写作时间先后串联，不互相替代）：
- `试写样场`：写之前——人物声音、关系张力、关键场拿不准时先试
- `批次写作单`：写当批之前——锁先写哪 `1-2` 集代表样稿、旧债回收、新债预算、关键集 `6` 节拍
- `单集达标检查`：单集写完之后——判断"能不能正式落稿"
- `改稿指令单`：单集 / 批次判分不过之后——把 ≥ 3 个具体改动收束成可执行修改清单
- `分集追踪`：贯穿正文阶段——累计记录稳定事实、悬念债务、关系变化、资产变化和承接口径

详细决策树见 [22-写作方法索引.md 第三章 四工作稿职责决策树](22-写作方法索引.md)。

不要混用：
- 不要把 `试写样场` 写成正式分集正文
- 不要把 `批次写作单` 写成第二份 `分集大纲`
- 不要把 `单集达标检查` 写成新的剧情设计稿
- 不要把 `改稿指令单` 写成第二份 `单集达标检查`；问题数 < 3 或单点判断清晰时，直接在 `单集达标检查` 里写改法
- 不要把 `分集追踪` 写成下一批剧情计划书

建议启用 `批次写作单-EPxxx-EPxxx.md` 的情况：
- 双时间线
- 多主线或多关系线
- 专业题材
- 长线项目
- 任何一集仅靠 `分集大纲.md` 仍然说不清主要动作时
- 即便是短项目，只要 `T10 分集大纲` 还不足以直接写场次，也先补这一层
- 所有项目默认按批次写作，不直接整季落正文
- 长线、海外项目或精品要求项目按更严格口径执行
- `分集追踪.md` 建议在 `/策划` 通过后、首批正文前先初始化，不要等写完几集才补
- 这层不是第二份 `分集大纲`，只服务当前批次的代表样稿、承接口径、旧债回收、新债预算和关键集 `6` 节拍拆解

关键集补充规则：
- 首个强卡点集、重大揭露集、大反杀集、关系确认或决裂集、结局集，先拆 `6` 节拍再写正文
- `6` 节拍默认写进对应批次的 `批次写作单-EPxxx-EPxxx.md`，或写进当前集写作备注
- 不再单独维护一份 `分集节拍.md`
- 如果关键集必须靠节拍拆解才能写清，说明它还不适合直接进入正文

试写样场补充规则：
- 如果还没正式写分集正文，就已经怀疑主角声线、核心关系张力、首次亮相高光、公开场调度或海外对白是否成立，先试写 `1-3` 个样场
- 优先试写：`主角首次亮相场 / 核心对峙场 / 高光翻盘场`
- 试写样场重点看：人物声线、动作是否立得住、关系戏是否真的有拉扯、场面调度是否可执行
- 试写样场不过关时，优先回改 `人物小传 / 故事大纲 / 故事节拍表 / 分集大纲`，不要直接往下铺整批正文
- 样场是找声音和找场的工作稿，不替代正式剧本集数

批次正文方法：
- 所有项目每批先写 `1-2` 集代表样稿
- 代表样稿至少覆盖：`1` 集强卡点或重大翻面集，`1` 集常规推进集
- 先检查代表样稿的声线、承接、动作具体度和场面调度，再铺满本批
- 如果代表样稿已经出现模板化、承接不落地或场次骨架重复，不继续铺本批
- 长线、海外项目或精品要求项目，对代表样稿的要求更严格，且默认每批都做
- 每一批剧本写完后，先更新 `项目状态.json` 的批次进度，再做独立 `剧本质检`；未通过前不进入下一批
- 每批都先写清 `计划回收的旧债务`、`新增悬念预算` 和 `债务平衡说明`
- 连续 `2` 集以上如果都只新增悬念、不推进旧债务，先回改 `批次写作单-EPxxx-EPxxx.md`，再继续正文
- 如果问题集中在开场、尾钩、关键对峙、单个关系翻面或单集局部，默认先做 `局部返工`，不要一上来整批重写
- `局部返工` 优先级通常是：`单场 -> 单集开头/结尾 -> 单集 -> 整批`

正文落稿附加要求：
- 单集默认按 `情绪冲击 -> 局势变化 -> 强期待 -> 结尾钩子` 这条顺序组织节奏
- 单集正文不是把 `分集大纲` 改写成对白，而是把这一集拆成若干 `戏剧单位`
- 对 `60-90` 秒主流单集，正文通常至少要有 `5-7` 个剧情拍点或 `3-5` 个戏剧单位；更短集长可以更少，但不能只剩一条线索说明
- 每个戏剧单位优先至少回答：`谁想要什么 / 谁或什么在拦 / 发生了什么动作 / 局势或关系怎么变了 / 留下了什么可见后果`
- `本集创作简卡` 默认补齐 `本集戏剧单位 / 本集情绪落点 / 本集信息投放顺序 / 本集中段错误放松或小释放 / 本集可见后果 / 本集预计有效正文行数`，不要只写摘要型字段
- 每场正文默认只保留少量前置锚点：`承接点 / 场景目标 / 场景状态差量 / 视觉记忆点 / 动作升级点 / 第二阻力或二次压迫 / 场内翻面 / 可见后果`。这些锚点服务正文，不替代正文
- 如果写完后场次前置锚点比正文更有内容，说明模板已经压过创作；优先把锚点里的动作、反应和后果写进场次正文
- `正文层厚度` 只看场次正文里实际发生了多少，不看前置字段写得多完整；前置信息是写作护栏，不替代戏剧过程
- 单场正文落稿时，常规集默认至少 `7-10` 行有效正文，短海外集默认 `6-8` 行；若正文动作对白总行数不明显多于前置锚点，视为没有真正写开
- 全集有效正文行数默认门槛：`45-75` 秒不少于 `22` 行，`60-90` 秒不少于 `28` 行，`90-120` 秒不少于 `34` 行；首集 `75-120` 秒不少于 `34` 行，`90-150` 秒不少于 `40` 行。低于门槛必须先回改分集大纲或明确说明为什么少场仍能成立
- 场次正文不能只写 `亮出物件 -> 对手否认 -> 旁观者反应` 这一种骨架；同一批里连续出现相同动作句式，先回改代表样稿
- 如果上游没有先回答“这集为什么必须单独存在”，正文阶段很容易把这集写成功能桥段；这种情况优先回改 `分集大纲`，不要只在正文里补句子
- 不把秒数写成死规则，但对 `60-90` 秒主流集长，开头几秒应先打情绪，前段应尽快给出变化，中后段应形成明确期待
- 先判断这一集有没有足够的戏剧单位，再判断拆成几场最合适；不要把“场次数达标”误当成“内容已经成立”
- 对 `60-90` 秒主流单集，默认优先按 `3-4` 场组织；`2` 场只适用于强对峙、强动作或单场内部能完成明显翻面和空间切换的集数
- 极少数情况下，`1` 场也可以成立，但前提是这一场本身就是完整的小序列，内部已有明确目标、冲突、翻面、动作变化、情绪升级和结尾钩子
- 如果多数场次正文只有 `1-2` 条动作 + `1-2` 轮对白就直达结果，这集通常不是“时长问题”，而是单集承载力不足
- 常见失败是 `简卡层厚、正文层薄`：前置栏很满，正文却只剩结果摘要。遇到这种情况，优先删减解释性前置栏，补 `第二动作 / 第二阻力 / 第二反应 / 第二轮压迫`，必要时回改上游把这集并入前后集
- `强期待` 优先写清目标、代价和压力；不要只写“事情很危险”
- `1` 分钟左右的单集优先保证至少 `3` 个明确爆点，不要整集只有一个模糊悬念
- `本集主回报` 尽量落成可见后果，例如公开失控、关系翻页、物证易手、位置被夺、规则被改；不要只停在“又确认了一条信息”
- 情绪、情节、信息密度不要都压在结尾一段台词里；优先让前中后段分别承担不同戏剧单位和不同类型回报
- 拉扯感不是单纯拖剧情或一直受苦；关键拉扯段要看有没有 `压低 -> 短松 -> 再压`
- 如果本批主打复仇、翻盘、关系反杀或逃生，正文不能只写成“直线受压到底”或“直线打脸到底”
- `结尾钩子` 优先写成会改写局势的变化，而不是只补一条信息或一个抽象资料名
- 中段不只靠集尾硬卡；场次内部也要有能续住观众的小钩子
- 悬念优先用 `提前预感 + 延迟兑现` 吊住观众，不要把突然出现的惊吓误当成完整悬念
- 如果本集承担主角首次亮相，第一次露面必须先立分量、身份感、反差感或高光感，不能只是“主角走进场”
- `开场承接` 不能只停留在 `本集创作简卡`，必须在第一场第一句动作或第一句对白里显式落地
- 上一集 `下一集必须承接`、下一集 `开场承接`、`分集追踪.md` 的承接描述尽量使用同一核心名词短语，避免三处各说一套
- `分集追踪.md` 里的 `尾钩` 和 `下一集必须承接` 尽量直接复用正文中的完整短句或核心半句，不要回填成过度抽象的缩写名词
- 从 `EP002` 开始，优先让上一集尾钩里的 `人物 / 道具 / 地点 / 动作` 至少有一项在本集 `1-1` 立即出现；只写“开始查资料 / 进入主线 / 去下一个地点”通常不算承接成立
- 如果 `批次写作单-EPxxx-EPxxx.md` 里还写不出 `上一集尾钩核心短语` 和 `开场第一动作或第一句对白`，不要直接进入正文
- 默认每场至少给一个最小 `发声单位`，可以是对白、OS、VO、广播、铃声提示后的反应等；纯动作静默场只在极少数高光段使用，并在单集检查里说明理由
- 场次不能只报 `时间 / 地点`；优先补 `1-2` 个和剧情、人物状态或权力关系强相关的环境或光影细节
- 每场尽量至少留一个 `视觉记忆点`，让场次不只剩功能对白
- 不要直接写抽象结论，如“她很压抑”“他很有威压”“气氛很危险”；优先把这些判断改写成可见细节、动作或他人反应
- 对白尽量发生在动作里，而不是所有人站着把话说完；重要冲突场优先让动作、道具、走位和现场变化先发生
- 对白的 `口语化 / 本地化` 是硬要求，不是风格偏好；完整方法库见 [12-单集写作协议.md 第七章](12-单集写作协议.md)，国内补充见 [07-抖音国内市场指南.md](07-抖音国内市场指南.md)，海外补充见 [08-欧美市场指南.md](08-欧美市场指南.md)
- 单句默认 `<= 15` 字（中文）或 `<= 12` word（英文）；超过就拆，不靠长句撑信息量
- 国内项目避免书面化、剧本腔、翻译腔；翻脸场之外不全名互称；网感词只用稳定词且不撑主角
- 海外项目英文对白默认完全 contractions 化（`I'm / don't / won't / gonna / wanna`），中文注释补潜台词而不是字面翻译
- 同场对手两人的句长 / 句式 / 词汇锐度必须能听出差别；把对白去掉 `角色名：` 后仍能凭句式分辨是谁在说
- 每场尽量至少完成一件事：`权力变化 / 关系变化 / 信息翻面 / 情绪释放 / 局势升级`
- 如果一场戏说完后，人物位置、关系强弱、危险程度和下一步行动都没变，这场通常还需要重写
- 关键节点允许少量景别或镜头提示，但只放在 `开场钩子 / 爽点瞬间 / 情绪爆发 / 悬念揭晓` 这类真正值钱的节点
- 不把整集正文写成分镜清单；镜头提示是点睛，不替代场次执行稿
- 如果本集承担奇观或大场面，正文优先把它写成 `触发 / 扩张 / 主角选择 / 局势翻面 / 余波` 的可生产序列，而不是一句“天地失色 / 全场震惊 / 万剑齐发”
- 奇观场先回答“这场具体改变了什么”，再回答“它看起来多大”；如果局势、关系、权力或公众后果都没变，这通常还不是成立的大场面
- 玄幻、末日、神战、宗门大会、公开审判、法庭、公祭、拍卖会、发布会都可以承担奇观场，但都必须先在上游写清叙事职责和核心视觉符号
- 如果一集主要由 `接电话 / 看文件 / 独处沉默 / 走进新地点` 组成，却没有至少 `2` 个以上表演节拍、反应回合或冲突转折，这集通常还撑不起目标时长
- 如果一场的前置信息明显比正文还厚，说明这场还没真正写开；优先补可见动作、反应回合和状态差量，而不是继续补说明字段
- 不能把一整批剧本写成同样的两场结构和同样的动作句式
- 不能把一整批剧本写成连续“只抛新债、不回收旧债”的悬念堆叠
- 能展示就别解释；人物身份、目的、威胁和权力关系，优先用动作、道具、他人反应和现场气氛传达
- `OS / VO` 是按需工具，不是默认推进手段；它们只在心理活动确实无法外化、或必须做开场定调 / 闪回衔接 / 录音广播转述时使用
- 如果 `OS / VO` 说清了原本能靠动作、反应、关系变化看懂的内容，优先删掉 `OS / VO`，把信息还给场面
- 但该标记时必须标记：广播、录音、系统播报、屏幕弹窗、文件文字、手机信息、内心独白、闪回旁白不能混写成普通动作或普通人物对白
- 角色首次出场、权力场、冲突场里，避免自报家门式台词；不要一上来就把“我是谁 / 我来干什么 / 你最好知道什么”全说破
- 对白更稳的顺序通常是：先动作，再反应，最后才给一句真正有分量的台词；不要让台词抢动作的活
- 如果一场戏删掉最说明的那句后仍然成立，优先删掉或改短那句
- 只解释前情、重复已知信息、替观众总结局势、说完后局势没有变化的台词，优先删、并、改
- 场次标题至少写清 `场景序号 / 时间 / 内外景 / 地点`
- `人物` 一栏必须和本集主要动作匹配，不能因为某人被提到，就默认他已经在场
- 角色首次进入正文时，默认在 `人物：` 行补身份说明
- 如果首次出场的是聚合角色或群像角色，例如 `学生志愿者若干 / donor 若干 / 媒体与家属成员（群像）`，也要在 `人物：` 行补功能身份说明，不要只写“若干”
- 海外项目里首次进入正文的角色，`人物：` 行默认写成 `中文身份 + English name` 的括注形式，不要只写人名
- 动作行默认用 `△`，兼容外部稿式时也允许使用 `A：`
- `OS：` / `VO：` / `【闪回】` / `【闪回结束】` 允许作为兼容标记，但不改变 `场次执行稿` 主体结构
- `屏幕字：` / `SFX：` 也是允许的制作标记；出现画面文字或关键声音时优先使用，不要临时造 `机械音：/ 黑伞灯浮字：/ 系统提示：` 这类不统一标记。画面文字推荐写成 `屏幕字（叠加/手机/文件/系统/姓名牌）：`，便于区分叠加提示字、画内信息字和环境标签字。
- 动作描述必须跟场景和当集回报绑定，不能连续多集复用同一套空泛推进句
- 连续多集不能反复复用固定开场句、固定施压句、固定反应句和固定收尾句
- 不要把“节奏快”写成连续吵架或连续大喊；如果没有升级、选择和后果，仍然是慢
- 长线项目每批都要检查是否出现重复打脸、重复证据、重复威胁方式或重复暧昧场面
- 尽量让重要推进发生在常驻角色、常驻场景和核心道具上，避免每逢关键集都新增高成本资产
- 避免为了“丰富”而频繁换装、换景、换状态；除非这就是剧情翻页本身
- 高难动作、多人混战、复杂机械变形、强透视追逐等内容要谨慎使用，先判断是否真的值这个资产成本
- 重要道具如果承担翻页，正文里要给出足够明确的特写价值和交互方式
- `本集创作简卡 / 本集回填 / 批次写作单 / 单集达标检查 / 复盘报告` 都属于过程稿，不进入最终 `完整剧本总稿.md`

## 分批建议

推荐：
- 首批 `3-5` 集
- 稳定后每批 `5` 集左右
- 所有项目每批后都先做 `/复盘`
- 长线项目每批通常 `5-10` 集，按代表样稿表现和复盘结果决定

不建议：
- 一次生成整季正文
- 任何项目在没有中间复盘的情况下直接写完整季

## 过关条件

- 每集都能对上大纲
- 每集都有主冲突和主回报
- 每集内容量足以支撑目标时长
- 角色声音不漂
- 结尾钩子不空
- 一批内部不能明显同构化
- 出现 `template-repeated / continuity-gap / debt-overload` 等中高风险问题时，先回改本批，不继续下一批

## 改稿方法

优先顺序：
- 先判断问题是 `上游结构问题`、`单集问题` 还是 `局部场次问题`
- 如果只是局部不成立，优先做 `局部返工`
- 只有当问题已经影响整批逻辑或上游结构时，才扩大到整批或上游重写

常见的局部返工对象：
- `开场承接` 没落到第一场
- `主角首次亮相` 不够立
- `关键对峙场` 只有对白，没有动作和权力变化
- `结尾钩子` 抽象、无后果、无具体拉力
- `关系翻面` 发生了，但情绪和后果没有真正落地

局部返工后的最小闭环：
- 如果返工改动了 `开场承接 / 主回报 / 尾钩 / 下一集必须承接 / 关键翻面 / 长期悬念债务`，先同步当前批次对应的 `批次写作单-EPxxx-EPxxx.md` 与 `分集追踪.md`
- 如果返工已经改变该集的主目标、主要冲突、主要回报或关键翻面，同步回改 `分集大纲.md`
- 局部返工后，至少重过一次对应集的 `单集达标检查`
- 如果局部返工已经影响到整批节奏、承接口径或悬念债务平衡，再补做一次 `/复盘`
