# 策划层细则

## 目标

- 把题材拆成会在 `/交付` 阶段汇总进 `剧本圣经.md` 的分项策划文件（策划阶段不生成 `剧本圣经.md`）
- 先让故事引擎、人物关系、世界规则、连载能力和 AI 制作口径成立
- 先锁 `分集大纲.md`，再进入正文
- 每份策划文件都回答：为什么成立、为什么能连载、为什么适合 AI 竖屏呈现、为什么现在能进入下一层

详细创作方法按需读：
- [22-写作方法索引.md](22-写作方法索引.md)
- [11-密度设计.md](11-密度设计.md)
- [13-质量检查表.md](13-质量检查表.md)

快速定位：
- 不清楚当前 T 节点怎么推进：读"流程"
- 卡在世界观 / 角色 / 场景 / 道具关系：读"世界观、角色、场景、道具的设计关系"
- 要落某个正式文件：读"文件分工"里对应文件小节
- 出海或跨市场改编：读"海外项目规则"
- 判断能不能进正文：读"过关条件"

正式策划文件：
- `项目设定.md`（含核心卖点与项目定位）
- `世界观设定.md`
- `人物小传.md`
- `场景参考卡.md / 道具设定卡.md`（条件）

- `故事节拍表.md`
- `阶段大纲.md`（命中产物矩阵时）

条件产物：
- `场景参考卡.md / 道具设定卡.md`：仅资产重、长线、强设定项目在策划层提前产出工作版

落盘规则：
- [../assets/templates/03-策划模板.md](../assets/templates/03-策划模板.md) 只作为轻量策划工作底稿
- 正式落盘时拆成独立文件，不把底稿注释、检查语句和方法说明复制进正式策划文件
- `/策划` 阶段不生成 `剧本圣经.md`，避免提前形成上下文过大的重复汇总；`剧本圣经.md` 只在 `/交付` 阶段汇总正式源文件

## 流程

策划阶段不要一次性填满模板。按编号任务包推进；适合一起创作的内容放在同一任务包，但最终仍拆成独立正式文件，便于质检和交付。

剧作主链仍按 `创意 -> 梗概 -> 大纲/情节点 -> 剧本初稿` 推进。`T04-T07` 是创意阶段的最小可写性校准，目的是确认这个创意能否写成梗概、拆成情节点并进入初稿；不要把它执行成大段设定百科，也不要因为场景/道具卡没扩写就阻塞 `T08`。

| 编号 | 任务包 | 主要产物 | 先回答的问题 | 不过关时回到 |
| --- | --- | --- | --- | --- |
| T04 | 项目定位与核心卖点 | `项目设定.md` | 观众为什么点开，为什么追，为什么适合 AI 竖屏呈现 | 立项 / 项目设定 |
| T05 | 世界观与规则 | `世界观设定.md` | 故事世界怎样压迫主角，规则和代价是什么 | 项目设定 / 世界观 |
| T06 | 角色与关系 | `人物小传.md` | 谁想要什么，谁压迫谁，关系如何拉扯 | 项目设定 / 世界观 / 人物 |
| T07 | 场景与道具 | `项目设定.md` 的 `资产与制作` section（必出）；条件卡 | 常驻场景、核心道具、视觉符号能否支撑正文 | 世界观 / 人物 / 资产 |
| T08 | 梗概与故事大纲 | `故事梗概.md / 故事大纲.md` | 这部戏是什么，以及全剧如何推进、升级、失控和收束 | 项目设定 / 人物 / 世界观 |
| T09 | 节拍与阶段大纲 | `故事节拍表.md / 阶段大纲.md` | 哪些节点改变局势；每个阶段/批次负责什么回报和变奏 | 故事大纲 / 节拍 |
| T10 | 分集大纲 | `分集大纲.md` | 每集为什么必须存在，怎么进入正文 | 阶段大纲 / 节拍 |

合并任务规则：
- `故事梗概` 和 `故事大纲` 通常合为 `T08`：同一轮先用梗概定戏，再用大纲定推进，最后分别落成两个文件
- `故事节拍表` 和 `阶段大纲` 通常合为 `T09`：同一轮定关键翻面和阶段回报；`30` 集及以上必须输出阶段大纲，短项目可记录为跳过
- `场景` 和 `道具` 通常合为 `T07`：轻量项目写入 `项目设定.md / 世界观设定.md` 的资产段，资产重项目再额外出卡
- 合并任务不是合并文件；正式文件仍按交付需要拆开

## 世界观、角色、场景、道具的设计关系

结论：创作上同轮设计，产物上分别落盘，结构稳定后再二次校准。

执行方式：
- `T05-T07` 是故事基底包，要在 `T08` 之前形成最小可用版本
- 世界观不单独做百科；只保留会压迫人物、改变行动、产生代价、制造冲突的规则
- 角色不能脱离世界规则；人物小传要写清欲望、开局受压、反击方式、关系拉扯和被规则限制的位置
- 场景/道具不能完全等大纲后再设计；先锁常驻空间、核心道具、视觉符号和可复用资产，让大纲有可见行动支点
- `T08-T09` 完成后回看 `T05-T07`，只补会影响阶段推进、分集回报、高潮场面和制作执行的内容
- 资产卡不是默认百科；只有资产重、强设定、长线复用或制作交底需要时，才把场景/道具从锚点扩成独立卡

### `T05-T07` 同轮迭代的执行步骤

`同轮迭代` 不是"同时写三份文件"，也不是"按顺序填三张表"。具体动作如下，五步走完才允许进入 `T08`：

1. `先做人物压力链`（T06 优先动作）：写主角的欲望→开局受压→受压来源→反击入口→反击方式；不写其它角色的小传细节，只锁压力链最小骨架
2. `再锁世界规则的对应压迫`（T05 配套动作）：从压力链里反向追问"这个'受压来源'在世界规则里叫什么、有什么名字、付出什么代价、谁从规则里获益"——世界规则只保留能直接压回主角的那几条
3. `再选视觉支点（场景/道具）`（T07 配套动作）：从前两步里挑 `2-3` 个最关键的"可被压迫的瞬间"——给每个瞬间分配一个常驻场景或核心道具作为视觉支点；其余只列名不展开
4. `三链对照`：把人物压力链、世界规则、场景/道具锚点三张纸放在一起，逐项检查"压力链上的每一环，是否在世界规则里有名字、是否在某个场景/道具上有可见落点"——任何一环找不到对应，回到第 2 步或第 3 步补
5. `分别落盘`：通过对照后，分别落成 `人物小传.md / 世界观设定.md / 项目设定.md（资产与制作 section）`；资产重项目再决定是否单独出 `场景参考卡 / 道具设定卡`

退化警示（出现以下情况说明同轮迭代退化为顺序填表，回到第 1 步重做）：
- 世界观写完后，角色小传里仍找不到"被这个世界规则压迫的具体动作"
- 角色压力链写完后，世界规则里没有这个"压迫来源"的对应名字
- 场景/道具锚点写完后，没有任何一个支点对应人物压力链的关键瞬间
- 三份文件可以互不引用、各自单独阅读——这意味着没有真正校准

`T08-T09` 完成后回看 `T05-T07`，按上面的"三链对照"再做一次校准；只有确实需要补的内容才回改，不重建文件。

推荐顺序：
1. `T04` 先锁项目卖点、受众入口、连载引擎和 AI 制作口径
2. `T05-T07` 按上面五步同轮做故事基底
3. `T08` 用梗概和故事大纲验证这套基底能不能推动全剧
4. `T09` 用节拍和阶段大纲验证中段变奏、阶段回报和尾钩能不能持续
5. `T10` 写分集大纲前，回补必要的角色、规则、场景、道具细节

策划工作流（决策表 + 正式骨架双层，不是双份维护）：
- 进入每个 T 节点前，先按 [03-策划模板.md](../assets/templates/03-策划模板.md) 决策表答 3-5 个必答问题；只在对话里答，不落盘
- 决策答清楚后，直接在 [25-正式策划产物骨架模板.md](../assets/templates/25-正式策划产物骨架模板.md) 对应段填正式文件
- 决策表是脚手架（用完即弃），25 骨架是正式产物（最终落盘）；不允许写完决策表再抄一遍 25 骨架
- 决策表答到一半发现卡住，那个卡住点就是当前的根本问题；不要先把字段填满再回头

策划阶段的模型复检仍分两道：
- `策划基底质检`：模型重新读取 `T04-T09`，判断故事引擎、人物压力链、世界规则、阶段回报是否已经形成可判断版本
- `大纲质检`：模型重新读取 `T10`，逐集判断存在性、承载力、观看回报、尾钩接口是否足以支撑正文

`T09` 的 `阶段大纲.md` 对 `30` 集及以上项目默认必出；`20-29` 集如果中段形态明显变化、双时间线、多主线或强设定，也应补出。

故事大纲厚度门禁：
- `故事大纲.md` 不能只写五个阶段标签或复述一句梗概
- 每个阶段至少写清 `阶段目标 / 主要阻力 / 推进方式 / 关键翻面 / 阶段回报 / 接口遗留`
- `故事大纲.md` 自身要能判断全剧升级链是否成立；不能让 `分集大纲.md` 单独替它兜底
- 如果读完故事大纲仍不知道主角如何一步步取得筹码、对手如何升级压迫、中段如何变形、终局如何兑现，先回改 `T08`

进入正文前必须满足：
- 全量 `分集大纲.md` 已整合为单一正式文件
- 已通过独立 `大纲质检`
- 评审结论已同步写入 `项目状态.json` 和 `90-内部工作稿/质检检查点.md`

## 策划步骤质检

每一步完成后，至少在本轮回复或 `质检检查点.md` 中给出 `通过 / 需修改 / 阻塞` 结论。结论必须来自模型对内容的复读和判断，不能来自脚本打分或字段扫描。

最小质检问题：
- `T04`：目标受众、核心卖点、题材组合、AI 制作口径是否清楚
- `T05`：规则、代价、压迫方式是否会影响人物行动和剧情推进
- `T06`：主角欲望、反派压迫、关系拉扯、关键配角功能是否能推动主线
- `T07`：常驻场景和关键道具是否服务冲突、视觉记忆和制作稳定
- `T08`：梗概是否定清主戏；大纲是否形成升级链，而不是单纯复述梗概
- `T09`：关键翻面是否改变局势；阶段主导动词、阶段高潮、阶段尾钩和高复用场景是否清楚
- `T10`：每集存在性理由、观看回报、结尾钩子和下一集接口是否成立
- `T10`：相邻集观看回报是否拉开；不能只换证物、地点或集号，仍然重复同一种“主角让对手失控”
- `T10` 还要按目标集长检查承载力：`60-90` 秒主流单集默认优先 `3-4` 场；如果只写 `2` 场，必须写清每场内部的动作升级、翻面或空间切换，否则先回改分集大纲
- `T10` 不能只写功能链。每集必须补 `本集剧情拍点` 和 `本集场次承载表`：剧情拍点默认 `5-7` 个，场次承载表必须写清每场的关键动作、阻力升级、翻面或余波和预计正文行数
- `T10` 对 `60-90` 秒常规集，默认要求全集预计有效正文行数不少于 `28` 行；对 `90-150` 秒首集，默认不少于 `40` 行。低于此值不是绝对禁止，但必须在 `本集时长承载判断` 里说明为什么仍然能撑住
- `EP001`：如果项目按首集加长设计，分集大纲必须写清首集建制任务、建议时长和多出来的戏剧单位；不能只是常规集换成第一集
- 追更设计：分集大纲必须写清剧作层的 `本集观众一句话 / 本集情绪回报 / 本集尾钩 / 尾钩如何改变下一集目标、危险、关系或信息差`
- 商业卡点：仅当用户明确商业化模式时，按 [24-卡点设计.md](24-卡点设计.md) 追加 IAA / IAP / 平台付费字段；商业化后置项目不得强制填写广告位、激励视频、付费触发位或 S/A/B 档

## 藏与显影机制

`藏` 是精品剧作的线索设计，不是把基本剧情写得看不懂。

定义：
- `显性事件`：观众当下必须看懂的人物目标、冲突、输赢和后果
- `暗藏信息`：暂不说破的真相、关系、规则、动机或情绪
- `可发现线索`：观众能从动作、道具、屏幕字、人物反应、环境矛盾或台词避让里察觉的异常
- `显影节点`：后续某一集或某个阶段把暗藏信息变成新理解的翻面
- `回收后效果`：显影后能改写观众对前文的理解，而不是只补一条新资料

硬边界：
- 开篇主危机、主角目标、当集输赢和可见后果不能藏；藏的是第二层意义
- 不用“藏”替代必要交代；如果观众第一遍不知道谁想要什么、谁在阻止、发生了什么后果，这不是藏，是信息缺失
- 不用台词解释线索；线索优先落在可见细节、动作错位、道具状态、屏幕字异常和他人反应上
- 每个长期暗藏信息必须有回收计划；只埋不收、回收时对不上前文，默认不合格

策划层最小动作：
- `故事大纲.md`：对核心真相写清 `显性推进 / 暗藏信息 / 首次线索 / 显影阶段 / 回收后改写什么`
- `故事节拍表.md`：重大揭露类节拍必须标明回收了哪些前文线索，不能空降
- `阶段大纲.md`：每阶段只承担 `1-2` 个主要暗藏信息；不要同时藏太多真相导致主线发散
- `分集大纲.md`：关键集或悬疑推进集写清 `本集显性事件 / 本集暗藏信息 / 可发现线索 / 回收接口`。常规推进集可以合并成一句，不强制每集开长字段

常见可用藏法：
- `道具反常`：同一物件在前后状态不一致，例如钥匙磨损、芯片权限、戒指裂痕、氧气腕表提前清零
- `反应错位`：人物对某个名字、地点、标志或声音反应过度或慢半拍
- `台词避让`：角色始终不说某个词，或用更安全的称呼替代真关系
- `屏幕字异样`：系统弹窗、名单、文件、监控时间戳和现场说法对不上
- `空间矛盾`：官方说封闭不可达，但场景里留下通行痕迹
- `能力/规则先后果`：先让观众看见后果，再在后续显影规则

返工信号：
- 角色直接说出幕后真相，导致前文线索失去乐趣
- 只在结尾突然补资料，前文没有任何可发现线索
- 线索只是作者知道的暗号，观众看不到或看不懂异常
- 全集都在藏，导致当集没有明确回报

## 文件分工

### `项目设定.md`

职责：回答这部剧为什么会被点开、为什么能连载、为什么能制作。

建议按 `7` 个 section 组织：
- `市场与组合`
- `故事引擎`
- `密度与节奏`
- `人物代入与出场`
- `钩子悬念拉扯`
- `题材策略与差异化`
- `资产与制作`

写法：
- 第一轮先写项目基底，不要一开始把整份设定写满
- 第二轮在大纲和节拍稳定后补深化项
- 不堆方法名词；每个字段都必须能指导后续大纲和正文

`故事引擎` section 必须直接引用 `00-立项/锁题摘要.md` 的对应字段，不在策划阶段重新发明：
- `连载引擎一句话` → 引锁题摘要同名字段
- `核心压迫来源` → 引锁题摘要同名字段
- `阶段回报节奏` → 引锁题摘要同名字段
- 在引用基础上扩写 `压迫→爽点映射`：写清"这个具体压迫如何让阶段爽点成立"；写不出来则压迫或爽点之一不成立，回改

`故事引擎` 写完必须做一次闭环校验：把 `观众入口 → 主角欲望 → 核心压迫 → 阶段爽点 → 连载引擎` 串成一句话；如果串不通顺、其中一环抽象到无法举出具体动作或机构，回改对应字段或回改锁题摘要，不带断点进 T05。

### `世界观设定.md`

职责：回答故事世界的秩序、规则、禁忌、代价和后果。

最小内容：
- 世界观一句话
- 时空与覆盖范围
- 公众秩序 / 私密秩序
- 明规则 / 潜规则
- 阵营与权力结构
- 代价与禁忌
- 世界如何压迫主角
- 世界后果与结局变化

字段填写硬标准：
- `主要规则` 字段：每条规则必须能在人物压力链上找到一个对应的具体压迫动作；填了规则却没有对应压迫的，回改
  - 不算成立：`命格不可逆 / 寿命可被转移 / 见证人不在场则失效`（只写规则名，没有压迫动作）
  - 算成立：`命格不可逆——主角试图改命，结果同僚集体认不出他、名册上空了一格`（规则 + 具体压迫后果）
- `代价与禁忌` 字段：必须写"违反后具体发生什么"，不允许写"会有严重后果"
  - 不算成立：`违反者将受到严厉惩罚 / 触犯禁忌的人会消失`
  - 算成立：`触犯禁忌的人，名字会被名册抹去，记得他的人会逐渐忘记他的脸`
- `世界如何压迫主角` 字段：必须举出 ≥ 2 个具体压迫场景或动作
  - 不算成立：`世界对主角充满敌意 / 主角处处受限`
  - 算成立：`EP001 同僚视而不见；EP005 名册空一格；EP012 旧纸上的字开始消失`

规则：
- 现实题材也需要轻量世界观，用来说明城市、行业、家族、媒体、司法、校园或阶层秩序怎么运转
- 不写百科式 lore，只写会进入主线、人物关系和阶段推进的内容
- 不和 `项目设定.md / 故事梗概.md / 故事大纲.md` 重复讲故事经过

### `故事梗概.md`

职责：回答这是什么戏、为什么抓人、为什么值得继续做。

最小内容：
- 类型与题眼
- 主角与开局困境
- 主目标
- 主要对手与核心对抗
- 主卖点与续看理由
- 中段升级
- 终局兑现

规则：
- 可以写出结局，不为“保悬念”故意藏终局
- 不写成逐集列表，不写成节拍表缩写
- 只保留推动主线的世界规则和代价，不搬运世界观文件

### `人物小传.md`

职责：锁主要角色的剧情功能、行动逻辑、关系位置和最小资产锚点。

字段填写硬标准（每个表格字段都必须给出"具体动作 + 具体后果"，模糊抽象词不算成立）：

- `剧情功能` 字段：必须写具体功能性动作，不允许只填角色定位标签
  - 不算成立：`主角的朋友 / 主角的对手 / 工具人`
  - 算成立：`在 EP005 给主角递关键证据；在 EP012 临阵反水拖延时间`
- `想要什么` 字段：必须写可见的、阶段性的具体目标，不允许抽象欲望
  - 不算成立：`想要复仇 / 想要爱情 / 想要被认可`
  - 算成立：`想拿到母亲死前留下的医疗记录，证明继母谋杀；想在年会公开订婚`
- `被谁/什么压迫` 字段：必须写具体机构 / 人物 / 道具，不允许抽象压迫
  - 不算成立：`被命运压迫 / 被社会规则压迫 / 被恶势力压迫`
  - 算成立：`被继母联合家族律师用医疗记录压迫；被 NDA 锁住嘴；被切流封号`
- `反击方式` 字段：必须写至少一个可见动作 + 阶段，不允许只写抽象反击
  - 不算成立：`智取 / 反击 / 翻盘 / 用智慧战胜对手`
  - 算成立：`EP008 用录音反向曝光；EP015 借家族律师的对手律师立案；EP022 当众撕碎 NDA`
- `与主线的关系` 字段：必须写具体推动 / 阻碍 / 转折动作，不允许只写关系标签
  - 不算成立：`主角的朋友 / 关系密切 / 重要支线`
  - 算成立：`EP010 因为相信继母而误导主角；EP018 反水提供关键线索；从此与主角并肩`

任何一个字段填的内容若可以"换到任意一部剧也成立"，都默认未达标，回改。

必须覆盖：
- 主角
- 核心关系位
- 核心对手
- 主要配角
- 关键配角 / 功能角色

主要角色最小内容：
- 身份
- 特征
- 境遇
- 行动
- 终局状态
- 外形关键词
- 固定识别锚点
- 常用服装层
- 首次出场必须交代的信息

主角额外要能看出：
- 可投射身份
- 观众站队点
- 开局受压点
- 翻盘入口
- 强反击方式
- 平凡面 / 不平凡面
- 首次出场核心印象和方式

核心关系位额外写清：
- 关系饥饿点
- 第一次关系翻面
- 与主角的核心拉扯方式

核心对手额外写清：
- 独家压迫方式
- 首次压迫动作
- 被主角反击后的升级方式

### `故事大纲.md`

职责：回答整部剧按阶段怎么推进、升级、失控并收束。

最小内容：
- 全剧升级链一句话
- 开篇与卷入
- 第一阶段推进
- 中段升级
- 后段失控
- 结局收束

每个阶段至少包含：
- 阶段目标
- 主要阻力
- 推进方式
- 关键翻面
- 阶段回报
- 接口遗留
- 藏与显影口径（仅关键真相必写）：本阶段显性推进什么、暗藏什么、哪处线索会在后续显影

规则：
- 不写成逐集列表
- 不复述故事节拍表
- 每段优先写阶段目标、主要阻力、推进方式、关键翻面和阶段回报
- 推进语汇和回报语汇必须符合当前题材
- 如果 `故事大纲.md` 的判断信息主要依赖 `分集大纲.md` 才能看懂，说明大纲偏薄，先补 `T08`

### `故事节拍表.md`

职责：钉住全剧真正翻页的关键节点。

最小内容：
- 引爆事件
- 主角被卷入主线
- 第一次翻面
- 第一阶段小高潮
- 中点翻面
- 假稳 / 假胜
- 重大揭露
- 最低谷
- 决战前选择
- 终局对决
- 结局余波

规则：
- 不是故事复述，也不是逐集列表
- 每个节拍优先回答：这一下之后，局势改变了什么
- 关键节拍尽量补出对应集数或阶段落点
- 重大揭露、背刺、身份翻面、规则真相类节拍必须写清前文有哪些可发现线索被回收

### `阶段大纲.md`

命中以下情况时产出：
- `总集数 >= 30`：默认必出
- `20-29` 集且中段戏剧形态明显变化：强推荐
- 双时间线 / 多主线 / 强设定项目
- 高概念改编 / 母体重构项目
- 资产重项目

最小内容：
- 每个阶段负责推进什么
- 阶段主导动词
- 阶段高潮
- 阶段尾钩
- 阶段反常识翻面点
- 阶段传播级高光
- 阶段优先复用的常驻场景

### `分集大纲.md`

职责：正文前最后一层低成本修正点，回答每一集怎么成立。

分集大纲是正文的底稿，不是文学提案。它必须先用普通观众能听懂的话写清人事因果，再补项目术语。

通俗化硬要求：
- `一句话剧情` 优先写成 `谁遇到什么事 / 谁阻止他 / 最后发生什么后果`，不要写成主题句、设定句或文学标语
- 每集先补 `本集观众一句话`，不用内部术语也能讲清本集发生了什么
- 专有词第一次出现时，旁边必须有日常解释或可见后果，例如 `许平三十年寿命被转给周临` 优于 `寿数债显形`
- 少写 `阶段高潮 / 规则反噬 / 身份危机 / 关系翻面 / 秩序崩塌` 这类抽象词；必须写清谁被抓、谁失声、谁变老、谁公开露馅、谁下一集要做什么
- `本集主要回报` 不写“观众明白玩法 / 证明规则成立”这种作者视角，改写成观众看见的结果：`许平能说话了`、`周临命纹裂黑`、`命镜照不到沈砚`
- `结尾钩子` 必须是普通观众能追问的具体问题，例如 `柳照雪命牌背后还有一个被盖住的字`，不要只写 `下一层秘密出现`
- 高概念项目的分集大纲语言要比正文更通俗；正文可以有风格，大纲必须像给执行编剧的白话任务单

文件开头先放 `分集总览表`：
- 集数
- 一句话剧情
- 本集观众一句话
- 本集爆点
- 本集回报类型
- 本集主要回报
- 结尾钩子
- 藏与显影提示（仅关键集填写一句：埋线 / 显影 / 回收 / 无）

每集至少写清：
- 本集定位
- 本集观众一句话（低术语版本）：合并旧版 `本集观众一句话 + 本集低术语口径` 为一个字段，要求一行讲清"谁遇到什么事 / 谁阻止他 / 最后发生什么后果"，且把规则 / 设定 / 机构名翻成具体人事后果
- 本集付费触发位（仅付费项目必填：`是 / 否`；若是，标注 `S/A/B` 强度档与对应付费层级）
- 开场承接
- 本集主要出场角色
- 本集主要冲突
- 本集回报类型
- 本集主要回报
- 首集建制任务（仅 EP001 必填）
- 本集即时目标
- 本集阻力来源
- 本集过程链（因果主链）
- 本集剧情拍点（默认 `5-7` 个可拍动作拍点）
- 本集场次承载表（逐场写 `戏剧任务 / 关键动作 / 阻力升级 / 翻面或余波 / 预计正文行数`）
- 本集时长承载判断
- 本集存在性理由
- 本集情绪走向
- 本集关键信息点
- 本集显性事件 / 暗藏信息 / 可发现线索 / 回收接口（关键集或悬疑推进集必写；常规集可压成一句）
- 本集主要场景
- 本集视觉支点
- 本集关键动作 / 翻面
- 预计场次数
- 结尾钩子
- 下一集接口

增强项按项目需要补：
- 本集题眼回响
- 本集关系饥饿点
- 本集公众后果
- 本集是否承担奇观场
- 奇观场叙事职责
- 奇观场核心视觉符号
- 奇观场可拆分段落

硬标准：
- 每集都要说明“为什么必须单独存在”
- 每集都要能被压成一句不用项目术语的普通话；压不出来，先回改大纲，不进入正文
- 分集大纲不能用文学性表达替代剧情动作；如果一句话里只有 `命债 / 天命 / 反噬 / 遮蔽 / 余孽` 等术语，却没有具体人和具体后果，默认不合格
- 每集都要先过 `可制作承载`：如果目标是 `60-90` 秒，常规集默认至少 `3-4` 场、`5-7` 个剧情拍点、`28` 行以上有效正文空间；首集 `90-150` 秒默认 `4-5` 场、`7-10` 个剧情拍点、`40` 行以上有效正文空间
- `本集剧情拍点` 不能写成 `尾钩落地 / 场域压迫 / 主角反击` 这种任务标签；要写成“谁在什么场景做了什么，被谁/什么阻止，因此发生什么后果”
- `本集场次承载表` 是正文前的硬门槛。没有写清每场内部的第二阻力、第二反应或余波，就不进入正文
- 批次内先做回报变体池，至少让 `反击 / 证据 / 关系 / 身份 / 权力 / 情绪 / 规则 / 代价` 中的若干类轮换
- 相邻集的本集主要回报不能只换名词；回报必须改变不同层面的状态，例如输赢、关系、入口、身份、资源、舆论或规则
- 首集要作为建制集处理：先打情绪和题材承诺，再交代主角处境、核心痛点、第一次压迫、第一次主动动作、完整小回报和 EP002 追更问题
- 结尾钩子必须具体改变下一集目标、局势、关系或信息差
- 视觉支点要写成物体、空间、光影、动作的组合，不写抽象氛围词
- 如果一集只能概括成 `确认一件事 / 进入一个地方 / 拿到一份资料 / 承上启下`，默认先怀疑它是功能桥段

## 复杂项目补充

高概念 / 母体重构：
- 新鲜感优先来自 `关系翻面 / 规则代价 / 权力重组 / 欲望重写`
- 不主要靠每集新增地点撑新鲜感

专业题材：
- 提前写清降理解成本策略
- 专业信息必须翻成观众一眼能看懂的输赢、风险或代价

双时间线：
- 提前写清两条时间线如何碰撞
- 过去线必须持续给现在线提供压迫、回报或反杀

现实题材：
- 提前写清夸张上限
- 可以强情绪和强压迫，不随手写明显失真的生理、时间、程序和社会反应

资产重项目：
- 提前区分常驻角色、阶段角色、一次性角色
- 提前区分常驻场景、阶段场景、一次性场景
- 核心道具写清持有人、状态变化和是否需要特写
- 必须按 [23-AI漫剧制作约束.md](23-AI漫剧制作约束.md) 校准角色识别锚点、视觉系统、规则后果和下游接口；资产多不是阻塞项，但功能不清、状态变化不清或只靠换场景撑新鲜感时，先回改世界观或大纲

资产表（场景/道具锚点）字段填写硬标准（防止填成抽象空泛词）：
- `叙事功能` 字段：必须写至少一个具体冲突或翻面动作，不允许写"建立压力 / 制造氛围 / 推动剧情"
  - 不算成立：`建立压力 / 推动主线 / 制造冲突`
  - 算成立：`第一次受压在此场景；EP005 主角在此发现继母伪造的医疗记录；EP012 在此公开撕碎 NDA`
- `视觉记忆点` 字段：必须写具体物体 / 颜色 / 光影 / 动作组合，不允许写"压抑 / 紧张 / 神秘"
  - 不算成立：`氛围压抑 / 紧张感强 / 神秘`
  - 算成立：`长走廊尽头一盏只剩半亮的水晶灯；继母的手永远戴着母亲的祖传戒指；雨夜窗格映出十字阴影`
- `复用方式` 字段：必须写在哪几集复用 + 每次复用如何改变状态，不允许写"多次出现"
  - 不算成立：`贯穿全剧 / 多次出现 / 关键场景`
  - 算成立：`EP001 主角入门时空荡；EP008 满座宾客打脸；EP015 暴雨夜门被反锁；EP022 主角带律师重返`

## 海外项目规则

### 文档语种

- `世界观设定 / 故事梗概 / 故事大纲 / 人物小传 / 项目设定 / 分集大纲` 一律中文
- 角色、场景、道具可以用 `中文 / English` 双名
- 不在策划层直接写英文整段梗概

### 本地化深度（不是只翻译，是重写）

底层逻辑：海外项目不是"国内剧本翻成英文"，而是"用美国当代社会纹理重新组织故事"。`/策划` 阶段必须先回答：

- 这个项目的题材在美国语境里 `天然成立` 还是要 `重新构造`
- 主角的身份、动机、阻力是否符合美国当代用户能秒懂的关系结构
- 主要矛盾是否落在美国语境下真实存在的机构、规则、阶层差异上
- 反派的压迫方式是否符合美国当代真实可发生的权力机制（HR 投诉 / 法律手段 / 阶层歧视 / 媒体审判 / 群聊截屏曝光），而不是国内式"切流 / 限号 / 网信办"
- 视觉符号（场景、服装、道具、地标）是否能让美国观众一眼识别为本地化作品

参考词库：[16-美国英语市场选题与设定词库.md](16-美国英语市场选题与设定词库.md)，特别是 `四点五、文化锚点与美国当代社会纹理`。

### 国内母体 → 美国市场 trope 重写矩阵

如果项目来自国内母体或受国内爽剧启发，不要直译，先按下表重写：

| 国内 trope / 设定 | 美国市场不可直译原因 | 推荐重写方向 |
| --- | --- | --- |
| 霸总抓手腕 / 强吻 / 公开命令 | 读作 toxic / harassment / abuse | `Dark romance`（明确风格化、双方知情）或 `protective CEO`（保护欲不是控制欲） |
| 家暴 + 复合 | 不可接受 trope | 改成 `离开 + 自我重塑 + 复仇` 弧光 |
| 抢婚 / 强娶 / 包办婚姻 | 读作 forced marriage 红线 | 改成 `last-minute objection at the wedding` 或 `arranged marriage with consent` |
| 切流 / 限号 / 封号 / 网信办 | 美国不存在对应监管机构 | 改成 `algorithm shadow ban / brand sponsor pulled / cancellation campaign / lawsuit threat / NDA` |
| 学区房 / 985 / 高考 | 没有对应制度 | 改成 `school district / Ivy League / Common App / SAT / legacy admission` |
| 户籍 / 农村户口 / 北漂 | 没有对应制度 | 改成 `green card / undocumented / first-gen American / moved from Ohio to NYC` |
| 体制内外 / 考公 / 铁饭碗 | 没有对应制度 | 改成 `government job / federal employee / pension / civil service exam`（但通常不是主推进） |
| 婆媳大战 | 文化语境不同（美国不流行三代同堂） | 改成 `mother-in-law from hell` + `Thanksgiving showdown` 或 `prenup battle` |

| 公开打耳光 / 当众羞辱 | 不是默认爽点，可能被读作 assault | 改成 `viral takedown / receipts dropped / public callout / social media destruction` |
| 绿茶 / 白莲花 | 没有对应文化标签 | 改成 `frenemy / two-faced / queen bee / mean girl` |
| 凤凰男 / 捞女 | 阶层 + 性别污名标签 | 改成 `gold digger / social climber`（要小心，避免性别歧视） |
| 直播带货 / 切片号 / MCN | 美国是 creator economy 但生态不同 | 改成 `influencer / creator / TikTok shop / brand deal / sponsorship / agency / management` |
| 家族祠堂 / 族谱 / 嫡庶 | 没有对应制度 | 改成 `family foundation / family trust / dynasty / boardroom succession`（big money 题材） |
| 重生 / 穿越古代 | 重生 trope 在美国可用，穿越古代要改 | `重生` = `reincarnation / rebirth / second chance / time loop`；`穿越古代` 通常改成 `regency / Victorian / fantasy kingdom` |
| 修仙 / 玄幻 / 仙侠 | 美国主流市场不熟悉 | 改成 `urban fantasy / paranormal / supernatural / witches / vampires / werewolves` |
| 系统流 / 金手指 | 美国小众但有圈子 | LitRPG / progression fantasy / isekai（动漫圈） |
| 宫斗 / 后宫 | 不是主流市场题材 | 改成 `royal court intrigue / Tudor-style / boarding school clique` |

| 美国 trope / 设定 | 国内市场不可直译原因 | 推荐重写方向 |
| --- | --- | --- |

### 重写过程要求

- `/立项` 阶段：在 `项目简报.md` 显式写"是否为国内母体"，如果是，必须列出至少 `3` 条最大本地化重写动作
- `/策划 T04` 阶段：在 `项目设定.md` 的 `市场与组合 / 题材策略与差异化` section 显式写"美国语境天然成立点"和"必须改写点"
- `/策划 T06` 阶段：人物小传里主角的 `开局受压点 / 翻盘入口 / 强反击方式` 必须用美国语境真实可发生的机制
- `/策划 T08-T09` 阶段：故事大纲和节拍表的关键翻面必须落到美国可识别的机构、节日、阶层冲突上

### 视觉与资产本地化

- 角色：欧美面孔为默认；除非项目刻意做亚裔主角（Asian American romance / first-gen drama）
- 场景：地标、街道、招牌、车牌、邮箱、门牌号要美式；不要中式商铺招牌或简体中文
- 道具：手机品牌（iPhone 默认）/ 报纸（NYT / WSJ）/ 咖啡（Starbucks / Dunkin'）/ 车（pickup / SUV / Tesla）
- 服装：按角色阶层和场景给具体锚点（Patagonia 中产 / Lululemon 主妇 / Brooks Brothers 老钱 / Off-White 新钱 / thrift 学生）

### 海外子市场切换提醒

- 美国英语首版默认即可；如果用户明确要做 UK / 加拿大 / 澳大利亚版，词库锚点要切换（UK：A-levels / NHS / council estate / Oxbridge；加拿大：universal healthcare / Tim Hortons / Toronto vs Vancouver；澳大利亚：HSC / Bondi / outback）
- 西语 / 葡语市场：另起词库，不在本 skill 默认覆盖

## 过关条件

- 故事能一句话讲清
- `故事梗概` 能讲清这是什么戏、为什么抓人、为什么值得继续做
- `故事大纲` 能讲清阶段推进、升级、失控和收束
- `故事节拍表` 钉住关键翻面
- 主要角色齐全，主角、关系位和对手都能直接指导正文
- 题材引擎能撑住目标集数
- 开篇抓手、续看引擎、情绪兑现引擎和回报升级方式成立
- 如果是专业题材、双时间线、现实题材或资产重项目，特殊风险已经前置设计
- `分集大纲.md` 已完整并通过大纲质检
- 长线项目的 `分集大纲.md` 已按阶段和窗口整理，便于检查中段推进和重复风险
