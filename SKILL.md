# AI漫剧策划与剧本生成

## 适用边界

这个 skill 只服务 `AI 制作的竖屏剧 / 漫剧`，不服务真人拍摄短剧。

固定边界：
- 策划层统一用中文输出
- 平台口径固定拆分：国内发行平台写 `抖音`；出海发行平台写 `TikTok`
- 目标市场单独填写：`国内` / `美国英语首版` / `泛欧美` / `东南亚` / `拉美` / `指定国家或地区`
- 出海项目只有 `剧本对白 / 屏幕字 / 必要制作语种字段` 使用目标语；未指定时美国英语首版使用英文
- 剧本阶段先服务故事质量、类型承诺、人物欲望、情绪回报和 TikTok 竖屏观看语法；商业化模式（IAA / IAP / 平台定制）默认后置，只有用户明确要求商业化设计时才前置
- 所有交付文件统一使用 `Markdown`
- 最终剧本正文统一采用 `场次执行稿`
- 剧本可以分批生成，但最终正文不按批次目录交付
- 本 skill 只负责到 `场次执行稿` 这一层，不生成分镜、视频提示词、配音脚本和音效表；下游的 `分镜生成器 / 视频生成器 / 配音引擎` 是独立流水线，本 skill 仅按 `production-enhanced` 要求在交付增强包里补齐与下游对接所需的统一资产命名和场景/道具卡
- AI 制作能力不作为题材降级理由。奇幻、异世界、狼人、女巫、吸血鬼、末日、规则世界、大场面等都可以进入候选；选题阶段只判断视觉锚点、角色识别、世界规则和情绪引擎是否清楚
- `/立项` 沟通阶段只在对话中做调研、比较和候选方向汇总；用户没有明确确认前，不初始化项目目录，不落任何立项文件，不进入 `T02 立项-目标锁定`

## 核心目标

最终目标是产出有吸引力、可交付、能让观众继续看的剧本，不是完成模板。

最低质量标准：
- 立项能说明观众为什么点开、为什么追下去
- 策划能说明人物欲望、关系张力、反派压迫和阶段回报如何持续升级
- 分集大纲能说明每集为什么必须存在
- 正文能在场面里呈现开场抓力、可见冲突、动作推进、情绪变化和尾钩拉力
- 交付包能让制作方直接执行，不需要反查工作稿
- 全链路本地化深度成立：题材 / 矛盾 / 压迫机制 / 爽点 / 视觉锚点 / 对白都长在目标市场真实社会纹理或目标市场熟悉的类型语法里，不是直译版

商业化原则：
- 未明确商业化时，不用 IAA / IAP / 付费阶梯倒推选题或大纲
- `卡点` 在剧作层先理解为“让观众继续看的具体戏剧问题”，不是默认广告位或付费点
- 用户明确要求投流、IAA、IAP、平台付费或商业化大纲时，再读取 `24-卡点设计.md` 并追加商业字段

本地化是这部 skill 的独立维度，贯穿立项到交付：
- `/立项`：用户明确确认方向后，在 `项目简报.md / 锁题摘要.md` 锁母体来源市场和跨市场重写动作
- `/策划`：在 `项目设定.md` 锁目标市场原生承诺、原生压迫机构或 genre-native 类型承诺、价值观爽点、地域 / 阶层 / 节日 / 奇幻规则锚点
- `/剧本`：对白按 `12-单集写作协议.md 第七章` 做口语化和本地化
- `/复盘`：用 `13-质量检查表.md` 的"市场本地化深度检查"做硬门槛

对白是单一真相源在 [references/12-单集写作协议.md 第七章](references/12-单集写作协议.md)；项目级文化锚点池在 [references/15-国内选题与设定词库.md](references/15-国内选题与设定词库.md) 和 [references/16-美国英语市场选题与设定词库.md](references/16-美国英语市场选题与设定词库.md)；跨市场 trope 重写矩阵在 [references/02-策划细则.md](references/02-策划细则.md)。

## 主流程

用户视角的完整流程分四段：

`立项与选题 -> 剧本圣经设计 -> 分集剧本创作 -> 交付整理`

剧作创作主链按这个顺序理解：

`创意 -> 梗概 -> 大纲/情节点 -> 剧本初稿`

对应关系：
- `创意`：`T01-T04`，加上 `T05-T07` 的最小可写性校准。这里只锁观众入口、主角欲望、压迫来源、关系张力、世界规则、核心场景/道具锚点；不要扩写成设定百科
- `梗概`：`T08 故事梗概`，先用短文本讲清这部戏到底是什么
- `大纲/情节点`：`T08 故事大纲 + T09 故事节拍表/阶段大纲 + T10 分集大纲`，从全剧推进到关键翻面，再到逐集情节点
- `剧本初稿`：`T13 第XXX集.md`，基于已通过的大纲写场次执行稿；首批可先写 `1-2` 集代表样稿

如果用户已经给了创意，可以直接补齐最小可写性校准后进入梗概；如果用户已经给了梗概，可以直接进入大纲/情节点拆解；如果用户已经给了大纲，可以先做大纲质检，再进入剧本初稿。不要为了流程完整反向补一堆不会影响梗概和初稿的设定文件。

编号任务包不在本文重复维护，统一读取 [references/21-产物矩阵与创建时机.md](references/21-产物矩阵与创建时机.md)：
- `21` 是 T01-T18 的任务包、产物等级和创建时机单一真相源
- [references/17-状态流转与阶段门禁.md](references/17-状态流转与阶段门禁.md) 是 `currentStep`、门禁节点和状态回退单一真相源
- 本文只负责入口路由、四步创作链解释和阶段导航

策划依赖关系：
- `T05-T07` 是同一轮故事基底包，不是三个孤立填表动作
- `T05-T07` 服务于“创意能否写成梗概和初稿”的可写性判断，不应压过 `梗概 -> 大纲/情节点 -> 剧本初稿` 这条主链
- 世界观要独立落成 `世界观设定.md`，但创作时必须和角色压力、反派压迫、常驻场景、关键道具一起判断
- 角色、场景、道具不要完全等大纲后才开始；先做最小可用锚点，保证 `T08-T10` 有行动空间、视觉支点和 AI 制作口径
- `T08-T09` 稳定后，再反向校准人物小传、世界规则、场景/道具锚点；不要提前写百科式世界观或资产大全

执行原则：
- 立项阶段先帮助用户完成市场判断、选题沟通和目标确定；用户未明确确认最终方向和必要立项信息时，只停留在对话讨论，不初始化项目目录，不生成任何立项文件，不进入策划
- 立项选题不要默认推高概念，也不要因 AI 制作误判为只能做现实题材。先判断用户更适合 `主流稳态 / 类型微创新 / AI-native 视觉型 / 高概念创新` 哪种策略，再给候选方向；高概念只是可选策略之一
- 策划阶段按内容依赖顺序推进，每一步完成后先做本步质量分析，没过就回改，不把问题带到后面
- `分集大纲.md` 是正文前硬门槛；未全量完成且未通过 `大纲质检`，不进入 `/剧本`
- `/剧本` 默认按批次写；每批先写 `1-2` 集代表样稿，通过后再铺满本批
- `/复盘` 是显式准入动作，不替代前面阶段的创作判断
- `/交付` 只整合正式源文件，不在交付阶段重写剧情

### 模板字段/章节三档分级（MUST / SHOULD / MAY）

正式模板里参与门禁判断的字段或章节都按 RFC 2119 风格分三档；模板顶部用注释明确标注：

- `MUST`（必填）：缺这个字段就视为流程不完整；独立质检会判 `需修改`
- `SHOULD`（默认填）：默认填，跳过须在阶段回复或质检检查点写明跳过原因
- `MAY`（可选）：常规项目可不填；只在命中具体触发条件时才需要

字段/章节触发条件示例：
- 商业化模式 = 商业化后置 → 不填写 IAA / IAP / 付费阶梯 / 广告卡位字段；只保留剧作层的追更钩、情绪回报和尾钩
- 商业化模式 = IAA → 追加 `本集完播抓手 / 本集追更钩 / 中插广告卡位`
- 商业化模式 = IAP → 追加 `本集付费触发位 / S/A/B 档 / 不可逆翻转`
- 项目类型 = 改编 → `故事理解.md` 触发；原创项目跳过
- 项目类型 = 资产重 → `场景参考卡 / 道具设定卡` 触发

字段/章节跳过的最小动作：
- 模型在阶段回复或 `质检检查点.md` 显式写"跳过 X 字段，原因：商业化后置 / 非改编 / 资产轻"
- 不允许把跳过的字段填成 `待定 / 暂略 / 后补`——这是反作弊红线
- 不允许把字段全部填满当成"完成度高"——SHOULD / MAY 字段在条件不满足时填了也不加分

结构化字段模板顶部注释格式：
```
<!--
字段分级（MUST / SHOULD / MAY）：见每个字段行末标注
跳过 SHOULD/MAY 字段必须在阶段回复或 质检检查点.md 写明原因
不允许填 待定 / 暂略 / 后补
-->
```

## 最小可用路径

`对话内市场判断 / 选题沟通 -> 用户明确确认方向和必要立项信息 -> 初始化项目并生成项目简报 / 锁题摘要 -> 立项质检 -> 最小可写性校准 -> 故事梗概 -> 大纲/情节点 -> 剧本初稿 -> 剧本质检 -> 交付前全剧体验通读复盘 -> 全剧复核 -> 交付整合（按需） -> 交付质检`

默认不要把所有内部工作稿全开。

- `试写样场`：人物声线、首次亮相、核心对峙或高光翻盘不稳时
- `批次写作单`：当前批次承接、旧债回收、代表样稿或关键集节拍不稳时
- `单集达标检查`：当前集准备落稿或返工后准备重新过关时

## 阶段路由

### `/立项`

目标：先在对话中做平台、目标市场、题材和风格沟通；用户明确确认方向、平台、目标市场、规格、AI 风格、受众、语言口径和必要边界后，再初始化项目并落正式立项文件。

分步：
- `T01 /立项-市场与选题`：在对话中判断市场空间、同质化风险、差异化抓手；题材未锁时给用户可选方向，不落盘
- `T02 /立项-目标锁定`：用户明确确认方向和必要立项信息后，初始化项目目录，形成 `项目状态.json / 项目简报.md / 锁题摘要.md`
- `T03 /立项质检`：确认是否能进入策划

核心文件：

### `/策划`

目标：把锁定题材拆成组成剧本圣经的分项策划文件，并让这些文件足以支撑正文创作。

建议顺序：
1. `T04 /策划-项目定位与核心卖点`：`项目设定.md`，锁核心卖点、观众入口、连载引擎、AI 制作口径
2. `T05 /策划-世界观与规则`：`世界观设定.md`，锁秩序、规则、代价、压迫方式
3. `T06 /策划-角色与关系`：`人物小传.md`，锁主角、关系位、反派、关键配角
4. `T07 /策划-场景与道具`：锁关键场景、核心道具、视觉符号；资产重项目另出卡
5. `T08 /策划-梗概与故事大纲`：同一轮生成 `故事梗概.md / 故事大纲.md`，先用梗概定戏，再用大纲定推进
6. `T09 /策划-节拍与阶段大纲`：同一轮生成 `故事节拍表.md / 阶段大纲.md`，30 集及以上阶段大纲必出
7. `T10 /策划-分集大纲`：`分集大纲.md`，正文前最后一层结构修正点；先写普通观众能复述的 `本集观众一句话` 和低术语口径，再补规则名和结构项

`T05-T07` 执行方式：同轮迭代、分别落盘。先做最小可用世界规则、角色压力链、常驻场景和道具锚点；`T08-T09` 稳定后再补校准，不把世界观写成脱离剧情的设定百科，也不把场景/道具拖到分集大纲之后才想。

每一步完成后都先做本步质检分析：`通过` 才继续下一步；`需修改` 先回改当前步或上游依赖文件。策划结束时不额外汇总成大文件，避免上下文过大和重复维护。

- `人物小传.md`

### `/剧本`

目标：基于已通过的 `分集大纲.md` 写分集正文。

核心文件：
- `02-剧本/第001集.md`
- `02-剧本/第002集.md`
- `...`

写作规则：
- 进入正文前必须已经通过 `大纲质检`
- 正式分集稿使用轻量 `本集创作简卡`
- 写每集或每批前按需读取 `项目设定 / 世界观设定 / 人物小传 / 故事梗概 / 故事大纲 / 故事节拍表 / 阶段大纲 / 分集大纲` 等源文件，不读取或维护一个提前汇总的大圣经文件
- 写每集前先做最小构造：`单集情节四链 / 钩子类型 / 信息差结构 / 三种密度 / 展示优先`
- 复杂判断进入 `批次写作单 / 单集达标检查 / 复盘报告`，不塞进正式分集稿
- 每集主体必须是场次执行稿；质量只看正文里的动作、反应、冲突、翻面和可见后果，不能用格式完整替代戏剧成立
- 正式分集稿必须按标准场次格式落稿：`### 1-1 日内 地点`、`人物：`、`△` 动作行、`角色名：` 台词行，必要时使用 `OS：/ VO：/ 屏幕字（类别）：/ SFX：`；不能写成小说段落或纯文学场面
- 场次前置字段只做锚点；如果正文动作/反应/对白轮次不明显多于前置字段，先补正文再放行
- 高概念或强设定项目正文必须先过易读性：每集压成一句普通观众能复述的话，术语每集新增不超过 `1-2` 个，抽象规则必须藏进可见后果，屏幕字不要写成规则说明书
- 可以单集创作，也可以多集一批；默认每批先写 `1-2` 集代表样稿，样稿过关后再铺满本批

## 强规则

- 题材未锁定前，不进入 `/策划`
- 没有全量 `分集大纲.md` 且未通过 `大纲质检`，不进入 `/剧本`
- 剧本正文必须基于已通过的大纲生成
- 每批正文未过 `剧本质检`，不进入下一批
- `项目状态.json` 和 `质检检查点.md` 缺失时，不能把项目视为真正过门禁
- 同一上下文里的自检不能写成门禁放行；没有独立复检证据时，`qcStatus` 保持 `未检查` 或既有状态
- 海外项目策划层统一中文，英文只进入剧本对白和必要制作字段
- `完整剧本总稿.md` 只汇总正式分集 `场次正文`，不包含 `本集创作简卡 / 本集回填 / 复盘 / 检查单 / 批次写作单`
- `剧本圣经.md` 只在交付阶段生成，只整合正式策划内容，不整合内部工作稿
- 如果 `剧本圣经.md` 汇总了场景 / 道具章节，仍保留独立场景 / 道具卡作为制作资产卡 source of truth；修改独立卡后必须重新生成圣经

## 脚本

脚本只做机械可计数的工作：交付汇总 + 单集硬阈值越线检查。质检、审稿、门禁放行必须由模型按 `references/18-过程质检与独立复检.md` 和 `references/19-质量门禁.md` 读实际稿件完成。

### 交付汇总

```bash
python3 skills/short-drama-scriptwriter/scripts/build_full_script.py projects/<project-slug>
```

默认模式会校验 `项目状态.json`、已通过批次和必需策划源文件；草稿模式加 `--draft`（不得作为正式交付放行依据）；mtime 检查模式加 `--check-mode`（只比对源文件 mtime 与交付包，不重生成）。

### 单集硬阈值越线检查

```bash
python3 skills/short-drama-scriptwriter/scripts/check_episode.py \
    --regulation-words "命格,暗文,命债,见证人" \
    projects/<project-slug>/02-剧本/第001集.md
```

检查叠加屏幕字字数 / 画内文字可读性 / 环境标签字 / 规则词频次 / 前置字段比例 / 有效行数等硬阈值；输出可作为 `单集达标检查` 或 `质检检查点.md` 的"硬阈值证据附件"，但**不替代独立复检的戏剧判断**。

详见 `references/18-过程质检与独立复检.md` 的"可机器验证的 clean-context 证据"。
