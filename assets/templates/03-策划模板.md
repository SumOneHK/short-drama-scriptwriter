## 使用规则

- 每次进入新 T 节点前，先在对话里把本节点的决策问题答一遍；答不清不进 25 骨架
- 决策回答只用一段话或几句话，不写大段文章
- T05-T07 用同一组决策问题（同轮迭代）；详见 [02-策划细则.md](../../references/02-策划细则.md) 的"同轮迭代五步法"
- T08 + T09 / T10 可单独决策
- 答完决策表后直接打开 [25-正式策划产物骨架模板.md](25-正式策划产物骨架模板.md) 落正式文件
- 如果某个决策问题答到一半发现卡住，那个卡住点就是当前的根本问题；先解决它再往下

## T04 决策表（项目定位与核心卖点）

进入 T04 前先答：

1. **观众入口**：哪一句话能让普通观众点开第一集？这句话不能用项目术语，要能让朋友圈看到立刻想点开
2. **连载引擎**：观众看第 1 集 / 第 10 集 / 第 20 集，分别拿到的"具体新东西"是什么？答不出 3 个具体新东西就回改锁题摘要
3. **核心压迫**：本剧主角被谁压迫、用什么方式压迫、付出什么代价？必须举得出具体机构名 / 道具 / 程序，不允许"被命运压迫 / 被恶势力压迫"这种抽象描述
4. **压迫→爽点映射**：上面这个具体压迫如何让观众爽点成立？写不出对应关系说明压迫和爽点至少一个不成立
5. **闭环检查**：把"观众入口 → 主角欲望 → 核心压迫 → 阶段爽点 → 连载引擎"串成一句话，串不通顺就回改

答完直接进 [25 骨架](25-正式策划产物骨架模板.md) 的 `01-策划/项目设定.md` 段；详细字段填法见 [02-策划细则.md](../../references/02-策划细则.md) `项目设定.md` 职责段。

## T05-T07 决策表（世界观 + 角色 + 场景道具，同轮迭代）

进入 T05-T07 前先答（注意：先答 1-2，再答 3-4，再答 5；不允许跳过顺序）：

1. **主角压力链**（先做）：主角的"欲望→开局受压→受压来源→反击入口→反击方式"分别是什么？只回答主角，不回答其他角色
2. **世界规则的对应压迫**（再做）：上面"受压来源"在世界规则里叫什么？有什么名字、付出什么代价、谁从规则里获益？规则只保留能直接压回主角的几条
3. **视觉支点（场景/道具）**（再做）：从前两步里挑 2-3 个最关键的"可被压迫的瞬间"，每个瞬间分配一个常驻场景或核心道具
4. **三链对照**（必做）：把人物压力链、世界规则、场景/道具锚点放在一起逐项检查——每一环在世界规则里有名字 + 在场景/道具上有可见落点；任何一环找不到对应，回到第 2-3 步补
5. **AI 视觉系统**：角色识别锚点、常驻场景、关键道具和规则后果分别是什么？详见 [09-规格建议.md](../../references/09-规格建议.md) 第六章 + [23-AI漫剧制作口径.md](../../references/23-AI漫剧制作约束.md)；高复杂度场不直接砍题材，先补清视觉锚点、动作顺序和规则后果

答完直接进 25 骨架的 `01-策划/项目设定.md（资产与制作 section） / 世界观设定.md / 人物小传.md`；详细字段见 [02-策划细则.md](../../references/02-策划细则.md) 各文件职责段的字段填写硬标准。

## T08 决策表（故事梗概 + 故事大纲）

进入 T08 前先答：

1. **题眼**：这部剧用一句话讲，是关于什么的？必须包含主角 + 主目标 + 核心对抗
2. **主角开局到终局的状态变化**：主角从哪个状态开始、终局到哪个状态？两个状态必须有结构性差异（身份 / 关系 / 资源 / 价值观）
3. **中段升级链**：开篇 → 第一阶段 → 中段 → 后段 → 结局，每段主角和对手分别升级到什么程度？写不出 5 个差异化阶段就先回改 T05-T07
4. **终局兑现**：终局给观众一个什么具体回报？不允许"主角终于胜利 / 真相揭露"这类抽象口径
5. **梗概 vs 大纲分工**：梗概只讲"是什么戏 / 为什么抓人"；大纲讲"按阶段怎么推进、升级、失控、收束"。两者不能互相复述

答完直接进 25 骨架的 `01-策划/故事梗概.md / 故事大纲.md`；故事大纲厚度门禁见 [02-策划细则.md](../../references/02-策划细则.md)。

## T09 决策表（故事节拍 + 阶段大纲）

进入 T09 前先答：

1. **关键翻面**：全剧的"引爆事件 / 第一次翻面 / 中点翻面 / 重大揭露 / 最低谷 / 终局对决"分别落在哪一集或哪个阶段？每个翻面必须能写清"这一下之后局势改变了什么"
2. **阶段主导动词**（仅 30+ 集项目）：开篇 / 中段 / 终局前 / 终局 各阶段用哪个动词描述主角的主要动作？（探查 / 反击 / 自我重塑 / 公开清算 / 决战 等）
3. **阶段回报与阶段尾钩**：每段进入下一段前观众拿到的具体回报是什么？阶段尾钩具体是什么？
4. **传播级高光**：每段是否有一处"切片可投流"的传播高光？没有的话这段中段会塌
5. **阶段长度合理性**：30+ 集项目，每段是 8-15 集还是 5-8 集？长度是否符合规格建议？

答完直接进 25 骨架的 `01-策划/故事节拍表.md / 阶段大纲.md`；阶段大纲只在 30+ 集项目（或命中条件的 20-29 集项目）出。

## T10 决策表（分集大纲）

进入 T10 前先答（先按整体 1-3 答，再按每集 4-7 答）：

1. **商业化状态确认**：项目商业化是否已经明确？未明确则写 `商业化后置`，分集大纲只做剧作层追更设计；明确 IAA / IAP / 平台付费定制时，再按 [24-卡点设计.md](../../references/24-卡点设计.md) 追加商业字段
2. **回报变体池**：全剧回报类型（反击 / 证据 / 关系 / 身份 / 权力 / 情绪 / 规则 / 代价）如何在 N 集里轮换？相邻集不重复
3. **场次预算**：按规格建议（[09-规格建议.md](../../references/09-规格建议.md)）确定每集场次数和正文行数下限

每集进入决策（每集都要问）：

4. **本集存在性**：删掉本集会丢什么？答不出"具体不可替代的过程 / 翻面 / 情绪 / 后果"，本集就是功能桥段，回改
5. **本集观众一句话**：能否用一句普通观众能复述的话讲清本集？（含人事因果，不用项目术语）
6. **本集追更设计**：先写剧作层字段（本集观众一句话 / 本集情绪回报 / 本集尾钩 / 尾钩如何改变下一集目标、危险、关系或信息差）；商业化已明确时再追加对应商业字段
7. **本集尾钩具体性**：尾钩是什么具体动作 / 状态变化？能否让观众一句话复述他想知道什么？

答完直接进 25 骨架的 `01-策划/分集大纲.md`；详细字段填法见 [02-策划细则.md](../../references/02-策划细则.md) T10 段。
